# 权益活动中心 MVP 需求确认

## 背景

`我的` 页仍存在 `权益活动`、`求职打印套餐`、`AI服务套餐`、`招聘会扫码凭证` 等建设中入口。P0/P1 已完成账号设置、帮助中心、我的权益、消息通知、意见反馈等闭环，当前可复用的真实底座是 `BenefitGrant`、`/me/benefits`、Admin `/member-benefits`。

## 本轮目标

把 `权益活动` 做成可上线的最小闭环：

1. Admin 配置并发布权益活动。
2. Kiosk 展示已发布且有效的权益活动。
3. 登录用户领取活动。
4. 领取成功后生成 `BenefitGrant`。
5. 用户在 `/me/benefits` 查看领取到的权益。
6. Admin 查看活动领取记录。

## 本轮不做

1. 微信、支付宝或任何线上支付。
2. 求职打印套餐购买。
3. AI 服务套餐购买。
4. 招聘会报名凭证、签到凭证、入场核销。
5. 合作机构端自助配置活动。
6. 资格自动审核、抽奖、裂变分享。
7. 权益真实消费核销。

## 合规要求

权益活动只能发放平台内服务权益、打印额度、优惠券或政策信息提示。不得承诺录用、面试、投递结果、补贴到账，不得把活动领取设计成招聘会报名、签到或平台投递闭环。
