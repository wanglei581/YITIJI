# 需求说明

## 背景

PR #60 已合并到 `main`，完成会员短信登录后的 P0 后端数据隔离：简历提取、AI 结果、文件读取、会员资产均按 `endUserId` 做归属门禁，CI `build-and-verify` 与 `postgres-readiness` 已通过。

本任务补齐用户登录功能闭环剩余代码与文档项，目标是让「短信登录 -> 个人数据独立存储 -> 我的页回看/删除 -> 退出/切换账号不串号 -> 禁用账号失效 -> 数据留存规则明确」成立。

## 必做范围

1. Kiosk 会员 API 统一 401 处理，退出登录后清理本地会员态，切换账号不残留上一账号数据。
2. 会员发送验证码 / 登录请求带 `deviceId`，对齐后端已有设备维度限流入口。
3. 收藏 / 活动写入前服务端校验目标真实存在，并由服务端派生安全标题或快照，避免前端伪造记录。
4. EndUser 账号禁用后，既有 Redis session 立即失效或在 guard 层 fail-closed。
5. 文档化个人数据留存矩阵，并用 verify 覆盖关键规则。

## 非目标

- 不新增重复登录入口或重复「我的」页面入口。
- 不重做 UI/UX 视觉风格。
- 不新增招聘平台闭环能力，不做平台内投递 / 候选人推荐 / 企业筛选。
- 不在本任务做正式生产部署、HTTPS、Windows 真机打印扫描或法务协议完整收口。

## 验收标准

- 本地相关 verify、typecheck、lint 通过。
- 双模型审查无 Critical / Warning。
- GitHub PR CI 通过。
- 后续线上验收可按文档逐项执行。
