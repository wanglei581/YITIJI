# 用户文件与简历资产 Gate 3/Gate 4 执行需求

```text
更新目标，继续推进，直到完成
```

本任务将该确认解释为继续推进用户文件与简历资产商用闭环剩余预生产验收，但仍只在本任务声明边界内执行。

## 前置状态

- 预生产 Gate 2 已通过。
- 预生产 `/srv/ai-job-print` 自报部署候选 `2187f6a7`。
- 本机与公网 health 均已验证为 `success=true`、`db=postgres`。
- Gate 3/Gate 4 尚未执行。

## 目标

- 在预生产环境执行 Gate 3 自动命令证据。
- 在预生产环境执行 Gate 4 受控账号与文件资产浏览器/API 验收。
- 记录脱敏证据摘要，更新执行记录和进度入口。
- 不把 Gate 3/Gate 4 通过误写成正式生产、Windows 真机或试运营完成。

## 非目标

- 不修改正式生产域名、证书、DNS 或 nginx。
- 不修改短信、OCR、TRTC、ASR/TTS 配置。
- 不修改 COS bucket 生命周期规则或 CAM 权限。
- 不触碰真实用户账号、真实简历正文或真实用户文件。
- 不执行 Windows 真机、Terminal Agent、奔图打印、扫描、断网/重启恢复验收。
- 不扩大到小范围试运营。

## 允许的外部写入

- 受控测试会员账号和测试会话。
- 受控测试文件和对应 COS 对象。
- 受控测试文件的保存期限变更、删除状态和过期清理状态。
- 与本轮测试文件相关的 AuditLog 记录。
- 预生产服务器上的临时命令日志和脱敏证据摘要。
- 浏览器截图，前提是截图不包含 token、完整手机号、签名 URL 查询串或简历正文。

## 禁止写入

- 生产密钥、`.env` 内容、云平台配置。
- 非测试会员、非测试文件、真实简历正文。
- COS bucket 级生命周期规则。
- 正式生产数据库、Redis 或 COS bucket。

## 停止条件

- 任一命令或浏览器链路泄露密钥、token、完整手机号、完整签名 URL 或简历正文。
- G3-06 执行前无法正向证明当前 COS bucket 为预生产用途。
- `verify:cos:live` 为 SKIPPED 或 FAIL。
- 会员 B 能访问会员 A 文件。
- 删除后三态不一致：UI、PostgreSQL、COS 状态任一不符。
- `long_term` 对照被过期清理误删。
- AuditLog 缺少保存期限变更、删除或过期清理记录。
- 需要修改非目标或禁止写入范围内资源。

## 验证

- Gate 3：G3-01 至 G3-09 自动命令证据。
- Gate 4：G4-01 至 G4-10 受控账号、文件、COS、DB、AuditLog 和 Admin 生命周期视图证据。
- 本地：`verify:file-assets-trial-acceptance`、API typecheck、`git diff --check`、本次变更敏感扫描。

## 回滚/清理

- 测试文件删除后必须验证 PostgreSQL deleted、COS 404、UI 不可见。
- 过期清理只允许命中本轮测试文件 ID；如需模拟过期，只允许把本轮受控测试文件的 `expiresAt` 回拨到过去，不得改真实用户文件。
- G4-09 前必须确认过期清理窗口内除本轮测试文件外无其它会被误清理的文件；否则必须停止或改走按测试文件 ID 限定的手动路径。
- G4-01 前必须记录测试会员/管理员账号开通方式，并证明不修改短信、OCR、AI、TRTC、ASR/TTS 配置。
- G4-06 签名 URL 过期验证优先使用短 TTL 或破坏过期参数验证失效，不等待最长 30 分钟。
- 预生产日志和截图只保留脱敏版本；仓库只提交摘要，不提交原始证据文件。
