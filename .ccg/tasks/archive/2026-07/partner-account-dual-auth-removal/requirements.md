# 合作机构账号双通道验证与安全移除：需求基线

## 用户确认的目标

- Admin 在既有合作机构详情中发起机构成员账号移除。
- 默认向待操作 Partner 账号已绑定且已验证的手机号发送验证码。
- 手机号无法接收验证码时，可切换为输入待操作账号本人的当前登录密码。
- 密码验证成功后，允许选择换绑手机号或移除账号。
- 用户已确认推荐方案，并明确授权多子代理与 Claude 协作完成实施和终审。

## 审查后必须保留的安全边界

- 继续使用墓碑式逻辑移除，保留历史主键与审计关联。
- 机构始终至少保留一个已启用且未移除的 Partner 账号。
- 删除、换绑均需要目标账号本人授权；验证票据绑定 Admin、机构、目标账号、目标操作和凭据版本，短期有效且单次消费。
- 短信验证码按用途隔离；密码与验证码均限流，任何日志和审计不得记录明文凭据。
- 换绑必须再次验证新手机号；删除或换绑后递增 tokenVersion 并发布禁用/新版本会话状态。
- 必须处理 Admin 可重置目标密码造成的本人验证绕过风险，不能只做表面密码输入框。

## 功能归位与文件预算（设计阶段）

- 前端：`apps/admin/src/routes/partners/` 既有账号管理区和 API 适配器。
- 后端：`services/api/src/orgs/`、`services/api/src/auth/`、既有 Redis/OTP/会话失效能力。
- 数据：如凭据来源或一次性授权需要持久字段，仅允许对 `User` 做最小 additive migration，并保持 SQLite/PostgreSQL 对称。
- 文档：设计确认后写入 `docs/superpowers/specs/`，实施完成后同步正式进度文档。
- 不涉及：Kiosk、Partner 新入口、Worker、支付、打印扫描、岗位招聘会、Terminal Agent、生产配置和密钥。
