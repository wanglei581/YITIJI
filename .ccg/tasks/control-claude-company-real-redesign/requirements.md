# 找企业真实数据联动改版要求

## 用户原始目标

用户要求由 Codex 控制 Claude 执行「岗位信息 / 找企业 / 企业详情」改版。该功能不能只做视觉优化，必须成为用户可正常使用的真实功能板块。

## 核心验收口径

- 用户可以从 `/jobs` 进入找企业，筛选企业，查看企业详情，查看来源岗位，进入岗位详情或扫码/外链去来源平台办理。
- 前端所有页面必须与真实后端接口联动，不允许用 mock、演示数据、假统计或前端自造数据填充正式体验。
- 企业、岗位、来源、筛选项、浏览记录、外部跳转记录之间必须有关联，能正常传输、展示、写入和回看。
- 正式页面不得出现「演示」字样；无真实数据时展示真实空态或隐藏对应模块。
- 改动必须限制在岗位信息、企业展示、企业详情、相关后端接口/字典/验证脚本，不得破坏首页、AI助手、我的、招聘会、打印扫描、智慧校园等其它功能。
- 必须继续遵守合规边界：不得新增平台内一键投递、平台内收简历、企业筛选候选人、面试邀约、Offer 管理、候选人推荐、企业发岗直收简历。

## 模块与后端归属

### Kiosk `/jobs`

- 归属前端：`apps/kiosk/src/pages/jobs/JobsPage.tsx`
- 归属后端：`services/api/src/jobs/*`
- 功能：
  - 保留岗位信息主入口。
  - 保留 `/companies` 找企业入口，不新增首页卡片。
  - 岗位列表仍来自真实 `GET /jobs`。
  - 岗位关联企业时，可进入 `/companies/:id`。

### Kiosk `/companies`

- 归属前端：`apps/kiosk/src/pages/companies/CompaniesPage.tsx`
- 归属后端：`services/api/src/companies/*`
- 功能：
  - 改为「来源企业导览」体验，而不是后台式筛选列表。
  - 展示真实企业卡片：logo、企业名、城市/区县、企业类型、行业、来源机构、更新时间、来源岗位数、代表岗位。
  - 支持完整地区、企业类型、行业、招聘类型、来源筛选。
  - 筛选数据必须来自后端统一接口或共享字典，不能只由当前三家企业推导出不完整选项。
  - 按钮文案只能使用「查看企业风采」「查看来源岗位」等合规文案。

### Kiosk `/companies/:id`

- 归属前端：`apps/kiosk/src/pages/companies/CompanyDetailPage.tsx`
- 归属后端：`services/api/src/companies/*` 与 `services/api/src/jobs/*`
- 功能：
  - 展示真实企业资料：logo、封面/宣传片（有真实媒体才展示）、简介、标签、荣誉、地区、来源机构、同步时间。
  - 指标卡只展示后端下发且有真实数据的指标。
  - 来源岗位必须来自该企业真实关联的已发布岗位。
  - 岗位行按钮为「查看岗位」「扫码前往来源平台投递 / 去来源平台投递」。
  - 来源信息区域提供「去来源平台查看」。

### Admin 企业管理

- 归属前端：`apps/admin/src/routes/companies/*`
- 归属后端：`services/api/src/companies/*`
- 功能：
  - 企业资料 CRUD、审核、发布、下架、指标开关、岗位关联必须仍可用。
  - 后台变更后，Kiosk 前台展示必须同步。

### Partner 企业资料

- 归属前端：`apps/partner/src/routes/companies/*`
- 归属后端：`services/api/src/companies/*`
- 功能：
  - 合作机构可导入、编辑、下架本机构企业资料。
  - 编辑后仍需回到待审核/草稿或现有项目约定状态，不得绕过审核发布。

### 浏览与外部跳转记录

- 归属前端：Kiosk companies/jobs 页面调用 activity service。
- 归属后端：activity / member assets 相关接口。
- 功能：
  - 企业详情加载后记录 `company_profile` 浏览。
  - 打开企业来源页记录 `external_open`。
  - 打开岗位来源投递入口记录 `external_apply`。
  - 不记录投递结果或预约结果。

## 筛选字典要求

地区：
- 必须支持完整省 / 市 / 区选择。
- 不能只显示当前已有企业的部分地区。

企业类型建议覆盖：
- 央企、国企、民营企业、外资企业、合资企业、上市公司、高新技术企业、专精特新、事业单位、校企合作单位、公共机构、其他。

行业建议覆盖：
- 互联网/软件、AI/大数据、智能制造、电子信息/半导体、新能源/汽车、金融/保险、医疗/生物医药、教育/培训、建筑/工程、物流/供应链、商贸/零售、文旅/酒店/餐饮、政务/公共服务、农业/食品、其他。

招聘类型建议覆盖：
- 社招、校招、实习、见习、兼职、招聘会参展、政府公共就业岗位、校企合作岗位、技工/蓝领、管培生。

来源类型建议覆盖：
- 人社平台、高校就业网、招聘会主办方、企业官网、公众号/小程序、第三方合规招聘平台、合作机构同步、企业公开资料。

## 禁止事项

- 禁止新增 mock adapter 或前端假数据。
- 禁止用「演示」数据替代真实数据。
- 禁止硬编码企业、岗位、统计数字。
- 禁止新增一键投递、立即投递、平台投递、企业收简历等红线功能或文案。
- 禁止改动 `legacy-miaoda/`。

## 验收要求

- Kiosk `/jobs`、`/companies`、`/companies/:id` 能在真实后端模式下完整走通。
- Admin 企业管理改动能同步到 Kiosk。
- Partner 企业资料改动不越权，并符合审核发布状态机。
- 企业筛选项完整，且筛选结果由后端真实查询返回。
- 企业详情里的岗位与该企业真实关联。
- 浏览记录和外部跳转记录能落库。
- 无「演示」正式文案。
- 相关 lint/typecheck/build/test/verify 脚本通过；如果脚本缺失，应补充聚焦验证脚本。
