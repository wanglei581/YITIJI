# 交底书 P3：基于业务忙碌态豁免与内容哈希缓存的一体机待机宣传屏调度方法

> 建议类型：**发明专利** ｜ 创造性：🟢🟢 中（屏保领域专利较多，务必先检索）
> 技术领域：自助终端人机交互、数字标牌、屏保调度

---

## 1. 背景技术与现有问题

普通屏保仅按"无操作超时"进入，存在三个问题：

- **(P1)** 在打印、扫描、AI 通话等**业务进行中**却无触屏操作的间隙，会错误进入屏保打断业务；
- **(P2)** 公共终端网络不稳定，屏保素材（图/视频）断网时无法加载导致黑屏；
- **(P3)** 素材签名 URL 定期轮换会使常规 URL 级缓存失效。

## 2. 发明目的

在公共自助终端真实网络与多业务并发条件下，待机时正常展示宣传内容，同时避免打断进行中的业务、避免断网黑屏、避免长播放列表内存溢出。

## 3. 技术方案（权利要求骨架）

### 独立权利要求（方法）

1. **业务忙碌态豁免**：终端以**引用计数**维护全局忙碌态，打印/扫描/上传/AI 通话等任一业务开始时计数加一并返回**幂等释放句柄**、结束时减一；空闲计时仅在 `激活 ∧ 非忙碌 ∧ 不在屏保路由` 时启用，从而在无触屏操作但业务进行中时**不进入屏保**。（解决 P1）

2. **内容哈希缓存断网兜底**：以素材内容的 **SHA-256 哈希**（而非可变的签名 URL）作为本地缓存键预取素材；播放时优先返回本地缓存对象、断网仍可播放；签名 URL 轮换或过期**不影响缓存命中**；按当前播放列表的哈希白名单清理过期缓存。（解决 P2、P3）

3. **播放容错**：视频以静音、内联、自动播放方式呈现以规避自动播放拦截；为每个素材设置 `时长 + 容差` 的**兜底计时器**，在解码异常导致结束事件不触发时强制切换下一项；仅预取下一项以控制内存。

4. **诚实降级**：当终端配置启用屏保但播放列表为空、素材全部禁用或被删除时，服务端返回"未启用"，终端**不进入空屏保**；删除播放方案时事务性解绑引用终端。

5. **空闲阈值约束与按终端下发**：服务端对每终端的无操作时长做区间约束（如 30–1800 秒），支持按终端独立下发播放方案与配置。

### 从属权利要求方向

- 空闲检测监听 pointerdown/touchstart/keydown/mousemove/wheel 多源事件，采用 passive 监听避免阻塞触控滚动；任意事件重置计时。
- 素材上传以 MIME 白名单 + 文件头魔数双重校验，防可执行文件改名上传。
- 素材签名采用命名空间前缀（如 `ad:`）与文件签名隔离，防跨用途签名复用；Kiosk 播放用较长 TTL（1h）、管理员预览用较短 TTL（10min）。
- 播放方案保存采用整体替换 items（事务删旧建新）以保证排序一致。

## 4. 关键创新点

| 创新点 | 与惯例对比 |
|--------|-----------|
| **引用计数忙碌态 + 屏保豁免**联动 | 普通屏保仅看无操作超时，会打断进行中业务 |
| **内容哈希作缓存键** | URL 级缓存遇签名轮换即失效；哈希键与签名解耦，断网可播 |
| **视频时长容差兜底计时器** | 依赖 onEnded 事件，解码异常会卡死 |
| 仅预取下一项 | 长列表全量预取会 OOM |
| 无可播素材**诚实降级**为不进屏保 | 普通方案进屏保后黑屏 |

## 5. 有益效果

待机正常展示宣传内容，业务进行中不被打断，断网不黑屏，长播放列表内存可控。
