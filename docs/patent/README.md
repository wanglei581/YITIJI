## 一、申报策略（先读这段）

1. **质量优先，不要硬凑数量。** 代码里能识别的"技术点"有十几个，但很多是同一系统的子模块（离线重试队列、内容哈希缓存、Webhook 验签等）。把它们拆成十几件单独申报，既浪费费用，又容易因"技术方案过小、缺乏创造性"被驳回。**正确做法是按"能独立成件"的粒度归并**，子模块写进对应主案的**从属权利要求**。

2. **软件类发明的关键：把"业务流程"写成"技术方案"。** 提交给代理人时强调**技术手段 + 技术效果**（数据一致性、容错、并发安全、内存可控），不要写成"方便运营""提升体验"这类商业效果，否则会卡"客体问题"。

3. **合规边界是差异化卖点。** 本项目刻意"不做招聘闭环、只做第三方信息入口、结构性阻断个人简历数据"，这些约束在权利要求里可转化为技术特征，反而是与现有招聘/打印 SaaS 的区别点。

4. **保密。** 在与代理机构签订含保密条款的委托合同前，不要公开技术细节、不要对外 demo 讲实现原理。申请日前的公开会丧失新颖性。

## 三、已识别但**不建议单独申报**的点（作为从属权利要求并入主案）

这些是真实实现、有技术细节，但单独成件创造性不足，**写进对应主案的从属权利要求**即可，能增强主案保护范围且不额外花钱：

| 子技术点 | 代码位置 | 归入 |
|----------|----------|------|
| 离线重试队列（指数退避 30s→30min、4xx 立即弃/5xx 重试、本地 SQLite） | `apps/terminal-agent/src/agent/offline-queue.ts:36-110` | P1 |
| 分级 TTL 文件签名 URL（HMAC + 命名空间前缀 `ad:` 防复用，文件 5min/素材 1h/预览 10min） | `services/api/src/files/signing.ts`、`content/content-signing.ts` | P1 + P3 |
| 文件类型四级降级推断（MIME→文件名→URL→默认 PDF） | `apps/terminal-agent/src/agent/task-runner.ts:160-167` | P1 |
| 图片转 PDF 的 A4 自适应缩放 + 临时文件清理 | `apps/terminal-agent/src/printer/image-to-pdf.ts:19-145` | P1 |
| 打印机状态心跳上报 + 前端实时展示 | `services/api/src/terminals/terminals.service.ts`、`PrintPreviewPage.tsx:71` | U1 |
| Webhook 验签 + 时间窗 + nonce 防重放 + 无差别拒绝 | `services/api/src/sync/sync.service.ts:46-80`、`sync/replay-guard.ts` | P2 |
| 数据源凭证 AES-256-GCM 加密落库（scrypt 派生 + GCM 认证标签 + 前端不回显） | `services/api/src/common/crypto/secret-cipher.ts` | P2 |
| 数据源字段映射运行时引擎（resolveKeys 多别名 + 类型归一） | `services/api/src/job-sync/job-sync.service.ts:57-71` | P2 |
| 同步任务队列幂等（BullMQ jobId 唯一 + 无 Redis 内存降级） | `services/api/src/job-sync/job-sync.service.ts:79-104` | P2 |

## 四、费用与周期（行情参考，非报价）

| 项目 | 发明专利 | 实用新型 | 外观设计 |
|------|---------|---------|---------|
| 代理费 | 约 5000–10000 元/件 | 约 2000–4000 元/件 | 约 1000–2000 元/件 |
| 官费（小微/个人可减缴至约 10%） | 申请+实审约 2000–4000 元 | 约 500 元 | 约 500 元 |
| 授权周期 | 约 1.5–3 年（可申请优先审查） | 约 6–12 个月 | 约 4–6 个月 |

> 多数地方政府对企业专利有**资助/奖励**，可向公司所在地科技局/知识产权局咨询，可能覆盖大部分费用。

## 五、对接专利代理机构

- 选**有计算机/软件/通信领域经验**的代理机构（软件类发明撰写难度高于机械类）。
- 交付材料、发明人/申请人主体、已公开情况确认等，见 [对接清单.md](对接清单.md)。
- 把本目录三份/五份交底书 + 各文件中的 `file_path:line_number` 代码位置一并提供，便于代理人撰写实施例与附图。
