# 交底书 U1：多功能一体机打印机多维状态故障诊断装置

> 建议类型：**实用新型**（亦可视方案深度走发明）｜ 创造性：🟢🟢 中-强
> 技术领域：打印设备状态监测、终端硬件交互

---

## 1. 背景技术与现有问题

自助打印终端需在打印前判断打印机是否可用，并在故障时给出**可区分**的原因（缺纸 / 离线 / 卡纸 / 其他错误）。现有方案常仅读取打印机单一状态位，存在两个问题：

- **(P1)** 单维状态位无法区分"离线"与"缺纸"等不同故障，导致提示不准、运维误判；
- **(P2)** 部分打印机驱动（如本项目机型）对某些故障**不设置**标准状态位（例如缺纸不置 PaperOut），直接按标准位判断会漏报。

## 2. 发明目的（实用新型的技术效果）

通过多维度状态采集与决策映射，对打印机故障做细粒度消歧，并针对特定驱动的已知局限做兼容处理，使终端在打印前给出准确的可用性与故障原因。

## 3. 技术方案（权利要求骨架）

### 独立权利要求（装置/方法）

一种打印机多维状态故障诊断装置，包括状态采集单元、决策映射单元与上报单元：

1. **多维采集单元**：经系统管理接口（如 WMI Win32_Printer）**异步并发**采集打印机的多个状态维度，至少包括打印机状态（PrinterStatus）、检测到的错误状态（DetectedErrorState）与离线标志（WorkOffline）。

2. **决策映射单元**：依据多维状态构建**决策树**，将组合状态映射为有限的诊断结论集合（ok / not_found / offline / paper_empty / error / unknown），其中：
   - 离线标志置位 → offline（优先级高于其他错误位）；
   - 特定错误位 → paper_empty / error；
   - 对**已知不设置缺纸位的驱动**，结合其它信号或保守结论进行**兼容处理**，避免漏报/误报。

3. **上报单元**：将诊断结论随心跳上报云端，并供终端打印前预检（preflight）使用，故障时阻止任务下发并提示具体原因。

### 从属权利要求方向

- 采集采用非阻塞子进程（spawn 而非 spawnSync），避免阻塞终端事件循环。
- 对"缺纸"等易抖动状态采用**连续 N 次确认**才判定，抑制驱动状态抖动误报。
- 诊断结论作为打印任务执行前的预检门控；预检不通过则不提交打印驱动。

## 4. 关键创新点

| 创新点 | 与惯例对比 |
|--------|-----------|
| **多维状态决策树**（离线/缺纸/错误消歧） | 惯例读单一状态位，无法区分故障类型 |
| 针对**特定驱动已知局限**的兼容处理 | 惯例直接信任标准状态位，遇非标驱动漏报 |
| 异步并发采集 + 抖动 N 次确认 | 惯例同步阻塞 + 单次判定，易卡顿/误报 |

## 5. 有益效果

终端在打印前给出准确的打印机可用性与故障原因，减少误判与无效打印，提升公共终端运维效率与用户体验；对非标准驱动有更好兼容性。
