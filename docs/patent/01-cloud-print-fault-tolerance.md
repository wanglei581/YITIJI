# 交底书 P1：面向离线多功能一体机的云端打印任务容错调度方法及系统

> 建议类型：**发明专利** ｜ 创造性：🟢🟢🟢 强（首选申报）
> 技术领域：计算机软件、分布式任务调度、自助打印终端

---

## 1. 背景技术与现有问题

现有自助打印 / 云打印方案通常假设打印机本身具备云打印能力，或假设终端与云端网络长期稳定。本项目的实际硬件（奔图 CM2800ADN 系列激光一体机）**不具备云打印能力，且无 WiFi**，只能通过本地 Windows 驱动打印；一体机部署在公共场所，网络不稳定、可能断网或宕机重启。

现有技术存在三个未解决的痛点：

- **(P1) 重复打印**：多终端并发拉取同一任务时易重复打印，浪费纸张并泄露隐私文件。
- **(P2) 崩溃后状态不明**：终端在"已提交给打印驱动、但尚未确认打印结果"的瞬间崩溃/断电重启后，无法判断该任务是否已打印，导致要么重复打印、要么漏打。
- **(P3) 驱动状态不可靠**：部分打印机驱动（含本项目机型）的作业状态长期停留在 "Printing, Retained"，无法从驱动层判定"打印成功"还是"缺纸等待"，乐观判定会造成数据不一致。

## 2. 发明目的

在离线、网络抖动、断电重启、驱动状态不可靠的真实条件下，保证云端打印任务记录与物理打印结果的**最终一致**，避免重复打印与漏打，并实现故障自愈。

## 3. 技术方案（权利要求骨架）

### 独立权利要求（方法）

一种云端打印任务容错调度方法，包括：

1. **租约式原子认领**：云端维护打印任务队列；终端代理（Agent）周期性发起认领请求；云端在**单个数据库事务**内，以 `WHERE 状态='待认领'` 作为更新守卫条件，将任务原子置为"已认领"，并写入终端标识与**认领租约到期时间**；若更新影响行数为 0，则判定已被他人认领并放弃。（解决 P1）

2. **本地中间态持久化**：终端代理在将任务提交给本地打印驱动**之时/之前**，先在终端本地数据库写入"已提交打印机"（spooled）**中间状态**，该状态区别于"完成"与"失败"两个终态。

3. **崩溃恢复幂等判定**：终端代理重启后，对每个待处理任务先查本地数据库：
   - 终态（完成/失败）→ 跳过，不重复打印；
   - spooled 中间态 → **不重复打印**，向云端上报"打印结果未确认"（PRINT_JOB_UNCONFIRMED）告警，提示人工核查。（解决 P2）

4. **驱动作业状态保守判定**：终端代理轮询本地打印队列作业状态，对"缺纸"等抖动状态采用**连续 N 次确认**才判失败；对无法区分完成/等待的驱动保留状态，**不乐观判完成**，而上报"未确认"。（解决 P3）

5. **服务端租约回收**：云端周期扫描，将超过认领租约仍未推进、以及处于"打印中"超过阈值时长的任务，自动重置回"待认领"，实现无人工干预的故障自愈。

### 从属权利要求方向

- 终端代理在状态回传失败时写入**本地离线重试队列**，按**指数退避**（基准 30s、上限 30min、最多 10 次）重试；对 4xx 错误立即放弃、对 5xx/网络错误才重试。
- 认领租约时长（如 5 分钟）**短于**"打印中"超时阈值（如 10 分钟），为终端预留下载、校验、预检的执行窗口；"打印中"超时以真正开始执行的时刻（claimedAt）为参考点。
- 打印文件采用**临时签名 URL**（HMAC + 有效期）下发，终端下载前验签，拒绝非本系统签发的 URL（防 SSRF）；非法 URL 在建任务阶段即被拒绝，不落库。
- 文件类型按 **MIME → 文件名 → URL 路径 → 默认 PDF** 四级降级推断，解决签名 URL 无扩展名的识别问题。
- 下载文件以 **SHA-256** 校验完整性，哈希不匹配直接失败、不重试。
- 心跳上报与离线重试为**相互独立**的两条通道；云端可经心跳响应**动态下发**轮询/认领间隔而无需重启终端。
- 终端以 **PID 文件 + 进程活性检测** 实现单实例互斥（Windows 以 `tasklist` 为唯一真理源，规避 `kill(pid,0)` 的 EPERM 歧义）。
- 终端代理令牌经 **Windows DPAPI（LocalMachine 作用域）** 加密落盘，令牌经 stdin 传入加密进程以防命令行泄露。

## 4. 关键创新点（创造性论述）

| 创新点 | 与惯例对比 |
|--------|-----------|
| 事务内 `WHERE 状态守卫` 原子认领 | 惯例"先查后改再验影响行数"，并发窗口大 |
| **spooled 中间态** + 重启保守上报 | 惯例只有"处理中/完成"，无法区分"已提交未确认" |
| 多层幂等（本地终态 + 服务端终态 + 离线队列幂等） | 惯例仅服务端判定，终端崩溃不可保证 |
| 认领租约 < 打印中超时的**双阈值** | 惯例单一超时，易误回收执行中任务 |
| 承认驱动局限的**保守未确认**判定 | 惯例乐观判完成，造成纸张/记录不一致 |

## 5. 有益效果

公共场所离线一体机在网络抖动、断电重启、驱动状态不可靠的真实条件下，避免重复打印与漏打，保证任务记录与物理打印结果最终一致，并实现故障自愈，降低运维成本。
