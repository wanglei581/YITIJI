# 文件保存期限与 COS 生命周期验收

> 适用范围：用户上传文件、AI 简历成果物、打印扫描文件、Admin 文件管理、生产 COS 私有桶验收。
> 本文是试运营验收口径，不替代正式法务审定文本。

## 一、文件保存期限矩阵

| 文件类型 | 默认策略 | 可由用户调整 | 清理方式 |
| --- | --- | --- | --- |
| 未登录上传文件、匿名打印文件 | 短期保存 | 不可延长 | 到期后按 `expiresAt` 物理删除 |
| 证件照、身份证复印件等高敏文件 | 短期保存 | 不可延长 | 到期后按 `expiresAt` 物理删除 |
| 登录会员原始简历、原始求职材料 | 90 天 | 可延长至 180 天 | 到期后按 `expiresAt` 物理删除 |
| 登录会员优化后简历、AI 派生成果物 | 90 天 | 可延长至 180 天或长期保存 | 到期后按 `expiresAt` 物理删除；长期保存为 `expiresAt = null` |

`long_term` 表示用户确认后的长期保存。长期保存文件在数据库中使用 `expiresAt = null`，不会被 API 的过期清理任务选中；用户主动删除时仍会物理删除对象文件并保留必要审计记录。

## 二、COS 生命周期红线

生产 COS 私有桶必须执行以下规则：

- **禁止配置 Bucket 全局过期规则**。不得对整个 bucket 设置 90 天、180 天或任意天数的全局 Expiration。
- COS 控制台 lifecycle 不得覆盖长期保存对象、会员简历、AI 成果物、机构资料、审计相关对象。
- `users/` 前缀和承载会员资产的前缀不得配置自动过期删除。
- 如需 COS 侧兜底清理，只允许对确认不会承载长期资产的 `tmp/` 临时前缀配置生命周期规则。
- 业务文件删除以数据库 `expiresAt` 和 API cleanup job 为准；COS 生命周期只能做临时对象兜底，不是用户文件主清理机制。

错误示例：

```text
Bucket: yitiji-prod-private-*
Rule: Expiration after 90 days
Scope: Entire bucket
```

该规则会绕过数据库和审计，直接删除 `long_term` 用户资产，禁止上线。

允许示例：

```text
Bucket: yitiji-prod-private-*
Rule: expire-temp-upload-parts
Scope: tmp/
Expiration: 7-14 days
Status: Enabled
```

该规则只能用于未完成上传、临时切片或可再生成缓存，且不得覆盖长期保存对象。

## 三、生产人工验收

本仓库的 verify 脚本只能验证文档和代码未引入桶级 lifecycle 写操作；真实 COS 控制台状态必须人工验收。

上线前必须完成：

- 登录腾讯云 COS 控制台，打开生产私有桶生命周期配置页。
- 确认不存在覆盖整个 bucket 的 Expiration 规则。
- 确认不存在覆盖 `users/`、会员简历、AI 成果物或机构资料前缀的 Expiration 规则。
- 如存在 `tmp/` 前缀规则，确认作用范围仅为临时对象，过期天数和规则名称截图存档。
- 对生命周期配置页进行截图存档，记录验收人、验收时间、bucket 名称、规则名称、作用前缀。

验收结论必须写入试运营记录；没有截图存档时，不能声称 COS 生命周期已验收完成。

## 四、保存条款版本

当前保存条款版本由 `FILE_RETENTION_CONSENT_VERSION` / `CURRENT_RETENTION_CONSENT_VERSION` 控制。用户延长至 180 天或长期保存需确认保存条款。

本分支只同步隐私告知和验收文档，不静默修改保存条款版本。若法务审定后认定长期保存说明属于实质性条款变更，应另起独立分支决策是否 bump 保存条款版本，并要求用户重新确认。
