# A. 腾讯云短信：签名 + 正文模板申请草稿

当前用途**仅一种**：C 端会员手机验证码登录（`SMS_PROVIDER=tencent`，审核通过前服务端
强制禁假发送，见 `services/api/.env.example` §C 端短信）。所以只需申请「验证码类」模板——
这是审核最快、通过率最高的一类。

## A.1 短信签名（签名管理）

- 腾讯云要求签名为真实主体名：可用**公司名 / 网站名 / APP 名 / 小程序名 / 公众号名**之一，
  2–12 字，并提供对应资质证明（营业执照 / 应用商店链接或截图 / 公众号主体等）。
- 候选签名（由运营方按真实可证明的主体二选一定稿）：

  | 候选 | 适用资质 | 备注 |
  |---|---|---|
  | `AI求职打印` | 产品名 / APP 名（需应用或落地页佐证） | 与产品定位一致，建议首选 |
  | `〈运营公司简称〉` | 营业执照主体 | 最稳，用工商名称简称 |

- ⚠️ 不要在签名里出现「招聘 / 求职平台」等可能被理解为网络招聘服务的词（与
  compliance-boundary §二红线一致：本平台不是网络招聘平台）。

## A.2 正文模板（正文模板管理，类型选「验证码」）

腾讯云验证码模板用 `{1}`、`{2}` 作变量占位。短信签名在「签名管理」里单独选择，
**不要写进正文模板**。提交以下任一正文：

```
您的验证码为{1}，{2}分钟内有效。请勿向他人泄露。
```

或（不含有效期变量，后端只传一个参数时用）：

```
您的登录验证码为{1}，请勿泄露。如非本人操作请忽略。
```

> 说明：
> - 实际发送时签名由腾讯云按签名管理拼接；正文模板里不要重复填写签名。
> - `{1}` = 验证码，`{2}` = 有效期分钟数；后端 `TencentSmsSender.sendCode()` 按此顺序传参。
> - 审核通过后拿到 **模板 ID** 和 **SDKAppID**，连同 **签名名** 和**已轮换的 CAM 短信密钥**，
>   按 §A.3 顺序落地（审核通过后仍需填齐生产凭证并完成真号 E2E 验收）。

## A.3 提交后落地步骤（⚠️ 审核通过 ≠ 可上线）

> `services/api/src/member-auth/sms/sms-sender.ts` 已接入腾讯云 SendSms（TC3-HMAC-SHA256 签名）。
> 外部审核只解除腾讯侧阻塞；生产上线仍必须填齐生产凭证，并用真实手机号完成端到端验收。

1. **拿凭证**：审核通过后获得 模板 ID / SDKAppID / 签名名（`TENCENT_SMS_SIGN_NAME`）/
   已轮换的 CAM 短信密钥（`TENCENT_SMS_SECRET_ID` / `TENCENT_SMS_SECRET_KEY`）。
2. **配 `.env`**：写 `SMS_PROVIDER=tencent` + 填齐 5 项 +
   `TENCENT_SMS_REGION=ap-guangzhou`；生产环境 `SMS_PROVIDER` 不得为 `log`
   （服务端已有启动期强制校验，prod 禁 log）。
3. **真号 E2E 验收**：用真实手机号端到端手验——登录页发码 → 收到短信 → 验证码登录成功；
   失败路径（频控 / 无效号）不伪造成功。

# B. 隐私政策 / 用户服务协议：法务审定输入包

目标：让法务**不必读代码**就能拿到「系统真实数据实践」的完整事实，产出可正式发布的定稿，
替换现有试运营文本。

## B.1 现有草稿位置与状态

- 文本在 `apps/kiosk/src/pages/legal/LegalDocPage.tsx`：`TERMS_SECTIONS`（用户服务协议）
  + `PRIVACY_SECTIONS`（隐私政策），更新日期标注 2026-06-11。
- 页脚已声明：**「本文本为试运营版本，正式运营前以运营方法务审定发布的版本为准。」**
- 草稿已尽量与真实数据实践对齐（短 TTL、不留简历库、OCR 不留原文、手机号加密存储），
  但**未经法务审定，不得当正式版**。法务定稿后替换该文件文本即可（纯文本，无外链）。

## B.2 产品定位（合规基线，法务须知）

- 本终端是「AI求职打印服务终端」= AI 简历服务 + 打印扫描 + 第三方岗位/招聘会**信息入口** +
  政策信息查询。**不是网络招聘平台**。
- 红线（见 compliance-boundary §二）：不做平台内投递、不收/转交简历给企业、不做企业候选人
  筛选 / 面试邀约 / Offer 管理 / 候选人推荐。岗位与招聘会只做第三方/官方来源信息展示，
  投递报名走来源平台入口。
- 系统只记录用户的浏览 / 收藏 / 外部跳转 / 打印 / AI 调用行为，**不记录第三方投递/预约结果**。

## B.3 个人信息采集清单（真实）

| 类别 | 字段 | 处理方式 |
|---|---|---|
| 登录标识 | 手机号 | **加密存储**，页面脱敏展示；用于验证码登录 |
| 用户上传内容 | 简历及文件（PDF/图片/扫描件/DOCX） | 短期有效期，到期自动删除；不进简历库 |
| AI 服务结果 | 诊断/优化/生成/匹配/规划结果、模拟面试报告 | 元数据落库 + 留存窗口到期清理；不存简历原文长期 |
| 打印 / 文档 | 打印任务记录、文档记录 | 签名 URL 访问，TTL 夹紧 |
| 行为记录 | 收藏、浏览记录、外部跳转记录、权益记录 | 仅本人可见；只记打开入口行为 |
| **不采集** | 人脸信息、设备内其他文件 | 明确不采集 |

## B.4 第三方数据处理者（数据流向，法务须逐一确认告知/委托处理）

> 均为**境内**服务商，无跨境传输；密钥只在服务端，原文/图像不写日志、不落盘留存。

| 处理者 | 处理的数据 | 用途 | 留存 |
|---|---|---|---|
| 百度智能云 OCR | 简历图片/扫描件的**图像数据** | 文字识别（accurate_basic） | 仅传输识别所需图像，识别原文不写日志，图像不云端留存 |
| 腾讯云 COS（对象存储） | 上传的文件 | 文件存储 | 私有桶 + 签名 URL，TTL ≤30min；敏感文件小时级清理 |
| 大模型服务（如 DeepSeek） | 分析所需**文本** | AI 诊断/优化/生成/匹配/规划 | 仅传输完成分析必需文本 |
| 腾讯云 ASR/TTS（启用时） | 模拟面试**音频** | 语音转写/合成 | 仅内存转发，不落盘、不写日志、不持久化 |
| 腾讯云 TRTC（语音数字人，启用时） | 实时语音会话 | AI 助手语音通话 | 实时通话，凭证仅服务端 |
| 腾讯云短信 | 手机号 | 验证码下发 | 见 A 部分 |

## B.5 留存与删除（真实 TTL）

- 上传简历等敏感文件：短期有效期（通常 1 小时内）到期自动删除。
- AI 结果：留存窗口（默认 24 小时，`AI_RESUME_RESULT_TTL_HOURS`）到期自动清理。
- 用户可在对应业务页面（我的文档 / AI服务记录等）自行删除本人文档与 AI 记录，**文件为物理删除**。
- 公共设备会话信息（含登录态）在退出 / 闲置超时 / 进入待机时即刻清除。
- 管理员因运维访问用户文件时**记录审计日志**。

## B.6 需要法务**补充决策 / 填写**的事项（草稿里目前是占位的）

- [ ] 运营方**正式主体名称**、统一社会信用代码、**ICP/相关备案**、联系方式（现草稿用「现场
      公示的运营方联系方式」占位）。
- [ ] 用户协议/隐私政策的**生效日期、版本号**与变更通知方式。
- [ ] **未成年人 / 学生**个人信息处理是否需要特别条款（求职/校园场景人群相关）。
- [ ] 个人信息**委托处理 / 第三方共享**告知措辞（按 B.4 清单逐一）是否符合个保法要求。
- [ ] 公共终端场景下「**同意机制**」的呈现方式（现为「继续使用即视为同意」，法务确认是否足够，
      是否需要勾选式明示同意）。
- [ ] 数据**安全事件**响应与用户投诉/行权（查询、删除、撤回同意）的对外渠道描述。
- [ ] 补贴/政策类信息为 **info-only**（只做说明+材料清单+官方入口，不承诺到账、不代申请、
      不长期存身份证/银行卡/社保等高敏材料，见 compliance-boundary §8.2）——法务确认表述。
