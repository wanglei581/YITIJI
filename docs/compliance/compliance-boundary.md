## 一、背景说明

当前公司**暂无人力资源服务许可证**。

本系统的定位是：AI求职打印服务终端，提供工具服务（简历、打印、扫描）和第三方信息入口（岗位、招聘会），**不构成自营招聘平台**。

开发前必须阅读本文档，确认不触碰合规红线。

---

## 二、绝对禁止的功能

以下功能**一律不得开发**，无论产品或设计如何要求：

| 禁止功能 | 说明 |
|---------|------|
| 平台内一键投递 | 用户不能在本平台内直接将简历投给企业 |
| 平台内收取求职者简历给企业 | 不能将用户简历转发给企业方 |
| 企业端候选人筛选 | 不能提供企业筛选候选人的界面或接口 |
| 企业端面试邀约 | 不能提供企业发起面试邀约的功能 |
| 企业端 Offer 管理 | 不能提供 Offer 发放和跟踪功能 |
| 候选人推荐给企业 | 不能主动将候选人信息推送给企业 |
| 自营网络招聘闭环 | 整体不构成招聘平台 |
| 企业自主发布岗位并直接收简历 | 企业不能在本系统内发布岗位并直接收简历 |

## 四、数据边界要求

### 4.1 岗位数据必须包含的字段

```typescript
interface JobData {
  source_org_id: string;      // 来源机构 ID
  external_id: string;        // 来源平台岗位 ID
  source_name: string;        // 来源机构名称
  source_url: string;         // 来源平台链接
  sync_time: DateTime;        // 数据同步时间
  review_status: string;      // 审核状态
  publish_status: string;     // 发布状态
}
```

### 4.2 招聘会数据必须包含的字段

```typescript
interface JobFairData {
  source_org_id: string;
  external_id: string;
  source_name: string;
  source_url: string;
  sync_time: DateTime;
  review_status: string;
  publish_status: string;
}
```

### 4.3 岗位详情页必须展示

- 来源机构名称
- 数据同步时间
- 外部 ID（可折叠展示）
- 外部投递链接（去来源平台投递按钮）
- 数据来源说明（例："本数据来自 XXX 平台，本系统不参与招聘流程"）

### 4.4 系统只允许记录的用户行为

| 允许记录 | 禁止记录 |
|---------|---------|
| 浏览岗位 | 企业筛选结果 |
| 收藏岗位 | 简历投递状态 |
| 外部跳转（记录跳转行为，不记录投递结果） | 面试邀约记录 |
| 打印记录 | 候选人评价 |
| AI服务调用记录 | 企业端任何数据 |
| 招聘会浏览记录 | |

## 五、文件安全要求

用户文件（简历、扫描件、证件照、身份证复印件）属于敏感数据，必须遵守：

| 要求 | 说明 |
|------|------|
| 临时签名 URL | 所有文件访问必须使用带时效的签名链接，不得使用永久公开链接 |
| 有效期限制 | 敏感文件签名 URL 有效期建议不超过 30 分钟 |
| 自动清理 | 系统应支持配置文件自动过期清理策略 |
| 管理员访问日志 | 管理员每次访问用户文件必须记录日志 |
| 不长期保存敏感文件 | 身份证、简历等文件不应永久保存，设置清理周期 |
| 删除日志保留 | 文件删除后必须保留删除操作日志（不保留文件本身） |

## 六、打印接口安全要求

奔图 CM2800ADN/CM2820ADN 系列对接时，安全规范如下：

| 要求 | 说明 |
|------|------|
| appKey / appSecret 只存服务端 | 禁止在前端代码、环境变量文件中保存 |
| 服务端签名 | 所有奔图 API 请求由后端服务签名后发出 |
| 禁止前端生成签名 | 前端不得包含任何签名逻辑 |
| 打印文件使用临时链接 | 提交给打印机的文件 URL 必须是有时效的签名链接 |
| 回调验签 | 接收奔图回调时必须验证签名，拒绝无效请求 |
| 回调幂等处理 | 同一打印任务回调可能重复触发，必须做幂等处理 |
| 打印任务状态落库 | 每个打印任务的状态变更必须持久化到数据库 |
| 操作日志 | 所有打印相关的管理员操作必须记录日志 |

## 八、营销 / 权益 / 套餐 / 补贴 / 支付合规边界

> 新增于 2026-06-07，关联规划：[next-tasks.md](../progress/next-tasks.md) §个人资产中心 + 权益活动 + 服务套餐 产品规划（Phase C-2 ~ C-6）、[feature-scope.md](./feature-scope.md) §六。
> 本节为「个人资产中心 / 权益活动中心 / 服务套餐 / 支付核销 / 政策补贴」补充合规边界，**不推翻 §二~§七，仅做维度补充**。

### 8.1 招聘闭环红线（重申，永不突破）

权益活动、服务套餐、企业赞助等任何商业化能力，**都不得**借机引入 §二 的 8 类招聘闭环功能（一键投递 / 收简历给企业 / 企业筛选候选人 / 面试邀约 / Offer 管理 / 候选人推荐 / 自营招聘闭环 / 企业发岗直收简历）。

### 8.2 补贴类功能边界

| 只能做 | 不能做 |
|--------|--------|
| 政策说明 | 承诺补贴到账 / 必得 |
| 材料清单 | 未授权代用户申请 |
| 官方申请入口（外链） | 长期保存身份证 / 银行卡 / 社保等高敏材料 |
| 申请材料打印 | 把「补贴券核销」包装成「政府补贴金发放」 |
| 申请指引 | — |

- 「我的权益」中的「补贴资格提示」只是**信息指引**，不得展示金额到账、发放状态等承诺性文案。
- 补贴申请材料如含身份证 / 银行卡 / 社保等，沿用 §五文件安全：临时签名 URL + TTL 清理，不长期留存。

### 8.3 企业赞助边界

- 企业可赞助权益（如赞助打印券 / 材料包），**仅体现为品牌露出 + 权益发放**。
- 企业**不得**通过赞助活动获取用户简历、候选人数据或任何可回流企业的求职者材料。
- 赞助不得形成「求职者 → 企业」的简历投递 / 候选人池。

### 8.4 服务套餐边界

- 套餐**只卖工具服务和打印服务**（AI 简历优化、面试训练、证件照、打印额度等），**不卖「录用结果」**。
- 套餐文案**禁止**「保面试 / 保录用 / 补贴必到账 / 名企内推 / 代投简历」等承诺。
- AI 模拟面试 / 面试训练结果**只给求职者本人**，不推送企业（沿用 feature-scope §六）。

### 8.5 支付 / 核销 / 退款边界

- 支付凭证 / 密钥（微信、支付宝、校园卡等）**只存服务端**，前端不得保存，沿用 §六打印接口安全的密钥原则。
- 券 / 免费次数 / 补贴券**核销必须幂等 + 落库审计**；**免费单也落库**。
- 退款**幂等**；**不得把支付异常伪装成打印任务状态**。
- 「补贴券核销」限于平台内服务 / 打印券，**不等于、也不承诺政府补贴金到账**。

### 8.6 营销 / 权益功能开发检查清单

- [ ] 是否新增了任何招聘闭环功能（投递 / 收简历 / 候选人 / 面试 / Offer）？
- [ ] 补贴类是否只做指引（说明 / 清单 / 官方入口 / 打印 / 指引），没有承诺到账或代申请？
- [ ] 企业赞助是否杜绝了简历 / 候选人数据回流企业？
- [ ] 套餐文案是否无「保录用 / 保面试 / 补贴必到账」承诺？
- [ ] 支付密钥是否只在服务端？券核销 / 退款是否幂等且落库审计？
- [ ] 高敏材料（身份证 / 银行卡 / 社保）是否只用临时签名 URL + TTL，未长期保存？

## 九、校园 / 学生数据隐私边界（智慧校园模块）

> 新增于 2026-06-08。关联：[feature-scope.md](../product/feature-scope.md) §6.8 智慧校园、[CLAUDE.md](../../CLAUDE.md)。
> 本章为「智慧校园」模块（按终端开关，含迎新系统 / 校园大数据 / 行李帮运 / 校园全景等）补充合规边界，**不推翻 §二~§八，仅做维度补充**。
> **本章为硬性前置：在本章合入 main 之前，「校园大数据」子功能不得开始开发（不写一行展示代码）。** 其余子功能（迎新只读 / 行李外链 / 校园全景）须同样遵守本章。

### 9.1 背景与新增风险面

智慧校园引入了一个项目此前从未处理过的合规面：**在一台公共、半开放、无登录的一体机上，处理并展示「非本平台用户」（学生群体）的个人信息**。这与现有产品「只处理求职者本人、最小化、加密、按机构隔离」的姿态完全不同。在公司**无人力资源服务许可证、产品边界刻意收紧**的整体姿态下，此风险面必须从严控制。涉及《个人信息保护法》（PIPL）《数据安全法》。

### 9.2 校园大数据：学生个人信息红线（最严）

「校园大数据」（新生男女比例 / 年龄 / 专业热度 / 生源地等）本质是对学生个人信息的聚合处理与公开展示。

| 只能做 | 绝对不能做 |
|--------|-----------|
| 展示经学校**书面授权**的、**已聚合脱敏**的统计数字 | 接收 / 存储任何**单条学生记录**（姓名 / 学号 / 身份证 / 手机号 / 照片 / 班级 / 宿舍名单） |
| 系统只接收学校**已聚合好**的汇总数字（如「男 3370 / 女 2870」） | 下钻 / 反查到个人；按个人维度展示或筛选 |
| 院校级 / 学院级聚合视图 | 把校园大数据页面 / 明细接入**打印或扫码导出**链路（本机是打印终端，看得到就打得出） |
| 对小样本分组强制合并 / 隐藏 | 在**免鉴权** Kiosk 端点返回任何学生统计 value |

**强制 k-匿名阈值**：任一统计单元格（含性别 × 专业 × 年龄等多维交叉）计数低于阈值时，一律合并区间或显示「<阈值」。建议 **k ≥ 10，最低不得 < 5**。阈值需学校与法务共同确认。

**数据来源合法性三件套（缺一不可，否则不做）**：
1. 学校**书面数据授权书**：明确授权我方展示其聚合统计数据；
2. **数据处理协议（DPA）**：明确我方为**受托处理者**，按学校指示处理、不得超范围使用、不得对外提供、合作终止即删除；
3. 数据范围、用途、保存期限、删除义务书面约定。

> 数据控制者为学校；我方为受托处理者。**我方不得自行采集学生数据**——若需我方在终端采集学生数据，直接触红线，不做。

### 9.3 免鉴权终端端点边界

智慧校园开关沿用 screensaver 的免鉴权 Kiosk 拉取范式（`GET /terminals/:terminalId/smart-campus`）。该端点**返回体必须白名单化**，只允许包含：`{ enabled, modules 开关位, 已审核的外链元数据 }`。

- **绝不允许**任何学生统计 value 进入此免鉴权端点（任何人猜到 terminalId 即可抓取）。
- 校园大数据的统计数字若上线，必须走**需鉴权端点**或**仅在后台展示给运营**，并写自动化测试断言免鉴权端点返回体不含任何统计 value 字段。

### 9.4 迎新系统边界

- 一期**仅限只读信息展示 + 官方外链**：报到流程指引、校园导览、办事窗口、学校官方报到系统外链 / 公众号二维码。
- **不得在本终端采集任何学生身份 / 报到信息**（填表、上传身份证、录入入学信息一律不做），与「Kiosk 不留存 >1h 敏感数据」铁律一致。
- 任何「在终端采集报到信息」的需求须单独立项、单独走 PIPL 同意 / 最小化 / 加密 / 留存评审，默认不做。

### 9.5 行李帮运 / 第三方服务边界

- **仅做第三方信息入口 + 外链 / 扫码跳转**，与岗位「去来源平台」同构。
- **不在本终端做下单 / 收款 / 支付闭环**（守 §8.5），**不承担承运 / 居间责任**。
- 页面必须明示服务方与免责声明（例：「本服务由 XX 提供，本终端仅提供信息入口，不承担承运责任」）。
- 文案禁止「一键下单 / 我们帮你运 / 包送达」等承诺式表述。

### 9.6 校园全景边界

- 一体机**无 VR 头显**，正式命名为「校园全景 / 云游校园」，**不得使用「VR」字样误导用户**；实现为 360° 全景图 / 视频拖拽浏览。
- 全景素材中如出现可识别师生人脸，须获授权或打码；素材版权归属清晰；不内嵌任何采集行为。

### 9.7 终端归属与多租户隔离

- 为实现「按学校 / 按机器控制」，`Terminal` 需新增 `orgId`（机构归属）。
- **学校用户不得使用管理员（admin）子账号**：admin 后台为平台全局视角，菜单隐藏 ≠ 数据隔离。学校自助管理（如有）须走**合作机构（partner）后台 + `Organization.type='school_employment_center'`**，service 层强制 `where terminal.orgId = user.orgId`（照抄 jobs.service 的 `ForbiddenException` 范式），跨校访问一律 403 并审计。
- RolesGuard 只校验角色、不做机构隔离；机构归属校验必须在 service 层手写。

### 9.8 审计

- 智慧校园开关变更、终端机构归属变更、校园大数据数据源接入 / 更新等所有写操作，必须落 `AuditLog`（含 operator、role、terminalId、orgId、旧新值、时间），便于事后追溯数据合法性。

### 9.9 文案

- 智慧校园所有子功能**不得出现招聘 / 投递 / 收简历语义**；涉及岗位 / 招聘会仍沿用 §三「查看岗位 / 去来源平台投递 / 查看招聘会 / 去来源平台预约」，CTA 禁用「一键 / 立即 / 我们帮你 / 包送达」等承诺式文案。

### 9.10 智慧校园开发检查清单

- [ ] 「校园大数据」开发前，本章是否已合入 main？
- [ ] 是否已取得学校书面授权书 + 数据处理协议（DPA），且只接收学校已聚合脱敏的统计数字（无任何学生明细）？
- [ ] 聚合是否强制 k-匿名阈值（小样本合并 / 隐藏），是否禁止下钻到个人？
- [ ] 免鉴权终端端点返回体是否白名单化、绝不含学生统计 value（是否有自动化测试断言）？
- [ ] 校园大数据是否未接入任何打印 / 扫码导出链路？
- [ ] 迎新系统是否仅展示 + 外链、未在终端采集任何个人信息？
- [ ] 行李帮运是否仅外链入口、无平台内下单 / 收款 / 承运承诺？
- [ ] 校园全景是否未使用「VR」误导文案、素材授权是否清晰？
- [ ] 终端是否按 orgId 隔离、学校用户是否走 partner 后台而非 admin 子账号、跨校访问是否 403 并审计？
- [ ] 所有写操作是否落 AuditLog？子功能文案是否无招聘闭环语义？
