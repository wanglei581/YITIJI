# 这份方案 PDF 改怎么改

结论先放在最前面：合规写得干净，但商业这块基本是空白，更像产品介绍，不像合作方案。拿着它去人社局谈，撑不到第二轮。

## 三件立刻要处理的

### 一、品牌名错了

v1 全文叫"AI 求职打印服务终端"。但你 BP、参赛材料、内部命名研究里全叫"**职易达 AI 求职服务终端**"。

这不是小事。B2G 客户最敏感的就是"你是不是招聘平台"。"AI 求职打印服务终端"主词落在"打印服务"上，第一眼会被读成"打印店"。你 BP §1.3 自己写过"避免使用'招聘平台''人才中介'等可能触发资质误解的表述"——同一个思路，"打印店"也得避开。建议立刻把主标题换掉。

### 二、商业模型是空白

这是 v1 最大的窟窿。没收入类型、没定价、没单台 TCO、没 ROI、没试点 KPI。

但你 BP §9 已经写好了：

- 三档规模测算（保守 7.5 万 / 基准 37.7 万 / 积极 175.5 万）
- 单台年度基础成本 1.1–2.75 万
- 6 类收入 + 6 类明确不做的收入

v1 一字没引。B2G 客户问"你们怎么挣钱""一年花多少""买一台多少钱"——答不出就是废纸。

### 三、已经做完的事 v1 几乎没说

模拟面试、职业规划、OCR（百度智能云）、数字人"小青"、Partner 后台完整、Admin 后台完整、Webhook 接入、浏览跳转记录、企业展示 CompanyProfile——这一串你 BP §6.5 和 current-progress.md 里写得明明白白，v1 一字没提。

客户看完会觉得：哦，又一个 PPT 项目。但其实你产品代码已经跑通。依据是 `production-deployment-and-windows-host-checklist.md §G` 自己写的"代码与文档层面完成度已就绪"。这是你的真东西，必须让它上桌。

## 按视角拆开看

### 行业这块

v1 的市场定位选得聪明——"线下点位服务"，不碰"全国平台流量"，BP §5.4 讲得很清楚，v1 继承得不错。客户分类也细：人社、高校、招聘会主办方、园区、集成商五类，每类都有痛点和切入话术，B2G/B2B 沟通里少见的认真。

但缺三样对决策人关键的：

**市场体量**。教育部 2026 届毕业生 1270 万（央视网 2025-11-20 公开报道），青岛每年招聘活动 1000 场以上、发布就业岗位 40 万以上（青岛发布青年人才若干措施新闻发布会）。这两个数字写进执行摘要，决策人立刻判断市场规模。你现在写"政策与市场机会"，只有政策没市场。

**竞品分析**。你 BP §11 那张 5 类对比表（打印店 / 在线简历工具 / 招聘平台 / 信息屏 / 自助政务终端）写得很好，v1 没用。B2G 客户会问"你们和 XX 有什么不同"，没准备答。

**试点案例**。最致命的一项。B2G 决策必须有"已签约""已服务""已跑通"。如果没有，就明确写"代码就绪，生产部署 / 真机 / 法务 / 试运营仍在执行"——别藏。

### 产品这块

四模块架构（Kiosk + Admin + Partner + Windows Terminal Agent）写得清晰，和 AGENTS.md 完全对得上，决策人一看就懂。场景-能力对照表也写得不错（公共就业大厅 / 高校 / 招聘会 / 园区社区 4 个场景）。

缺三块：

**AI 能力矩阵**。你 BP §7.2 有 8 项 AI 能力（OCR / 简历诊断 / 简历生成 / 简历优化 / 岗位匹配参考 / 模拟面试 / 职业规划 / 政策问答）加上数字人小青。v1 只写"AI 简历服务"——这是一个名次，不是一个能力清单。客户问"你们 AI 能做什么"，你要能直接答："OCR、简历诊断、生成、优化、岗位匹配参考、模拟面试、职业规划、政策问答，加上数字人顾问"。

**硬件能力详细说明**。奔图 CM2800/CM2820 v1 只在 §三提了一下名字。但 `pantum-cm2820adn.md` 写得很清楚：已确认 A4 黑白 / 彩色 / 双面 / ADF / SMB 扫描，不支持 A3 / 无线 WiFi / 云端远程扫描，Windows 驱动识别名是 `Pantum CM2800ADN Series`。这些不写出来，集成商会以为你啥都能干，到现场翻车。

**视觉锚点**。`visual-design-spec.md` 你已经定义了"真实、克制、专业、可信赖"的视觉语言（科技蓝 `#2563eb`、Inter + PingFang SC、不开暗黑）。`apps/kiosk/public/assets/` 下已经有 2 张本地素材（ai-advisor.png、kiosk-home-hero-job-fair.png）——但 v1 全文一张图没有。BP §6.5 列了 8 张待补图：2 张本地有、6 张待拍/待截。

### 合规这块

v1 做得最好的就是合规。允许/禁止对照表干净，按钮文案（"去来源平台投递""扫码投递""去来源平台预约""扫码预约"）完全符合 `compliance-boundary.md §三` 的规范。"资料来源"列的 3 条公开政策 URL（gov.cn / moe.gov.cn）我都抽查过，全部可访问。

但有四个问题：

品牌名那个第一点说过了。

**"明确不做"的收入没写**。BP §8.3 列了 6 类不做（按简历数向企业收费 / 候选人筛选 / 站内发岗 / 代投 / 按录用收费 / 向企业回流简历），v1 没体现。合规要双向防御——光说"我们不做招聘平台"不够，要说"我们不做这些事"。B2G 决策人最怕被合规问题拖下水，他们想看的就是这个。

**数据安全机制太粗**。v1 §七只写了"管理员只查看必要元数据，敏感文件设置清理和访问审计"——但 `compliance-boundary.md §五 §六` 写得细得多：临时签名 URL + 30 分钟 TTL + 访问日志 + 管理员访问留痕 + 打印回调验签 + 任务状态落库 + 退款幂等。这些写出来，对 B2G 合规审查是过命的。

**当前生产状态没说**。依据 `production-deployment-and-windows-host-checklist.md §G`，你当前真实状态是：代码与文档层面就绪，**生产服务器 / Windows 真机 / 法务审定 / 试运营 4 件事仍在执行**。这事必须讲清楚，否则客户期待和现实差距会变成后续麻烦。

### 对外材料这块

副标题"把就业服务放到求职者面前"写得好，对决策人有冲击力。章节标题统一、合规单独成章、建议合作对象放封面——这些都做对了。

但决策人看 PDF 是扫的，v1 缺五个硬通货：

**信任锚点**。没有团队、没有公司、没有联系方式、没有媒体引用、没有监管背书。v1 像"项目介绍书"，不像"合作方案书"。

**数字证据**。v1 没有任何具体数字。毕业生 1270 万、青岛 40 万岗位、单台成本 1.1–2.75 万——这些 BP 里都有，v1 一字没用。没有数字决策人不会停下来细读。

**成功案例**。同行业务案例 / 已签合同 / 已服务客户——v1 空白。

**下一步行动 CTA**。v1 §十只有"资料来源"，没有"下一步怎么合作""找谁对接"。一份好的合作方案 PDF 最后一页应该是 CTA + 联系人，不是文献清单。

**章节字数过于平均**。7 章每章 600–900 字，读者找不到"重点"。建议把执行摘要扩到 500–700 字（含核心数字），把商业模式单独拉出来当一章。

## 改写怎么改

### 命名

主标题换成"**职易达 AI 求职服务终端**"。副标题保留"把就业服务放到求职者面前"，或调整为"AI 智能体驱动的线下自助就业服务终端"。封面"建议合作对象"四个字保留，对决策人有用。

### 章节结构

v1 是 7 章正文 + 1 章资料来源。v2 建议改成 **8 章正文 + 1 章附录**：

- §一 执行摘要，500–700 字。**必含**核心数字 + 当前生产状态。
- §二 市场机会与政策环境，600–800 字。加毕业生数据 + 青岛政策 + 4 场景痛点表。
- §三 产品与 AI 能力，1200–1500 字。**最大块**。加 AI 能力矩阵 + 已实现清单 + 视觉锚点图。
- §四 客户价值与合作模式，600–800 字。
- §五（**新增**）商业模式与定价，600–800 字。三档测算 + 单台 TCO + 6 类收入 + 6 类不做收入。
- §六 落地试点路径，500–700 字。**加 KPI 阈值**（日均使用人数、AI 服务次数、打印成功率、故障响应时间等）。
- §七 合规边界与数据安全，700–900 字。加数据安全机制 + 明确不做的收入 + 合规审查流程。
- §八（**新增**）团队、下一步行动与联系方式，300–500 字。
- 附录 资料来源：v1 §十 + BP 附录 B。

总字数 v1 是 4199 字 → v2 建议 5500–7400 字（扩 30–75%）。总页数 v1 是 7 页 → v2 建议 10–12 页。
