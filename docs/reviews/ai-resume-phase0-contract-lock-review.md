# AI 简历优化商用闭环 Phase 0 审查记录

> 日期：2026-06-29
> 分支：`codex/ai-resume-phase0-contract-lock`
> 范围：产品 / 合规 / 进度文档口径锁定，不改运行时代码。

## 目标

本阶段只锁定 AI 简历优化商用闭环的产品与合规合同：

- 诊断后可进入优化，但必须由用户主动触发。
- 岗位参考首期只支持系统已审核已发布岗位，或用户手动粘贴 JD / 岗位要求。
- 任意外部 URL 首期不自动抓取、不入库为已审核岗位、不触发投递或预约记录。
- 优化前后对比、在线编辑、AI 一键调整、模板渲染、图片 / PDF / 打印一致性、语音生成和多格式导出都按分阶段能力落到后续任务。
- AI 简历优化只能售卖工具服务、成果导出、保存和打印，不承诺录用、面试、通过率、补贴到账或原文件 100% 格式保真。

## 子代理审查摘要

### 产品口径审查

子代理建议把 Phase 0 主落点放在 `docs/product/feature-scope.md`，同步 `docs/progress/current-progress.md` 与 `docs/progress/next-tasks.md`，不修改 `.ccg/spec/guides/index.md`。关键风险是误把任意 URL 自动抓取、语音生成、多格式导出、模板素材库或收费支付写成 Phase 0 已完成。

处理结果：

- 已在 `feature-scope.md` 增加 AI 简历优化商用闭环口径。
- 已明确不新增重复入口，不回到 Profile 聚合资产中心。
- 已把语音生成、多格式导出、模板自动填充写为后续分阶段能力。
- 已在 `current-progress.md` 明确本分支只改文档，不代表运行时代码或真机验收完成。

### 合规 / 收费边界审查

子代理建议在 `docs/compliance/compliance-boundary.md` 第八章新增 AI 简历优化商用闭环条款，并覆盖工具服务收费、招聘闭环红线、任意外部 URL、文件安全、重复导出扣费、长期保存同意和支付 / 退款边界。

处理结果：

- 已新增 `8.7 AI 简历优化 / 导出 / 收费边界`。
- 已明确禁止保面试、保录用、录用概率、通过率、名企内推、代投简历、100% 格式保真。
- 已明确成果物不得回流企业或合作机构。
- 已明确已付费且未修改成果在有效保存期内不重复扣同一导出权益。
- 已明确长期保存必须按文件保存期限策略和用户确认执行。
- 已明确支付 / 退款 / 对账进入订单支付域，不挂靠 `PrintTask`。

## Claude + Antigravity 双模型审查

### Claude 审查

结论：无 Critical，Approve。Claude 确认本分支没有把运行时代码、多格式导出、语音生成、模板自动填充、支付、退款、套餐或真机打印误写成已完成；招聘合规红线、外部 URL 边界、收费幂等和文件安全边界总体完整。

需补强项：

- 「已付费且未修改的同一成果」必须定义服务端判定方式，避免只靠前端状态判断。
- DOCX / TXT / Markdown 作为可编辑导出，也必须声明不是上传原件、不保证与 PDF / PNG 像素一致。
- `PNG` 描述避免使用容易被误解成保真承诺的「高还原」。
- 文件保存期限矩阵应给出可追溯链接。

处理结果：已在 `feature-scope.md` 和 `compliance-boundary.md` 补成果版本 ID / 内容哈希 / `FileObject` 幂等键、DOCX / TXT / Markdown 非原件说明、保存期限矩阵链接和长期保存状态提示，并移除「高还原」措辞。

### Antigravity 审查

结论：无 Critical，Approve。Antigravity 确认产品闭环路径清楚，入口没有扩张，外部 URL 不自动抓取，支付与退款不挂靠 `PrintTask` 的方向正确。

需补强项：

- 匿名用户优化 / 编辑后触发登录或注册时，需要保留草稿、`taskId` 和操作历史，避免草稿丢失。
- DOCX 导出容易被误解为可编辑 Word 原文，需要明确其只是内容结构导出，不承诺样式一致。
- 「未修改」计费边界需要以服务端版本或内容标识判定。

处理结果：已在 `feature-scope.md` 增加匿名草稿登录承接要求，并与 Claude 建议合并为服务端版本标识和导出格式免责口径。
