# Kiosk 8177 / 5299 Fusion W2 Print and Scan Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将打印扫描域的 19 条生产路由换装为冻结的 fusion-youth 呈现，同时原样保留上传、材料检查、价格、支付、打印、扫描、轮询、恢复和硬件契约。

**Architecture:** W2 只改 `apps/kiosk` 打印扫描页面及其本地呈现/测试文件。先用 TypeScript AST 和文件摘要锁定路由、只读逻辑文件与 19 路由 ownership，再把 886 行材料检查页的 JSX 抽成无服务调用的呈现组件，并把 2114 行 CSS 拆为五个页面族；页面容器继续拥有 hook、service、轮询、`sessionStorage`、路由 state 和导航。浏览器验收只对 production build 使用 fail-closed `ApiRouter` 注入真实协议形状，不增加 demo route、生产开关或静态假页面。

**Tech Stack:** React 18, React Router 7, TypeScript 5.7 compiler API, Vite 6, Tailwind CSS 4, frozen `@ai-job-print/ui` W1 primitives, Node 22, Playwright Chromium.
