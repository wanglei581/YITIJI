# Kiosk 8177 / 5299 Fusion W1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Freeze a Kiosk-only 5299-led presentation API, apply it to the shared shell and the real home page without changing business behavior, and prove the new page, state, action, and modal primitives at Kiosk and mobile viewports.

**Architecture:** Keep the existing `VisualTheme = 'legacy' | 'service-desk'` compatibility contract intact and add an orthogonal Kiosk presentation attribute. New components are presentation/accessibility-only and consume no hooks or services. The production home remains the representative real page; an isolated Playwright-only Vite fixture renders primitives that do not yet have a W1-owned production consumer, avoiding test routes and demo flags.

**Tech Stack:** React 18, TypeScript 5.7, Vite 6, Tailwind CSS 4, `@ai-job-print/ui`, Node 22 verifier scripts, Playwright 1.54.2 Chromium.

**Frozen visual source:** Kiosk stage, spacing, target sizes, radii, and palette values are derived from `docs/design/kiosk-proto-2026-07/shared.css`; home-specific composition is derived from `docs/design/kiosk-proto-2026-07/01-home.html`.

## 1. Frozen public API

Implement these exact exported types. W2–W5 may consume them but may not change their signatures without a new reviewed foundation task.

```ts
export type KioskPresentation = 'legacy' | 'fusion-youth'
export type KioskViewport = 'kiosk' | 'mobile'

export interface KioskPageFrameProps {
  children: ReactNode
  header?: ReactNode
  footer?: ReactNode
  className?: string
}

export interface KioskPageHeaderProps {
  title: string
  description?: string
  onBack?: () => void
  backLabel?: string
  leading?: ReactNode
  aside?: ReactNode
  headingId?: string
  className?: string
}

export interface KioskActionBarProps {
  children: ReactNode
  leading?: ReactNode
  className?: string
}

export type KioskStateTone =
  | 'loading'
  | 'empty'
  | 'error'
  | 'offline'
  | 'success'
  | 'permission'

export interface KioskStatePanelProps {
  tone: KioskStateTone
  title: string
  description?: string
  icon?: ReactNode
  meta?: ReactNode
  actions?: ReactNode
  compact?: boolean
  className?: string
}

export interface KioskModalProps {
  open: boolean
  onClose: () => void
  title: string
  description?: string
  children?: ReactNode
  actions?: ReactNode
  closeLabel?: string
  closeOnBackdrop?: boolean
  closeOnEscape?: boolean
  className?: string
}
```

`KioskPageFrame` renders a semantic `<section>`, not another `<main>`, because `KioskLayout` already owns the page `<main>`. `KioskModal` derives stable `aria-labelledby`/`aria-describedby` IDs with `useId`, focuses the dialog on open, restores the prior focused element on close, locks body scroll, and defaults backdrop/Escape dismissal to `true`. It does not own countdowns, logout, payment, retry, or other business behavior.
