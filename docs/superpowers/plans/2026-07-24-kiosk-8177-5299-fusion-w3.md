> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the 19 existing resume, AI advisor, career-planning, job-fit, and mock-interview route patterns to the frozen W1 fusion presentation API without changing any AI, OCR, TRTC, resume, interview, authentication, file, print, or session behavior.

**Architecture:** Keep every route component as the owner of its existing business state and service calls. First freeze route, service, hook, session, and shared-upload contracts with a fail-closed W3 verifier; then mechanically split the legacy resume CSS into five non-duplicated scoped files behind compatibility entrypoints. Migrate diagnosis, authoring, library, job-fit/career, assistant, and interview presentation in small commits, and finish with production-build Playwright fixtures that exercise real page state machines through intercepted HTTP contracts rather than test-only production routes or storage injection.

**Tech Stack:** React 18, React Router 7, TypeScript 5.7, Vite 6, Tailwind CSS 4, frozen `@ai-job-print/ui` Kiosk primitives, Node 22, Playwright Chromium, existing `verify:*.mjs` contracts.

## 5. Required presentation contract

Every concrete W3 page branch must expose a stable scoped root:

```tsx
<KioskPageFrame
  className="fusion-w3 fusion-w3--resume <existing-root-classes>"
  data-kiosk-domain="resume"
  data-kiosk-screen="resume-source"
>
  {/* existing business-owned content */}
</KioskPageFrame>
```

Because the frozen W1 `KioskPageFrame` interface does not currently declare arbitrary data attributes, do **not** add the two `data-*` props to it. Put them on a single inner route landmark instead:

```tsx
<KioskPageFrame className="fusion-w3 fusion-w3--resume <existing-root-classes>">
  <main data-kiosk-domain="resume" data-kiosk-screen="resume-source">
    {/* existing business-owned content */}
  </main>
</KioskPageFrame>
```

For an existing root `<main>`, keep that element and wrap it with `KioskPageFrame`; do not create nested `<main>` landmarks. For an existing `<div>` root, change the route content landmark to `<main>` only when it does not create nested-main semantics. `AssistantCallPanel` is a dialog/sub-state and uses `data-kiosk-screen="assistant-call"` on its existing dialog content, not a new page frame or route.

Use the frozen primitives as follows:

- `KioskPageHeader`: route title, description, back action, and status/summary aside.
- `KioskActionBar`: persistent primary/secondary route actions only; keep button handlers unchanged.
- `KioskStatePanel`: loading, empty, error, permission, and honest missing-context branches.
- `KioskModal`: simple confirmations only when it can preserve the existing callback sequence. Complex voice/recording overlays keep their existing semantic dialog and are visually scoped instead of being forced into a lossy generic modal.

Each state must preserve its existing text source and service error. Do not replace a real server error with generic static copy merely to match a prototype.
