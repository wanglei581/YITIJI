# 智慧校园与首页百宝箱设计方案

> 日期：2026-06-29
> 范围：Kiosk 首页智慧校园展示、设备授权、百宝箱占位框设计
> 状态：设计确认稿；首页百宝箱占位已最小落地，智慧校园完整策略守卫仍待后续实现和真机复验。

## 目标

在 Kiosk 首页按现有视觉语言增加一个可长期承载扩展能力的「百宝箱」区域，同时把「智慧校园」调整为设备授权后才出现的条件模块。

最终效果：

- 智慧校园关闭时：首页不显示智慧校园，不留下空白。
- 百宝箱当前不放任何业务功能卡，先显示可见占位框；后续由产品确认后再逐项接入。
- 智慧校园开启时：首页显示智慧校园专区，样式参考用户草图：浅蓝标题栏、学校端已开启状态、进入专区按钮、下方展示已启用校园子模块。
- `/smart-campus/*` 子页面也受设备策略守卫，关闭后不能继续浏览业务内容。
- 27 寸竖屏触控显示器下布局稳定，不挤压、不溢出、不出现大块无意义空白。

## 当前明确口径

百宝箱不是现有服务的搬家容器。本期不把以下内容放入百宝箱：

- AI 面试训练。
- 政策服务。
- 打印扫描、招聘会、岗位信息或 AI 简历服务。
- 任何后续未确认的新功能。

本期只建立百宝箱的首页占位与承载规则：

- 没有内容时不渲染功能卡。
- 需要渲染一个可见的百宝箱占位框，保持首页结构稳定。
- 占位框不提供业务操作，不跳转，不伪装成可用功能。
- 占位文案只表达「百宝箱」与「内容待配置」这类状态，不写具体未确认功能。
- 后续新增内容必须逐项确认真实入口、数据来源、合规边界和验证方式。

## 信息架构

首页保持现有主业务结构：

1. 顶部欢迎与账号状态。
2. 核心服务分组：AI 简历服务、岗位信息、招聘会、打印扫描等现有主业务。
3. 条件增强区：智慧校园。
4. 预留扩展区：百宝箱。
5. 合规来源说明。

智慧校园与百宝箱的关系：

- 智慧校园不是百宝箱默认内容。
- 智慧校园是设备授权后的条件增强模块。
- 百宝箱当前为空，但显示占位框；只有未来产品明确配置内容后才显示功能卡。
- 如果未来百宝箱有普通内容，智慧校园仍按设备授权独立显隐。

## 智慧校园显示规则

Kiosk 只相信后端返回的本机有效策略，不在前端硬编码显示。

显示智慧校园必须同时满足：

- Kiosk 配置了正确的 `terminalId`。
- 终端已注册并能解析到 `terminalCode`。
- 终端绑定到有效机构。
- 机构类型允许智慧校园能力。
- 终端智慧校园总开关为开启。
- 至少一个可渲染子模块开启。
- `bigdata` 仍保持冻结，不参与可渲染模块判断。

任一条件不满足：

- 首页不显示智慧校园。
- 不预留空白。
- 直达 `/smart-campus/*` 时显示「本机暂未开启智慧校园服务」并引导返回首页。

## 设备授权模型

沿用现有 Terminal Agent 注册链路：

- Terminal Agent 采集主机名和非内部网卡 MAC。
- 生成不可逆 `deviceFingerprint`。
- 后端保存 `terminalCode`、`deviceFingerprint`、`agentToken`。
- Admin 绑定终端到机构。
- Partner 学校账号只能查看和配置本机构名下终端。

不建议在浏览器端直接读取或传递明文 MAC。浏览器通常不能可靠读取 MAC，且明文 MAC 属于敏感设备标识，不适合暴露到前端。

## Kiosk 首页布局

### 智慧校园关闭

关闭态页面不出现智慧校园。百宝箱当前显示占位框，但不显示任何业务卡。

页面应保持稳定：

- 渲染一个与现有首页视觉语言一致的百宝箱框体。
- 框体内不放功能卡、不放假入口、不放未确认服务。
- 占位高度要克制，不能形成大面积无意义空白。
- 后续配置功能后，功能卡必须在该框体内展示。
- 首页服务区和底部合规说明自然衔接。

### 智慧校园开启

开启态在现有首页服务区后方插入智慧校园专区：

- 外层：全宽横向专区，宽度对齐当前首页内容容器。
- 顶部：图标、标题「智慧校园」、状态徽标「学校端已开启」、说明文案、进入专区按钮。
- 内容：只显示后端授权且可用的校园子模块。
- 当前可用模块：迎新服务、行李帮运、VR 校园。
- 冻结模块：校园大数据，不显示入口。

当只有一个模块开启时，模块卡不应显得孤立或像页面缺内容。推荐策略：

- 仍保持整块专区宽度。
- 模块卡使用固定最小宽度。
- 右侧留白保持干净，不额外补假卡。
- 后续模块开启后自然扩展为多卡布局。

## 智慧校园专区交互

- 点击「进入专区」进入 `/smart-campus`。
- 点击子模块卡进入对应子页面。
- 子页面进入前再次读取或消费当前策略。
- 如果策略关闭或模块未授权，展示受控空态，不渲染业务内容。
- 返回按钮始终回到首页或智慧校园专区，不进入死路。

## 后台管理

Admin 需要能完成：

- 查看终端列表、在线状态、所属机构。
- 查看设备指纹摘要，不展示完整原始设备信息。
- 绑定和解绑终端机构。
- 开关智慧校园总开关。
- 开关已开放子模块。
- 明确显示校园大数据冻结状态。
- 记录修改人、修改时间和审计日志。

Partner 需要能完成：

- 只查看本机构终端。
- 只配置本机构终端的智慧校园开关和开放子模块。
- 不能绑定、解绑或改绑设备。
- 不能配置其他机构终端。
- 不能开启校园大数据。

## 合规边界

智慧校园本期只做校方服务入口和信息指引：

- 不采集学生个人信息。
- 不采集报到登记信息。
- 不做校园大数据统计展示。
- 不做人脸、定位、未成年人或生物识别能力。
- 不承接岗位投递、企业筛选、面试邀约或招聘闭环。

相关文案必须保持：

- 「学校专属服务专区」
- 「仅校园终端开启时显示」
- 「关闭后整块消失，不占首页空白」
- 「报到登记、缴费等请前往学校官方系统办理」

## 文件拆分建议

实现时不继续把首页逻辑堆到 `HomePage.tsx`。
当前运行时代码先在 `HomePage.tsx` 内最小落地 `ToolboxSection`；后续继续扩展百宝箱内容或智慧校园策略时，再按本节拆分。

建议拆分：

- `apps/kiosk/src/pages/home/HomePage.tsx`：首页编排。
- `apps/kiosk/src/pages/home/components/SmartCampusPanel.tsx`：智慧校园专区。
- `apps/kiosk/src/pages/home/components/HomeToolboxSection.tsx`：百宝箱占位框与后续承载规则。
- `apps/kiosk/src/pages/home/homeServiceGroups.ts`：核心服务分组配置。
- `apps/kiosk/src/pages/smart-campus/SmartCampusRouteGuard.tsx`：智慧校园子路由守卫。

后续百宝箱有明确内容后，再新增对应配置和组件，不在本期预置假入口。

## 验收标准

静态与单元验证：

- `verify:smart-campus-ui` 继续通过。
- `verify:partner-smart-campus` 继续通过。
- 新增首页显示门禁验证：
  - enabled=false 时首页源码不渲染智慧校园专区。
  - enabled=true 且有可用模块时渲染专区。
  - bigdata 不出现在首页入口。
  - 百宝箱当前无业务卡配置，但会渲染可见占位框。

浏览器验收：

- 27 寸竖屏比例下，智慧校园关闭态无空白专区。
- 27 寸竖屏比例下，百宝箱占位框可见但不空洞、不伪装成可用功能。
- 27 寸竖屏比例下，智慧校园开启态专区与首页容器对齐。
- 只有一个校园模块开启时页面不破、不挤、不假补位。
- 子路由直达在关闭态显示受控未开启状态。

安全与权限验收：

- Partner 不能看到其他机构终端。
- Partner 不能配置其他机构终端。
- Admin 解绑终端后，Partner 不再可见该终端。
- 保存 `bigdata=true` 时后端仍强制返回 `bigdata=false`。
- 所有配置变更写入审计日志。

## 非目标

本期不做：

- 百宝箱具体业务内容。
- 全局无感数据刷新机制。
- 校园内容管理后台。
- 校园大数据。
- 学生个人信息采集。
- 明文 MAC 绑定。
- 新增底部 Tab 或新的首页一级入口。

## 后续决策点

后续需要产品确认后才能推进：

- 百宝箱第一批放哪些内容。
- 智慧校园是否需要学校自定义图标、学校名称或校区名称。
- 迎新服务内容是否从静态前端迁移到学校后台配置。
- 行李帮运和 VR 校园是否需要外部官方链接审核流。
- 智慧校园开启后是否需要专门的校园模式首页，而不是仅首页专区。
