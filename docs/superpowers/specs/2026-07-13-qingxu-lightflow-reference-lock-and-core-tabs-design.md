# 青序 LightFlow 4188 参考冻结与核心三 Tab 设计补充

> 日期：2026-07-13
> 状态：用户已批准本书面规格；首页与真实登录弹窗实施计划已形成，尚未修改运行时代码。
> 适用范围：Kiosk 首页、AI助手、我的，以及三端全页面 LightFlow 迁移的视觉一致性。
> 上位规范：`docs/superpowers/specs/2026-07-11-service-desk-commercial-ux-migration-design.md`。

## 1. 决策摘要

用户确认继续以 4188 当前页面作为青序 LightFlow 的视觉与 Kiosk 基础布局参考，只有以下明确改动：

1. 首页删除“先问清楚，再开始办理”大标题及其旁边的小青人物。
2. 首页顶部改为纯文字品牌和真实设备状态，不显示额外图形 Logo。
3. 原首屏“直接办理服务”位置改为登录入口。
4. 登录入口上方保留一张简洁的服务价值说明卡。
5. AI助手页删除左上角重复的“AI助手”可见标题。
6. 我的页删除左上角重复的“我的”可见标题。

除以上六项外，4188 的蓝白视觉、分类导航、业务卡片、底部三 Tab、留白、圆角、层级与交互方向保持不变。实施不得借本次调整重新设计另一套首页或局部主题。

第 1–4 项属于首页校准批次；第 5 项只随 K2a AI助手波次落地，第 6 项只随 K4 本人资产波次落地，不在首页批次跨文件提前修改。

## 2. 与既有规范的关系

本补充只收紧 4188 参考的权威范围，不覆盖产品、合规、权限、API 或真实状态机：

- 首页及 Kiosk 核心三 Tab 的视觉和基础布局以 4188 为直接参考，并执行本文列出的六项例外。
- 其他 Kiosk 页面使用同一视觉语言和组件语法，但按真实任务流程组织内容，不把首页布局机械复制到任务页。
- Admin / Partner 继续使用同一 LightFlow token、表面、色彩和状态语言，但保留后台所需的信息密度、侧栏、表格、筛选、抽屉和权限结构。
- “项目全部更新为 4188 效果”指三端最终退出墨青纸感、暖米纸、墨绿主色和宋体大标题，不指把所有页面做成同一张首页。
- 4188 的演示数据、假登录、原型提示和本地假状态不得进入正式代码。
- 本文将上位规范中“小青真人顾问作为 Kiosk 品牌人物资产”的展示位置收窄为 AI 顾问内容区；首页不再展示小青，发生冲突时以本文为准。

本补充与既有规范“共享设计系统分批迁移”不冲突。仍禁止一次性重写全部页面，仍按业务波次独立验证、提交和回滚。

## 3. 首页冻结结构

首页从上到下固定为以下顺序：

1. 纯文字顶部栏。
2. 服务价值说明卡。
3. 登录身份卡。
4. “当前可使用功能”区块标题。
5. 六类业务导航。
6. 对应业务卡片与真实启用 / 禁用状态。
7. 终端配置决定的扩展服务。
8. 合规来源说明。
9. 首页 / AI助手 / 我的固定底部导航。

### 3.1 顶部栏

- 左侧只显示“AI求职打印一体机”文字，不显示额外 Logo 图形。
- 右侧显示真实设备状态，视觉形态为简洁文字和状态点。
- “打印机在线”只能在真实设备状态为在线时出现；离线、异常、未知必须使用对应真实文案。
- 网络正常时不单列；网络异常时附加到同一紧凑状态组，不新增第二条大型状态栏。

### 3.2 服务价值说明卡

- 不再出现“小青”人物、咨询入口或“先问清楚，再开始办理”文案。
- 标签固定使用“一站式求职服务”，不使用设计过程中的“推荐方案”元文案。
- 主标题固定表达为“简历、打印、岗位信息一趟办完”。
- 说明文案表达真实服务范围，不承诺一键投递、录用结果或平台内招聘闭环。
- 卡片只说明价值，不承载业务按钮，避免与登录入口和下方业务入口争抢。

### 3.3 登录身份卡

登录身份卡紧接服务价值卡，是首页第二个主要内容区，不做右上角孤立按钮，也不把输入框常驻首屏。

游客态：

- 标题：“未登录 · 可先体验”。
- 说明：“登录后查看本人简历、文档、AI记录、打印订单和收藏”。
- 主操作：“登录 / 注册”。
- 基础业务入口继续可用，不要求用户先登录。

已登录态：

- 显示脱敏昵称或手机号，不显示完整手机号。
- 说明本人记录已绑定当前账号。
- 主操作改为“进入我的”。
- 提供清楚但不抢主操作的“退出”入口。

登录中、失败、过期与清场态：

- 登录请求进行中防重复提交，并保持取消或关闭能力。
- 验证码错误、网络失败、频率限制和会话过期显示可恢复错误。
- 登录失败不清空用户在受保护业务流程中的安全草稿。
- 自动退出或主动退出后，身份卡恢复游客态并清理可见敏感信息。

## 4. 登录窗口

点击首页登录身份卡、底部“我的”游客入口，或需要本人记录的功能时，统一打开同一真实登录窗口。

### 4.1 位置与尺寸

- 1080×1920 一体机：当前页面居中弹层，背景克制遮罩，不离开原业务上下文。
- 390×844 / 390×700：使用接近全宽的底部弹层；可用高度不足时由同一弹层扩展到安全全屏高度，确保虚拟键盘不遮挡输入、错误和主操作。
- 弹层打开时锁定背景滚动，焦点进入手机号输入；关闭后焦点返回触发按钮。

### 4.2 内容

- 标题：“手机号登录”。
- 手机号输入。
- 验证码输入与“获取验证码”。
- 主操作：“登录 / 注册”。
- 次操作：“继续游客体验”。
- 公共设备提示：长时间无操作自动退出并清理本次会话。
- 用户协议与隐私政策继续使用项目已有真实文案和链接，不复制原型占位内容。

### 4.3 行为边界

- 复用现有真实短信验证码、deviceId、token、会话失效通知和安全回跳机制。
- 不在首页复制第二套认证请求、验证码倒计时或登录状态。
- 登录成功后返回原页面、原滚动位置和原任务；受保护入口继续原动作。
- 用户关闭或选择游客体验时，不伪装成已登录，不创建本人资产。
- 弹层实现如需从现有登录页抽取可复用面板，只允许无行为拆分；认证逻辑改造必须另立高风险任务。

## 5. AI助手与我的页面标题

### 5.1 AI助手

- 删除页面左上角重复的“AI助手”可见标题。
- 保留底部导航中的“AI助手”标签和选中态。
- 保留文档标题、路由语义和屏幕阅读器可识别的页面名称。
- 小青人物只在 AI 顾问内容区出现，不恢复到首页首屏。

### 5.2 我的

- 删除页面左上角重复的“我的”可见标题。
- 保留底部导航中的“我的”标签和选中态。
- 保留本人资产、登录态、隐私和账户操作的语义标题。
- 不因删除重复标题而删除登录入口、资产分区、状态、错误或合规说明。

上述删除只针对视觉上重复的页头文字。若可访问性需要一级标题，应使用内容区真实任务标题或视觉隐藏标题，不允许让页面失去可访问名称。

## 6. 4188 视觉冻结项

除本文明确变更外，下列视觉规则保持：

- 浅冰蓝画布、白色表面、深海军蓝文字和亮蓝主操作。
- 清晰无衬线字体、克制阴影、细边框与大留白。
- 首页六类业务导航和真实业务卡片结构。
- 业务分类可有柔和点缀色，但状态色不与分类色混用。
- 主要触控目标不小于 56px，普通交互不小于 48px。
- 全局底部导航固定为首页 / AI助手 / 我的，不新增 Tab；`/campus` 等已存在的沉浸子区自有导航不属于全局三 Tab，不因本文增删。
- 390 与 1080 使用同一信息层级，移动端只重排，不删除关键状态和隐私信息。
- 动效只用于状态反馈与层级过渡，并遵守 reduced motion。

任何实施批次不得：

- 重新加入暖米纸、墨绿主色、宋体大标题或纸纹背景。
- 在 LightFlow 页面混入第三种视觉语言。
- 为“更丰富”新增重复入口、营销 Hero、装饰性 AI 卡片或假统计。
- 改变 4188 已确认的业务分类和底部导航结构。

## 7. 全项目迁移边界

- Kiosk、Admin、Partner 最终都使用青序 LightFlow 共享 token 和状态语言。
- Kiosk 强调触控和单任务；Admin 强调高密度运营；Partner 强调机构工作流。
- 视觉迁移不得改变路由、API、DTO、Prisma、权限、支付、打印、扫描、AI、TRTC 或硬件状态机。
- 当前真实功能、入口、禁用状态和合规来源必须优先于原型展示。
- 未实现能力继续诚实禁用或隐藏，不能因 4188 有视觉卡片就伪装已上线。
- 旧主题只在所有目标路由、构建、测试和文档引用清零后退出。

## 8. 并行任务与实施顺序

1. 首页按本文做小范围布局校准，不重写业务源、统计、续办、扩展服务或终端配置。
2. AI助手迁移必须先吸收或等待当前透明顾问素材与助手页面任务完成，禁止覆盖未提交资产。
3. 我的页必须等待“我的页商用闭环第一批”完成、迁移或明确废弃后再执行 LightFlow K4。
4. 其他页面继续按 K1–K6、A1–A4、P1–P3 波次推进。
5. 最后统一状态、可访问性、旧主题退出和三端终验。

## 9. 验收标准

首页：

- 不存在“小青”首屏人物和“先问清楚，再开始办理”。
- 顶部为纯文字品牌与真实设备状态。
- 服务价值卡、登录身份卡、业务分类顺序准确。
- 4188 其余视觉与 UI 结构无未经批准的变化。
- 游客和已登录身份卡都由真实状态驱动。

登录窗口：

- 复用真实认证链路；无演示登录或假成功。
- 关闭、失败、过期、重复提交、虚拟键盘和焦点恢复可用。
- 登录成功后返回原任务；游客取消后仍可使用基础服务。

核心三 Tab：

- AI助手与我的左上角不再出现重复可见标题。
- 底部三 Tab 文案、路由与选中态保持。
- 1080×1920、390×844、390×700 无横向溢出，触控尺寸合格。

全项目：

- 每批 typecheck、lint、production build、相关 verify、浏览器矩阵和 diff 检查通过。
- 超过 30 行或跨模块变更按项目规则调用前端模型与 Claude 双模型审查；无有效报告时如实记录。
- 未 push、未部署、未预生产或未真机验证时，不得宣称全页面或商用上线完成。
