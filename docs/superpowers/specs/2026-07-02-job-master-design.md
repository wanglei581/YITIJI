# 岗位大师(岗位决策分析台)设计文档

> 定稿日期:2026-07-02
> 状态:设计已确认(方案A),开发排期为**上线收口完成后**
> 决策记录:用户已确认采用方案A「决策中枢」,并确认本轮只沉淀设计文档、不动代码

## 一、背景与定位

「岗位大师」目前是 Kiosk 首页「岗位信息」组内的禁用占位磁贴(`apps/kiosk/src/pages/home/HomePage.tsx` 中 `disabled: true`,标注"即将上线")。

**产品定位**:一次 5-10 分钟的站立触控会话,输入"我的简历 + 我看中的岗位",输出一份可打印的《岗位决策参考报告》,为求职者与转行者提供数据化决策参考。融合四项分析能力:

1. 岗位适配度(分档 + 可解释技能命中清单)
2. 薪资区间参考(来源方区间 + 站内同类岗位统计)
3. 晋升路径模拟(当前 → 1-3年 → 3-5年 三节点)
4. 职业风险标注(定性三档 + 依据说明)

**差异化核心**:纸质报告带走——所有线上竞品都做不到,是一体机场景独有优势。

**不是什么**:不是投递工具、不是招聘闭环、不是薪资预测模型、不是职业焦虑制造机。

## 二、合规边界与红线(硬约束)

继承并扩展现有 job-fit 防护栏:

- 不输出百分比、录用概率、通过率、"精准命中"等表述(现有 LLM 输出过滤已覆盖,含小数百分比)。
- 适配度只用三档参考等级(`reference_high/medium/low`)+ 可解释命中项。
- 薪资只展示"来源方提供区间"与"站内同类岗位统计区间",标注来源与"仅供参考";样本不足时显示"暂无参考数据",禁止硬给(Glassdoor 做法)。禁止自建薪资预测模型。
- 风险/前景只用定性三档(前景较好/一般/需谨慎)+ 依据说明;禁止"自动化替代概率 N%"等强预测强焦虑数字(O*NET 做法)。
- 行动按钮仅限合规白名单:查看岗位 / 去来源平台投递 / 扫码投递 / 打印报告 / 去优化简历。
- 报告页脚固定:数据来源(本人简历 + 第三方岗位信息 + 站内统计)、生成时间、"结果仅供求职参考,不构成录用承诺或薪酬承诺"。
- LLM 建议不编造经历(evidence 必须来自简历原文),不构造投递闭环语义。
- 对外宣传"AI 岗位分析"前,确认生成式 AI 备案状态(智联"AI改简历"先例)。

## 三、竞品调研结论(2026-07-02 调研,取其精华)

| 借鉴对象 | 移植点 |
|---|---|
| LinkedIn Career Explorer | "已具备技能 ✓ / 建议补足 ✗"双栏呈现;免责声明写明数据口径 |
| 超级简历(JD 关键词匹配) | 技能命中清单是可解释性最强、合规风险最低的适配度算法 |
| BOSS直聘 | 匹配结果给"推荐理由"文字而非裸分数 |
| Glassdoor / Levels.fyi | 薪资区间+中位数;样本不足不显示;浮动薪酬不估 |
| FutureFit AI / Lightcast | "职业 GPS"叙事:当前位置→目的地→路线;每步挂缺口技能 |
| O*NET My Next Move | 前景只做三档定性标签+官方统计依据 |
| 广州「海纳职通」终端 | "诊断分可提升"叙事(优化简历后可重测)在政务终端场景被验证有效 |
| 脉脉竞争力报告 | 轻输入重输出:少量字段生成单屏卡片化报告 |

**明确不移植**:自建薪资预测模型、自动化替代概率、可拖拽职业网络图、长对话 AI 职业教练、基于历史行为的个性化推荐、任何形似投递闭环的"匹配后一键行动"。

## 四、用户流程

```
入口①首页「岗位大师」磁贴(点亮) / 入口②岗位详情页「分析这个岗位」(改线,避免同义入口)
  ↓
Step 1 技能画像
  有简历(kind=parse 完成) → 自动提取画像摘要
  无简历 → 引导上传/扫描(复用现有简历上传流程)
  ↓
Step 2 选岗(复用 job-fit 双模式交互)
  A. 站内岗位选择(M1 单岗位;M2 支持 1-3 个对比)
  B. 手填目标岗位(标题 ≤50 字 + 要求 ≤2000 字)
  ↓
Step 3 生成分析(单次 LLM 任务 + 站内统计,四段输出,进度条)
  ↓
Step 4 结果页(竖屏纵向四段卡片)+ 行动出口(合规白名单按钮)
  ↓
Step 5 打印《岗位决策参考报告》→ 我的文档 + 打印订单;记录进「我的」AI服务记录
```

## 五、功能规格

### 5.1 适配度分析(M1)

- 输入:简历原文(`ResumeExtractionService` 凭 fileId 重提,不落库)+ 岗位上下文(站内 Job 记录含 `requirements/skillsJson/educationRequirement/experienceRequirement`,或手填)。
- 输出:
  - `fitLevel`:三档参考等级(沿用 job-fit 语义)。
  - `matchedSkills[]`:已具备项,每项含 `skill + evidence`(evidence 源自简历原文)。
  - `gapSkills[]`:建议补足项,每项含 `skill + suggestion`(只谈准备方向,不编造经历)。
  - `summaryReason`:一句话理由(BOSS直聘式"为什么")。
- 防线:复用 job-fit 禁词拦截、自相矛盾检测、防编造(FABRICATION_HINT),重试 1 次仍命中则诚实失败。

### 5.2 薪资区间参考(M2)

- 双轨呈现:
  1. **来源方区间**:岗位自带 `salaryMin/salaryMax/salaryUnit`,标注"来源方提供";缺失显示"来源平台未提供"(现有口径)。
  2. **站内统计区间**:对 `approved+published` 岗位按 `category`(必要时叠加 `city`)聚合 P25/P50/P75;纯 SQL 计算,不进 LLM,不引入外部数据。
- 降级:同类样本数 < N(建议 N=10,实现时定稿)时显示"同类岗位样本不足,暂无统计参考",禁止硬给。
- 命名用"参考",不用"预测"。

### 5.3 晋升路径模拟(M1)

- 输出 `careerPath`:固定三节点纵向结构(竖屏时间轴):
  - `current`:当前定位(基于简历,含 evidence)。
  - `next`(1-3年):进阶方向 + 2-3 项待补技能 + 第一步行动。
  - `target`(3-5年):目标方向 + 2-3 项待补技能。
- 与 2E 职业规划的边界:职业规划是"泛方向+技能计划+行动清单"(不挂具体岗位);岗位大师路径**锚定所选岗位**向上模拟。结果页提供"想要完整规划?去职业规划"交叉引流,不重复造轮子。

### 5.4 职业风险标注(M1 基础版 / M3 增强版)

- M1 基础版(LLM 可解释判断,不做预测):
  - 硬门槛风险:学历/经验/证书与岗位要求存在明确硬性差距时标注(evidence 为岗位要求原文)。
  - 信息完整度风险:岗位关键信息缺失(薪资/要求不全)时提示"信息有限,建议到来源平台核实"。
  - 每条风险 = `label(定性三档) + reason + basis(依据出处)`。
- M3 增强版:职业前景标签(前景较好/一般),依据人社部门公开的紧缺职业目录/重点产业目录,由 Admin 维护数据,不由 LLM 生成。

### 5.5 决策报告 PDF(M1)

- 复用 `CareerPlanPdfService` 模式新建 `JobMasterPdfService`:A4 单页优先(最多 2 页),五区块:岗位与本人概要 / 适配度双栏 / 薪资参考 / 路径三节点 / 风险与建议 + 页脚免责。
- 走既有链路:PDF buffer → `FileObject(purpose='job_master_report')` → `/me/documents` → `PrintTask` 打印确认。

## 十、商业化与付费设计(2026-07-02 补充)

### 10.1 免费/付费切分原则

免费=获客与信任,付费=深度、载体、次数。切分线砍在"用多深、带不带走、用几次",不砍在"能不能用"。

**必须免费(底线)**:
- 岗位/招聘会浏览与来源跳转(合规定位为信息入口,禁止收费卡通道)。
- 简历上传 + 基础诊断分(付费漏斗的流量入口)。
- 岗位大师**结果预览**:适配度三档结论 + 1-2 条关键命中/差距 + 一句话理由。
- 每人**首次完整分析免费**(手机号登录领取,匿名转会员)。

**付费点**:
- 完整决策报告(全部命中/差距清单 + 路径三节点 + 风险 + 薪资参考),单次个位数定价,对标线下职业咨询。
- 纸质报告打印(与打印业务统一定价,一体机独占付费点)。
- 多岗位对比(2-3 岗,单岗价 1.5-2 倍)。
- 优化后重测(绑定"可提升叙事"驱动复购)。

### 10.2 付费载体:单次 / 套餐次卡 / 卡券补贴三轨

1. **单次扫码支付**(微信/支付宝 Native,不强制注册)。
2. **套餐按求职旅程打包**(必须含打印额度,与主业互导):求职加油包(诊断+优化+岗位大师×3+打印20页,主推)、面试冲刺包(岗位大师×2+模拟面试×2+打印)、转行规划包(职业规划+多岗对比+RIASEC)、岗位大师 5 次卡(90 天有效,绑会员)。
3. **卡券/补贴核销(政务场景关键)**:政府补贴券(人社经费买单、用户 0 元、后台核销对账,呈现"本次服务由 XX 就业补贴支持,补贴后 0 元")、合作机构/学校批量赠券(B 端团购)、打印↔AI 双向抵扣券、新用户券/推荐券。

### 10.3 价值感知设计

- **先给再要,锁而不藏**:免费预览展示付费内容的标题与条数("已识别 7 项匹配技能、4 项待补技能"),内容模糊;用户付费是"拿走已算好的东西"。
- **物理锚点**:核心交付物为 A4 纸质报告("把职业顾问的分析带回家")。
- **时机**:付费推荐放在岗位详情页"分析这个岗位"→免费预览之后的情绪高点;首页冷流量先给免费首次。
- **失败不扣费**:LLM 失败/样本不足诚实失败且不收费。
- **可提升闭环**:报告末尾印"优化简历后 30 天内凭本单重测半价"。
- **明码标价**:终端与报告公示价目、可开发票;文案禁止暗示"付费=更高录用率",付费买的是更完整的分析报告。

### 10.4 后台与技术要点(独立 M-Pay 阶段,排 M2 之后)

- 权益模型:`QuotaWallet`(次数余额)+ `VoucherCode`(券批次/核销)+ `PriceRule`(Admin 配置);复用已合入订单底座,`PaymentAttempt`/退款需重新设计补齐(当年未迁入)。
- Admin:价格策略、券批次生成发放、核销记录、政府补贴对账报表、退款处理、免费额度、困难群体减免通道——与打印扫描收口清单已规划的"价格策略、权益券、免费额度、补贴核销"共用同一套底座,不建两套。
- 合规:退款政策公示、未消费次卡可退、政务场所收费须与场地方/政府事先约定(部分场景必须走全补贴模式)。

## 十一、风险与开放问题

- 单次 LLM 输出四段结构较长,需评估 token 上限与超时;必要时拆两次调用(适配度+风险 / 路径),实现时定稿。
- 站内岗位量少时统计区间可信度低——样本阈值降级是底线,阈值 N 实现时用真实数据校准。
- 「岗位大师」与既有 `/resume/job-fit`、`/jobs/:id` AI 解读/匹配存在能力交叠,M2 完成入口改线前需复查 user-data-flow-matrix,确保不堆同义入口。
- 生成式 AI 备案范围是否覆盖本功能,对外宣传前与法务确认。
