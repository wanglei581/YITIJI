# COS 生命周期与隐私文案验收设计

## 目标

本分支收口用户文件与简历资产商用闭环中的 COS 生命周期与隐私告知部分：用户看到的保存期限口径必须与代码中的 `months_3`、`months_6`、`long_term`、`system_short` 一致；生产验收必须明确腾讯云 COS 私有桶不能配置会误删 `long_term` 对象的全局过期规则。

## 范围

本分支只做本地文档、文案和防回退验证：

- Kiosk 采集点隐私承诺、帮助中心、隐私政策说明更新为分级保存口径。
- Admin 文件管理合规横幅更新为分级生命周期管理口径。
- 新增 COS 生命周期合规文档，明确 DB `expiresAt` + 清理任务是业务删除主路径，COS 控制台生命周期只能作为受限兜底。
- 生产部署 runbook/checklist 增加 COS 生命周期人工验收项。
- 新增静态 verify，防止文案退回旧的“1 小时内自动删除/默认 24 小时”通用口径，防止代码引入桶级 lifecycle 写操作。

## 非目标

- 不连接、读取或修改真实腾讯云 COS 资源。
- 不修改 `retention-policy.ts`、`files.service.ts`、`files.cleanup.task.ts`、Prisma schema 或迁移。
- 不静默变更 `FILE_RETENTION_CONSENT_VERSION`。本次只记录隐私文案与保存条款版本的耦合：若未来法务确认长期保存条款实质变更，需要独立分支决策是否 bump，并触发用户重新确认。
- 不新增招聘平台闭环功能。

## 保存期限口径

- 游客、证件照、身份证复印件、匿名/系统文件：短期保存，到期自动物理清理。
- 登录会员原始简历/求职材料：默认保存 90 天。
- 用户主动确认后：可延长至 180 天；优化后或派生成果物可选择长期保存。
- 删除：用户主动删除或到期清理均应物理删除对象文件，并保留必要审计记录，不保留文件本身。

## COS 生命周期原则

腾讯云 COS 私有桶不得配置 bucket 全局过期规则。`long_term` 文件在数据库中的 `expiresAt` 为 `null`，只能由业务逻辑决定是否删除；如果 COS 控制台按桶或按包含长期对象的前缀配置过期规则，会绕过数据库和审计直接删除用户长期资产。

允许的 COS 生命周期只作为兜底：

- 仅限确认不会承载长期资产的临时前缀。
- 必须在生产验收时截图存档：规则名称、作用前缀、过期天数、是否启用。
- `users/`、会员简历、优化成果、机构资料、审计相关对象不得被过期规则覆盖。

## 验证

- `verify:legal-retention-copy`：扫描 Kiosk 法务页、帮助中心和 shared 合规文案，要求包含 90 天、180 天、长期保存、短期高敏口径，并禁止旧通用短留存文案。
- `verify:cos-lifecycle-policy`：扫描 COS 生命周期合规文档、部署 runbook/checklist、COS backend 代码，要求文档包含禁止全局生命周期、人工验收、截图存档、`long_term`/`expiresAt=null` 说明，并禁止后端代码出现 `putBucketLifecycle` 等桶级 lifecycle 写操作。
