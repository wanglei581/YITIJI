## 一、Kiosk(一体机前台)— 23 张

### 1. 主框架(首页 + 登录 + 我的)

#### kiosk/01-首页上半 + 02-首页下半

- **意图**:7 大模块入口的卡片墙首页。三列布局:
  - **AI 简历服务**(7 入口):简历上传 / 纸质扫描 / AI 诊断 / 简历优化 / 简历打印 / 求职材料 / 智能匹配
  - **岗位信息**(4 入口):全职 / 实习 / 兼职 / 智能匹配
  - **招聘会**(3 入口):招聘活动 / 预约参会 / 扫码签到
  - **打印扫描**(6 入口):文档打印 / 证件复印 / 文档扫描 / 云打印 / 格式转换 / 证件照打印
  - **AI 面试训练**(3 入口):模拟面试 / 面试技巧 / 面试复盘
  - **政策服务**(3 入口):就业政策 / 补贴申请 / 档案/登记
  - 顶部 banner:背景图 + "AI 求职打印一体机" 标志 + 欢迎卡 "立即登录/注册"
  - 底栏 tab:首页 / AI 助手 / 我的
- **合规**:✅ 全部入口名称合规("智能匹配"是岗位匹配建议,不涉及反向推送企业)
- **现项目对应**:[apps/kiosk/src/pages/home/HomePage.tsx](apps/kiosk/src/pages/home/HomePage.tsx)
- **差距**:现项目首页入口较简,缺少打印扫描的细分入口(证件复印/格式转换等)、求职材料、政策服务入口
- **实施方案**:重做首页为「卡片墙」布局,每张卡 = 一个一级模块,卡内列子入口。**用 shadcn 卡片 + Tailwind grid-cols-3**;**不抄秒哒的渐变色和毛玻璃**(CLAUDE.md §8)。卡片用白底 + 1px 边框 + 轻阴影。
- **工作量**:**M**(1-2 天 — 重新设计模块卡 + 子入口路由占位)
- **优先级**:**P0**

#### kiosk/03-登录页面

- **意图**:登录 tab(手机号/邮箱/扫码)+ 6 位验证码 + 用户协议 + "回到首页"。
- **合规**:⚠️ **争议点** — CLAUDE.md §9 与现项目原本设计 Kiosk **无登录**;但秒哒原型设计为有登录,主要为查"我的"模块(我的简历/订单/打印记录/预约记录)。
  - **替代方案**(用户已倾向轻量化):**取件码 / 扫码取件 / 手机号免密**,不强制注册;长期记录(简历库、订单史)走"扫码绑定手机"快速识别。
- **现项目对应**:**暂无**
- **差距**:Phase C 已实现 Admin/Partner 登录,Kiosk 端尚无登录闭环
- **实施方案**:做"扫码 / 手机号+验证码 / 取件码"三 tab;**不做注册流程**(避免触红线"求职者数据收集");登录后只解锁"我的" + AI 简历库;**未登录用户可用打印扫描和岗位浏览**(无障碍模式)。
- **工作量**:**M**(2-3 天 — 后端要发短信验证码 / 二维码 token / 取件码)
- **优先级**:**P1**(先做 P0 模块,登录可后置)

#### kiosk/04 + 05-我的(上+下)

- **意图**:个人中心,4 大区块:
  - **我的求职**:外部投递记录 / AI 服务记录 / 收藏岗位 / 岗位浏览记录
  - **我的简历**:我的简历 / AI 诊断报告 / 优化记录 / 简历模板
  - **招聘会预约记录**:我的预约 / 缴费记录 / 入场凭证 / 签到记录
  - **打印 / 扫描**:我的订单 / 我的文档 / 扫描文件 / 优惠券 / 附近设备 / 简历打印
  - **账户与服务**:消息通知 / 账号设置 / 身份切换 / 帮助中心 / 意见反馈
  - **后台管理入口**:管理员后台 / 合作机构后台
- **合规**:
  - "外部投递记录" ✅(只记跳转,不存简历给企业)
  - "招聘会预约记录" ⚠️ 预约本身走第三方,我方只记"已提交跳转"
  - "缴费记录 / 入场凭证 / 签到记录" ⚠️ 如果是第三方招聘会的票务,我方只展示外部 ID 和跳转码;如果是我方运营的招聘会,需要确认是否涉及"代收个人信息"
  - **"身份切换 / 管理员后台 / 合作机构后台" 入口 ❌**:Kiosk 是公共终端,**绝对不能**出现后台入口(运营安全风险 — 任何人都能点)
- **现项目对应**:[apps/kiosk/src/pages/profile/ProfilePage.tsx](apps/kiosk/src/pages/profile/ProfilePage.tsx)
- **差距**:现项目"我的"较简,缺少 AI 服务记录、收藏岗位、优化记录、附近设备等
- **实施方案**:抄区块结构,**删除后台入口**;"身份切换"只在登录态切换"展示语言/求职目标",不切换角色。
- **工作量**:**M**(1-2 天)
- **优先级**:**P1**
