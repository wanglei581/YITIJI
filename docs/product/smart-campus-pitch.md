# 智慧校园 · 给学校的价值话术单页

> 用途：去高校（就业中心 / 学工处 / 招生处）谈合作时的销售话术与演示脚本。
> 配套演示：一体机前端 [smart-campus-demo.html](../design/prototypes/smart-campus-demo.html) + 学校后台 [smart-campus-admin-demo.html](../design/prototypes/smart-campus-admin-demo.html)。
> 合规边界（对外可讲，建立信任）：[compliance-boundary.md §九](../compliance/compliance-boundary.md)；对接细节：[smart-campus-integration-design.md](./smart-campus-integration-design.md)。

---

## 一句话定位（开场）

> 「这不是一台打印机，是一台**能按场景变身的校园服务终端**——迎新季帮你接住新生、引导报到、现场出证件照和材料；就业季帮你的就业中心做简历、出材料、对接岗位招聘会，**直接服务就业率**；领导要看数据，它还能展示一块迎新成果看板。学校不用管运维，我们后台一键开关。」

---

## 解决学校什么问题（按场景讲，最有共鸣）

| 学校的痛点 | 我们怎么解 | 对应功能 |
|------------|-----------|----------|
| **迎新季**人流爆炸：证件照、入学材料/表格打印需求集中，新生不熟流程找不到路 | 报到指引 + 办事窗口 + 导览，同时把人导进**证件照/材料打印**（现场就能办） | 迎新系统 + 打印主业 |
| **就业季**有就业率考核，就业中心人手不够、缺工具 | AI 简历诊断/优化 + 打印 + 岗位/招聘会信息入口，**帮就业中心完成 KPI** | AI 简历 + 岗位/招聘会入口 |
| 领导要「智慧校园」政绩、招生要宣传素材 | 一块聚合数据看板（生源/专业/性别结构、报到率），**看得见的成果墙** | 校园大数据 |
| 新生/家长扛行李难、想要科技感 | 第三方行李帮运入口 + 360 校园全景 | 行李帮运 / 校园全景 |

---

## 四个核心优势

1. **按机器变脸**：同一台机、同一套后台，校内开、校外关——这是「智慧校园终端」和「普通打印机」的分水岭。
2. **全年触点**：9 月迎新 → 全年就业，不是只有就业季才用，黏性强。
3. **学校零运维**：开关、内容、数据都在后台，学校提需求我们配；要自助也行（后台已留管理区）。
4. **低成本、可控**：行李/全景只做第三方入口、零履约成本；数据只接聚合、不碰个人信息、合规可控。

## 我们做什么 / 不做什么（主动讲边界 = 建立信任）

| 我们做 | 我们不做（合规红线） |
|--------|---------------------|
| 信息展示、指引、材料打印、AI 简历工具 | 不收求职者简历给企业、不做候选人筛选/面试/Offer |
| 校园大数据：只展示学校**授权的聚合脱敏统计** | 不采集、不存储任何学生个人明细（姓名/学号/身份证…） |
| 行李/全景：第三方入口、外链跳转 | 不自营物流、不在终端做支付下单、不承运 |
| 迎新：流程指引 + 官方系统外链 | 不在公共终端采集学生报到/身份信息 |

> 主动说「学生个人数据我们一条都不存、只展示你们授权的聚合数字」，反而让学校放心、显专业。

## 对接很简单（按学校能力分三档，别吓到对方）

- **绝大多数学校**：后台**手工填几个聚合数字** + 录入迎新内容即可，无需任何接口。
- **有数据的学校**：上传一张**已汇总的 Excel**（一行一指标，不是一行一学生）。
- **有系统且 IT 能配合的学校**：对接一个**只返回聚合数字**的接口，我们周期拉取。

> 关键承诺：**接口只对接聚合数据，学生个人明细一律不接**——既合规，也省去学校的数据安全顾虑。

## 演示动线（5 分钟脚本）

1. 打开**一体机前端原型**，点右侧「智慧校园·总开关」**关掉** → 模块整张消失：「出了校园，这台机器就是普通求职打印机」。
2. 打开 → 进「迎新系统」，滑到「入学&求职准备」：「迎新流量直接变成你们的打印/证件照/简历订单」。
3. 进「校园大数据」：「这是给领导和招生看的成果墙，全是聚合数字，不含任何个人信息」。
4. 切到**学校后台原型**：「这些内容和开关，你们在后台就能管/看，还能看到使用统计——知道这台机器到底有多少人用」。

## 谈判要确认的事项（带着这张清单去）

- [ ] 牵头部门是谁：就业中心 / 学工处 / 招生处？谁签、谁日常运营？
- [ ] 校园大数据要不要做？能否签**数据授权书 + 数据处理协议(DPA)**、只给聚合数字？（拿不到则该功能先不做）
- [ ] 数据怎么给：手工 / Excel / 接口？
- [ ] 终端放哪几个点位、归属哪个校区？
- [ ] 行李帮运第三方是否已有合作物流方（有正规资质、纯外链）？
- [ ] 校园全景素材谁提供（学校/供应商）、版权与人脸授权是否清晰？
- [ ] 开关由我方运营代配，还是学校要自助账号？

---

## 不要承诺的话（避免踩坑）

- ❌ 不说「我们帮你运行李 / 包送达」（只做第三方入口）
- ❌ 不说「保面试 / 保录用 / 补贴必到账」
- ❌ 不说「能看到每个学生的数据」（只有聚合）
- ❌ 涉及岗位一律用「查看岗位 / 去来源平台投递」，不说「一键投递 / 平台投递」
