# 校园招聘会产品设计说明

> 最后更新：2026-06-16

## 定位

校园招聘会是第三方/学校官方来源信息入口 + 现场求职服务入口，不是平台自营招聘闭环。

前台采用“全公开聚合 + 学校分类 + 推荐优先”的展示方式：

- 不做“仅本校学生可见”的私域隔离。
- 所有已审核发布的校园招聘会可公开浏览。
- 终端有宿主学校配置后，可在推荐位优先展示本校场次；无宿主配置时展示“推荐场次”，按进行中、即将开始、举办时间排序。

## 端口职责

| 端口 | 职责 |
|------|------|
| Partner 学校账号 | 创建/维护本校招聘会基础信息、主办/承办学校、面向对象、现场服务、入场方式；保存后重新进入管理员审核 |
| Admin 管理员 | 审核来源、补全展示字段、检查合规文案、发布/下架；维护参展企业、岗位明细、展区、活动资料、场馆导览 |
| Kiosk 前台 | 只读展示已审核发布真实数据，提供来源平台预约/投递入口、活动资料打印、AI 求职准备入口 |

## 页面结构

### `/campus`

学校优先公开聚合页：

- 参考图首页结构：顶部返回按钮 + 居中标题“校园招聘会 / 第三方/官方来源信息入口”
- 蓝色“本校优先”主推卡，展示学校/招聘会名称、状态、日期时间、地点、参会企业数、岗位数和“查看招聘会”
- 四个触控筛选按钮：全部高校、全部城市、全部状态、全部时间；点击后弹出选项并真实过滤列表
- 分组列表：本校场次、同城高校、公共就业机构；只展示真实 API 返回的已审核发布数据，不为凑版式硬造假场次
- 底部服务入口：AI简历诊断、活动资料打印、场馆导览
- 合规动作：查看招聘会、扫码预约

### `/campus/:id`

校园招聘会详情页按用户四张参考图拆成三种前台状态：

- 活动概览：顶部返回 + 蓝色活动 Hero + 横向 Tab；下方左侧活动介绍、亮点、面向人群、现场服务、准备清单，右侧来源机构、官方二维码、来源链接和数据说明。
- 参展企业与岗位：顶部标题 + 横向 Tab；统计条、左侧筛选、企业列表、企业详情与岗位列表、右侧“投递方式说明”二维码。投递只进入来源平台，不在系统内收简历。
- 场馆导览 / 活动资料 / AI求职准备：顶部返回/收藏 + 横向 Tab；场馆平面导览优先，活动资料可查看/打印，AI 求职准备只作为本人参考入口。

### `/campus/:id/materials`

校园招聘会活动资料页，复用招聘会资料能力，但保持校园专区路由和沉浸式布局。

- 从详情页“打印资料”进入。
- “返回详情”回到 `/campus/:id`，不跳转到普通招聘会详情。
- 只展示已发布招聘会下已发布资料；没有资料时显示真实空态。

## 合规动作

允许：

- 扫码预约
- 去来源平台预约
- 查看岗位
- 扫码投递
- 去来源平台投递
- 打印资料
- AI简历诊断 / AI简历优化 / 模拟面试 / 职业规划

禁止：

- 平台内投递
- 平台内收取求职者简历给企业
- 企业端候选人筛选
- 企业端面试邀约
- Offer 管理
- 候选人推荐给企业

外部跳转只记录“打开来源入口”行为：

- 招聘会预约：`external_appointment`
- 岗位/企业来源投递入口：`external_apply`

系统不记录预约结果、投递结果，不保存或转发简历给企业。

## 数据真实性与上线护栏

- 本地开发 seed 中的“某大学”“验证招聘会”“example.com/example.org”仅用于开发预览，不能作为生产数据。
- 生产环境或 `EXCLUDE_DEMO_PUBLIC_DATA=true` 严格模式下，公开招聘会接口会过滤演示/验证来源；前台无真实数据时显示空态。
- `db:seed`、`db:seed:fairs`、`db:seed:companies` 在生产环境默认拒绝执行，避免演示数据误写生产库。
- 真实上线数据必须由 Partner 学校账号维护，经过 Admin 审核发布后，才会进入 Kiosk 前台。
