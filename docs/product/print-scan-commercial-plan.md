# 打印扫描板块商用级方案

> 日期：2026-06-30
> 状态：方案已确认，未进入运行时代码实现
> 关联文档：[feature-scope.md](./feature-scope.md) | [user-data-flow-matrix.md](./user-data-flow-matrix.md) | [print-material-pack-prd.md](./print-material-pack-prd.md) | [compliance-boundary.md](../compliance/compliance-boundary.md)

## 1. 定位

打印扫描板块不是单纯的文件上传入口，而是求职终端的线下交付中心。它承接 AI 简历、求职材料、招聘会资料、政策材料和用户自有文档，提供可验证的打印、扫描、复印、证件照、格式转换和材料包服务。

用户已确认首期目标调整为**全功能商用版本**：只要对求职终端有价值的打印扫描能力，都纳入首期开发范围，不再把扫描、证件复印、证件照、U 盘、云上传、格式转换、签名盖章或材料包作为“二期再说”的能力。

首期全功能不等于带病开放。实现策略是：

1. **全部纳入首期开发**：文档打印、手机扫码上传、材料包、真实扫描、证件复印、证件照、U 盘、云上传、格式转换、签名盖章、价格 / 权益、Admin 运维看板、AI 文件体检和 AI 扫描增强。
2. **按模块独立验收**：每个能力必须有前台页面、后端数据模型、Terminal Agent / 外设链路、Admin 管理、数据流、异常处理、审计和验证脚本。
3. **按开关控制开放**：能力完成开发但未通过真机 / 生产链路验收时，可以进入测试模式和 Admin 配置，不对普通用户宣称已上线；测试模式必须由后端鉴权拦截，不能只靠前端隐藏入口。
4. **不出问题优先**：任何会造成错打、重复打印、高敏文件泄露、虚假成功、支付 / 订单状态混淆的问题，都必须 fail-closed，不允许用提示文案掩盖。

## 3. 当前系统边界

### 已具备商用雏形

- Kiosk 上传文件，后端生成私有 `FileObject`、短期签名 URL 和 SHA-256。
- Kiosk 创建 `PrintTask` / `Order`。
- Terminal Agent claim 任务、下载文件、SHA-256 校验、WMI 打印机预检、SumatraPDF 出纸、状态回传。
- 会员端可查看我的文档和打印订单。
- Admin 已有打印任务、终端、打印机和派生告警基础。

### 必须修正的风险

- `PrintTask` 已有 `terminalId` 字段，但当前任务 claim 逻辑按全局 `pending` 领取，未按目标终端过滤；多终端环境会有错打和隐私泄露风险。
- Terminal Agent 本地 SQLite 不可用时会降级为无重启幂等；商用必须 fail-closed 并在 Admin 标记终端 degraded。
- `/print-scan` 中所有能力都进入首期全功能清单，但页面展示必须由能力开关和设备能力驱动；未通过真机验收的能力只能进入测试 / 待启用状态，不能对普通用户展示为可立即使用。
- 扫描、身份证、证件照虽然已有文件 purpose / TTL / 敏感等级基础，但还没有真实硬件流喂入并完成“采集 -> 使用 -> 物理删除 -> 审计”验收。
- 身份证复印、证件照、签名 / 印章源图和高敏扫描件必须强制短 TTL 与禁止长期保存；清理任务必须调用对象存储物理删除并写审计，不能只改数据库状态。

## 5. AI 功能设计

### AI 文件体检

- 判断纸张尺寸、方向、低分辨率、空白页、页数异常。
- 检测疑似身份证、银行卡、手机号、邮箱、住址等敏感信息。
- 输出风险提示，不自动修改用户文件；如要生成遮挡版本，必须产生新的派生文件并让用户确认。

### AI 求职材料包

- 基于用户本人简历、我的文档、收藏岗位、招聘会资料和求职材料库生成“打印建议”。
- 输出建议组合：简历版本、求职信、作品集封面、材料清单、招聘会资料。
- 只能用于个人自用打印建议，不得形成平台投递、企业候选人推荐或简历流转。

### AI 扫描增强

- 真机扫描后自动裁边、纠偏、去阴影、去空白页、OCR 和文档分类。
- 扫描结果默认短 TTL，用户确认后才能保存到我的文档；身份证 / 证件照不允许长期保存。

### AI 证件照

- 支持上传照片后自动抠图、换底色、规格检测、排版成可打印 PDF。
- 首期不接摄像头也可先做“上传照片 -> 检测 -> 排版 -> 打印”。
- 证件照文件 purpose 应独立，默认高敏短期保存。

### AI 故障解释

- 将 Agent 错误码翻译为用户可理解动作：缺纸、卡纸、打印机离线、文件过期、校验失败、任务已取消。
- Admin 侧聚合失败原因 Top N，辅助运维和耗材管理。

## 6. 后端与数据模型

### 核心对象

- `FileObject`：文件归属、purpose、assetCategory、sensitiveLevel、expiresAt、sha256、storageKey、createdBy。
- `PrintTask`：目标 `terminalId`、fileId、fileUrl 短签名、printParams、status、claimExpiry、taskExpiry、errorCode。
- `PrintTaskStatusLog`：状态流转审计，记录 pending / claimed / printing / completed / failed / expired。
- `Order`：统一订单壳，承载 print / scan / copy / photo / material_pack，后续接支付、权益、退款。
- `ScanTask`：扫描类型、DPI、单双面、ADF / 平板、输出格式、结果 fileId、状态日志。
- `CopyTask`：复印类型、份数、色彩、单双面、是否上传存档、临时 fileId。
- `PhotoTask`：规格、底色、裁剪结果、排版 PDF、打印参数。
- `MaterialPackTask`：材料包主任务，包含多个子 `PrintTask`，支持单份失败重试。
- `DeviceCapability`：终端 / 打印机能力、纸张、彩色、双面、扫描、U 盘、摄像头、Agent 版本。
- `Alert`：终端离线、SQLite degraded、缺纸、卡纸、墨粉低、状态回传积压、文件清理失败。

### 关键规则

- 创建打印任务时必须绑定目标终端；Agent 只能 claim 目标为自己的任务。
- Agent claim 必须有租约 TTL，Agent 断网 / 崩溃 / 长时间不回传时，任务自动释放或失败，不能永久停留在 claimed。
- SQLite 不可用时 Agent 不进入打印循环，并向后端上报 degraded。
- 高敏 purpose 强制短 TTL，不能被会员长期保存策略覆盖。
- 所有签名 URL 只短期有效，日志和 Admin 页面不得展示完整签名。
- `PrintTask` 需要任务级 TTL，未领取 / 未取件任务自动 expired 并清理文件引用。
- 奔图彩色、双面和纸张 mode 必须来自设备能力或 Admin 配置映射；厂家确认前禁止硬编码 `"color"` 等驱动值。

## 7. 数据流

### 单文件打印

1. 用户在 Kiosk 上传文件或从我的文档选择文件。
2. 后端创建 `FileObject`，返回短签名 URL、fileId、sha256。
3. AI 材料检查产出检查结果，必要时生成派生文件。
4. 用户确认打印参数和隐私提示。
5. 后端创建绑定当前 `terminalId` 的 `PrintTask` 和 `Order`。
6. Terminal Agent 按自身 `terminalId` claim 任务，下载短签名文件并校验 sha256。
7. Agent 打印并持续回传状态。
8. 会员端和 Admin 端展示状态；完成后按保存策略清理或留存。

### 扫描

1. 用户选择扫描类型和参数。
2. Kiosk 创建 `ScanTask`，绑定当前终端。
3. Agent / Session Helper 调用 TWAIN / WIA 或扫描目录监听。
4. Agent 上传扫描结果，后端创建对应 purpose 的 `FileObject`。
5. AI 增强 / OCR 处理，结果进入短期预览。
6. 用户选择打印、下载、保存到我的文档或删除。
7. 高敏扫描件按 TTL 自动物理删除并写审计。

### 材料包

1. 用户从简历、我的文档、收藏岗位、招聘会资料选择材料。
2. AI 给出材料组合建议，但用户必须逐项确认。
3. 后端创建 `MaterialPackTask` 主任务和多个子 `PrintTask`。
4. Agent 顺序执行子任务；每份材料有独立状态和重试。
5. 完成页展示面数、费用、失败项、再打印和反馈。

## 8. Admin 后台

### 配置管理

- 终端档案：终端编号、位置、机构、启停、Agent 版本、IP、MAC、心跳。
- 设备能力：打印机名称、纸张、黑白 / 彩色、双面、扫描、U 盘、摄像头。
- 文件策略：各 purpose 的大小、TTL、是否允许长期保存、是否高敏。
- 价格策略：打印、扫描、复印、证件照、材料包、权益券和补贴。

### 任务管理

- 打印 / 扫描 / 复印 / 证件照 / 材料包统一任务列表。
- 支持取消、重试、释放卡住任务、标记人工确认、重新派发到指定终端。
- 对 `PRINT_JOB_UNCONFIRMED` 建立人工处理流程。

### 告警与审计

- 独立 `Alert` 模型记录终端离线、SQLite degraded、缺纸、卡纸、墨粉、磁盘、文件清理失败。
- Admin 访问用户文件、生成签名 URL、下载、删除、查看高敏文件必须写不可删除审计。
- 统计任务数、完成率、失败率、平均等待、页数、黑白 / 彩色占比、文件清理成功率。

## 9. 首期全功能路线

首期全功能按模块拆分实施，但目标都是首期交付。以下批次只表示依赖关系和验收顺序，不表示延期到二期。

### A. 安全底座与入口

- 打印任务创建时绑定目标 `terminalId`，Agent claim 按 `terminalId` 过滤。
- Agent SQLite fail-closed，不允许无幂等继续打印。
- `/print-scan` 成为首期全功能总入口，所有能力由 FeatureGate / DeviceCapability / Admin 配置共同控制。
- 首页打印扫描组标题进入 `/print-scan` 服务中心。
- 打印进度补任务编号、目标终端、状态解释、失败反馈。

### B. 基础打印与用户文件

- 手机扫码上传和安全取件码。
- PrintTask 任务级 TTL、未取件 expired、过期清理。
- 我的文档和打印订单在 `/print-scan` 显式承接。
- Admin 任务干预、终端 degraded、告警、价格策略和统计看板。
- 文档打印、图片打印、简历打印、求职材料打印和招聘会资料打印统一进入真实 `PrintTask`。

### C. 材料包与 AI 打印助手

- 材料包：手动选择材料、AI 建议组合、逐项参数、顺序打印、子任务失败重试。
- AI 文件体检：尺寸、方向、空白页、清晰度、页数、敏感信息提示。
- AI 故障解释：把 Agent 错误码转换成用户动作和 Admin 运维建议。

### D. 扫描、复印、证件照和外设

- 真实扫描：TWAIN / WIA、ADF、PDF / JPG 输出、AI 增强、OCR、短 TTL。
- 证件复印：身份证正反面 A4 合成，默认不长期保存。
- 证件照：上传照片、抠图、换底、规格检测、排版打印。
- U 盘：Agent 只读枚举、扩展名 / MIME / 大小校验、拔出清理。
- 云上传：会员或一次性会话绑定当前终端，不能公网远程直控打印机。
- 格式转换 / 签名盖章：完成真实文件生成、免责声明和验收后开放。

### E. 商业化与后台运营

- 订单金额、价格策略、权益券、免费额度、补贴核销和退款 / 异常处理。
- Admin 统一任务中心：打印、扫描、复印、证件照、材料包。
- Admin 设备中心：终端、打印机、扫描仪、摄像头、U 盘、Agent 版本和耗材状态。
- Admin 文件生命周期和审计中心：高敏文件访问、下载、删除、TTL、清理失败告警。

## 10. 首期开放原则

首期必须开发完整功能，但上线开放必须遵守以下原则：

- 未通过真机验收的能力不能对普通用户展示为“可立即使用”。
- 未完成物理删除和审计验收的高敏能力不能采集身份证、证件照或扫描件。
- 未完成真实文件生成的格式转换、隐私遮挡、签名盖章不能进入打印确认。
- 未完成当前终端绑定的云上传和 U 盘打印不能创建正式打印任务。
- 未完成正式域名 HTTPS、真实短信上线 E2E、Windows 真机出纸和扫描真机验收时，不能对外宣称正式生产或试运营完成。

## 11. 验收标准

- 首期各模块代码实现后必须通过 API / Kiosk / Terminal Agent 最小相关 typecheck、verify 和 `git diff --check`。
- 多终端验收：A 终端提交的任务只能被 A 终端 claim，B 终端不可领取。
- SQLite 验收：禁用或破坏本地 SQLite 后 Agent 不打印，并在 Admin 告警。
- 能力开关验收：生产 HTTP 模式下未通过验收的扫描、复印、证件照、U 盘、云上传、格式转换、签名盖章不会对普通用户开放正式任务。
- 文件验收：高敏 purpose 在 TTL 后完成物理删除和审计记录。
- 真机验收：Windows Terminal Agent、奔图打印机、真实出纸、断网 / 重启恢复、错误回传通过。
- 合规验收：岗位 / 招聘会只保留“去来源平台投递 / 扫码投递 / 去来源平台预约 / 扫码预约”，不出现平台内投递闭环。

## 12. 审查结论

本方案已经过竞品调研、前台体验审计、后端 / Terminal Agent 审计、产品合规审计，以及 Claude + Antigravity 交叉审查。

共识结论：

- 首期开发范围覆盖完整打印扫描服务集，不把有价值能力延期为二期。
- 多终端错打、SQLite 无幂等和虚假扫描可用性是上线前阻断项。
- 扫描、复印、证件照、U 盘、云上传、格式转换、签名盖章必须在首期做完整，但由开关和验收门禁控制对外开放。
- AI 应增强用户自用材料准备和打印体验，不得进入招聘闭环或企业候选人处理。
