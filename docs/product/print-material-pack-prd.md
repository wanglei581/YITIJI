## 1. 问题陈述

### 1.1 用户问题

在招聘会场景下，求职者通常需要同时携带多份材料：个人简历（多份）、目标岗位列表、招聘会参展企业手册等。当前一体机的打印流程是**单文件单次排队**——用户每打印一个文件，都需要走完「选择文件 → 预览 → 设置参数 → 确认 → 等待完成」的完整链路。打印 3 份不同材料意味着重复 3 次完整流程，总操作步骤可达 15 步以上。

这在 21.5 寸触控一体机的公共场所场景下尤为痛苦：排队等候、触屏操作疲劳、每轮等待不确定的打印时间，导致大量用户在完成第 1-2 份打印后放弃后续材料。

### 1.2 影响量化

| 维度 | 现状 | 成本 |
|------|------|------|
| 单次招聘会材料打印 | 3-4 次独立打印流程 | 每次约 2-3 分钟操作 + 1-2 分钟等出纸 |
| 用户放弃率（假设） | 预估第 2 份后 40%+ 用户放弃 | 失去后续材料打印收入 |
| 招聘会高峰排队 | 每人 10+ 分钟占用终端 | 高峰期终端吞吐量低 |
| 业务收入 | 单次打印 1 份材料 | 每用户平均打印面数低 |

### 1.3 不解决的代价

- 用户体验差 → 复购/复用意愿低 → 终端日活不达预期
- 招聘会高峰吞吐不足 → 运营方投诉 → 合作机构满意度下降
- 无法形成"招聘会场景 = 一体机标配"的认知 → 竞品替代风险

## 2. 目标

### 2.1 用户目标

1. **减少操作步骤**：招聘会场景下，3 份材料的打印操作从 15+ 步降至 5 步以内（选材料包 → 确认内容 → 确认参数 → 确认打印 → 完成）
2. **减少等待焦虑**：用户提交一次即可等待全部材料依次出纸，无需反复回到终端前操作
3. **不遗漏材料**：系统推荐"招聘会标配包"，避免用户现场才发现少打了某份材料

### 2.2 业务目标

1. **提升单用户打印面数**：从平均 5 面/次提升至 12 面/次（+140%）
2. **提升招聘会场景终端吞吐**：高峰期单终端服务人次从 5 人/小时提升至 8 人/小时（+60%）
3. **提升材料打印完成率**：从预估 60% 提升至 85%+（减少中途放弃）

## 3. 非目标（Non-Goals）

| 非目标 | 原因 |
|--------|------|
| **不替用户投递简历** | 合规红线：平台不构成招聘闭环，打印材料包只负责出纸，不把文件发送给任何企业或第三方 |
| **不做"一键打印材料包"** | 合规文案规范：禁止"一键"暗示，按钮文案为"确认打印"或"按以上设置打印" |
| **不做自动选材** | v1 不做 AI 智能推荐材料组合，用户手动选择材料包内容；AI 推荐作为 P2 未来考虑 |
| **不做跨终端编排** | 材料包仅在当前终端打印，不支持"终端 A 打简历、终端 B 打岗位列表"的分布式编排 |
| **不做预约打印** | v1 不支持"提前选好材料包、到场扫码即打"，仅支持现场即时打印 |
| **不做付费套餐绑定** | 材料包本身不绑定套餐/优惠逻辑，计费仍按实际打印面数 × 单价；套餐优惠作为 C-4 阶段另行设计 |

## 4. 用户故事

### P0 — 核心流程

1. **作为** 正在准备招聘会的求职者，**我希望** 选择一个"招聘会标配包"（简历 × 5 + 岗位列表 + 招聘会手册），**以便** 一次确认就打印全部参会材料，不用反复操作。

2. **作为** 求职者，**我希望** 在材料包确认页预览每份材料的缩略图和参数（份数/色彩/单双面），**以便** 确保每份材料都按我想要的方式打印。

3. **作为** 求职者，**我希望** 各份材料可以设置不同的打印参数（如简历彩色双面、岗位列表黑白单面），**以便** 针对不同材料做最优打印选择。

4. **作为** 求职者，**我希望** 提交材料包后看到统一进度追踪（"第 1/3 份打印中…"），**以便** 知道整体进展而不必反复查看。

5. **作为** 求职者，**我希望** 如果某份材料打印失败，可以单独重试该份而不必重新提交整个材料包，**以便** 节省时间和纸张。

### P1 — 增强体验

6. **作为** 求职者，**我希望** 也可以自由组合材料（从我的简历/文档/岗位收藏中勾选），**以便** 不局限于预设模板。

7. **作为** 招聘会参与者，**我希望** 从招聘会详情页直接进入"打印此招聘会材料包"，**以便** 减少导航步骤。

8. **作为** 运营人员（Admin），**我希望** 在后台配置预设材料包模板（名称、包含材料类型、推荐参数），**以便** 不同招聘会/场景可以提供不同的标配包。

9. **作为** 求职者，**我希望** 打印完成后看到费用汇总明细（各材料面数 × 单价），**以便** 清楚知道每份材料花了多少钱。

### P2 — 未来考虑

10. **作为** 求职者，**我希望** 系统根据我的简历和意向岗位智能推荐应打印的材料组合，**以便** 不遗漏重要材料。

11. **作为** 合作机构（Partner），**我希望** 为本机构主办的招聘会配置专属材料包模板，**以便** 提升参会者体验。

## 5. 需求

### 5.1 Must-Have（P0）

#### 5.1.1 材料包入口

- **位置**: 打印扫描服务首页（`/print-scan`）新增"打印材料包"能力入口卡片
- **入口条件**: 用户已登录；至少有 1 份可打印材料（已上传的简历/文档、或收藏的岗位/招聘会可导出为列表）
- **空态处理**: 用户无可打印材料时，卡片灰色不可点击，提示"先上传简历或收藏岗位后再来"
- **合规**: 卡片文案"打印材料包"，禁止"一键打印材料包"

**验收标准**:
- [ ] `/print-scan` 首页显示"打印材料包"入口卡片
- [ ] 未登录用户点击后跳转登录页
- [ ] 已登录但无可打印材料的用户看到灰色卡片 + 提示文案
- [ ] 卡片文案不含"一键"

#### 5.1.2 预设材料包模板

- **默认模板**: "招聘会标配包"
  - 简历（默认 5 份，黑白双面）
  - 目标岗位列表（默认 1 份，黑白单面）
  - 招聘会参展企业手册（默认 1 份，黑白单面，如有）
- **模板来源**: 前端硬编码 v1，后续 Admin 可配置
- **材料识别**:
  - "简历" → 用户最近一次 AI 解析/优化后的简历 PDF
  - "目标岗位列表" → 用户收藏的岗位导出为 PDF 列表
  - "招聘会手册" → 当前浏览招聘会的参展企业信息页

**验收标准**:
- [ ] 进入材料包页后默认展示"招聘会标配包"
- [ ] 各材料行显示材料名、来源说明、缩略图（如有）、当前参数
- [ ] 无某类材料时该行灰色 + "暂无"标注 + "去添加"链接

#### 5.1.3 材料选择与参数配置

- **材料增删**: 用户可从材料包中移除不需要的材料，可从"我的简历"/"我的文档"/"收藏"中添加材料
- **参数独立配置**: 每份材料可独立设置份数、色彩模式、单双面（复用现有 `PrintJobParams` 结构）
- **批量参数**: 支持一键"全部黑白"/"全部双面"快速设置
- **预览**: 每份材料点击可查看缩略图预览

**验收标准**:
- [ ] 每份材料有独立参数设置行
- [ ] 可移除材料包中的材料
- [ ] 可从"我的简历"/"我的文档"/"收藏"添加材料
- [ ] "全部黑白"/"全部双面"快捷按钮可用
- [ ] 至少保留 1 份材料才允许继续

#### 5.1.4 确认与提交

- **确认页**: 展示材料包完整清单（材料名 × 份数 × 参数）、总面数、预计用纸、预计费用
- **隐私检查**: 对每份材料执行已有的隐私检查流程（复用 `PrintMaterialCheckPage` 逻辑）
- **费用明细**: 各材料面数 × 单价 = 小计；合计费用
- **提交**: 一次提交生成 1 个材料包任务（内含 N 个子任务），后端按顺序执行
- **合规按钮文案**: "按以上设置打印"（与现有打印确认页一致）

**验收标准**:
- [ ] 确认页展示所有材料的参数摘要
- [ ] 显示总面数、预计用纸、预计费用
- [ ] 隐私检查对每份材料执行并展示结果
- [ ] 按钮文案为"按以上设置打印"，不含"一键"
- [ ] 提交成功后跳转进度页

#### 5.1.5 统一进度追踪

- **进度展示**: 显示材料包整体进度（"第 1/3 份打印中…"）
- **子任务状态**: 列出每份材料的独立状态（等待中 / 打印中 / 已完成 / 失败）
- **失败处理**: 某份材料失败时，标记失败并允许单独重试；其余材料继续执行
- **完成确认**: 全部完成后显示成功页 + 费用汇总

**验收标准**:
- [ ] 进度页显示"第 N/M 份"整体进度
- [ ] 各材料行显示独立状态标签
- [ ] 失败材料行显示"重试"按钮
- [ ] 重试仅重新提交该份材料，不影响已完成材料
- [ ] 全部完成显示汇总页

## 6. 成功指标

### 6.1 北极星指标

**材料包打印完成率**: 用户进入材料包页面后最终完成全部材料打印的比例

- **基线**: N/A（新功能）
- **目标**: 30 天内 ≥ 70%
- **测量方法**: `材料包完成打印数 / 材料包创建数`，按周统计

### 6.2 驱动指标（Leading Indicators）

| 指标 | 定义 | 目标 | 测量方法 |
|------|------|------|---------|
| 材料包使用率 | 使用材料包打印的用户占所有打印用户比例 | 30 天内 ≥ 25% | `材料包打印用户数 / 总打印用户数` |
| 单材料包平均材料数 | 每个材料包包含的材料份数 | ≥ 2.5 份 | `材料包中材料总数 / 材料包数` |
| 单材料包平均打印面数 | 每个材料包的总打印面数 | ≥ 10 面 | `材料包总面数 / 材料包数` |
| 材料包 vs 单文件操作时间差 | 材料包 3 份材料 vs 3 次单文件打印的操作时间 | 减少 ≥ 50% | 埋点计时（从进入打印页到提交） |
| 子任务失败率 | 单份材料打印失败的比例 | ≤ 5% | `失败子任务数 / 子任务总数` |

### 6.3 健康指标（Guardrail Metrics）

| 指标 | 定义 | 红线 | 测量方法 |
|------|------|------|---------|
| 单文件打印使用率 | 不用材料包、仍走单文件流程的用户比例 | 不应大幅下降（>15pp） | `单文件打印用户数 / 总打印用户数` |
| 打印投诉率 | 与打印相关的用户投诉比例 | ≤ 现有水平 | 客服记录 / 日志分析 |
| 终端占用时长 | 招聘会高峰单用户占用终端总时长 | 不应增加（材料包应减少操作时间） | 埋点计时 |
| 材料包退款率 | 因材料包打印问题导致的退款比例 | ≤ 3% | `材料包退款数 / 材料包订单数` |

## 7. 技术方案要点

> 本节为工程参考，非最终实现。具体以开发阶段的技术设计文档为准。

### 7.1 后端

| 要点 | 说明 |
|------|------|
| 新增数据模型 | `PrintPack`（材料包主表）+ `PrintPackItem`（子材料表），1:N 关系 |
| API 设计 | `POST /api/v1/print/packs` — 创建材料包并按序生成子任务；`GET /api/v1/print/packs/:id` — 轮询包状态及子任务状态 |
| 任务编排 | 后端按 `PrintPackItem.sortOrder` 顺序依次下发子任务给 Terminal Agent；前一份完成后自动下发下一份 |
| 幂等保障 | 同一材料包不可重复提交；子任务重试带 `idempotencyKey` |
| 文件生成 | "岗位列表导出"和"招聘会手册导出"需后端新增 PDF 生成服务（现有打印链路仅支持已上传文件） |

### 7.2 前端（Kiosk）

| 要点 | 说明 |
|------|------|
| 新增路由 | `/print/pack` — 材料包选择页；`/print/pack/confirm` — 确认页；`/print/pack/progress` — 进度页 |
| 复用组件 | 参数设置行复用 `PrintPreviewPage` 逻辑；隐私检查复用 `PrintMaterialCheckPage`；进度展示复用 `PrintProgressPage` 轮询逻辑 |
| 状态管理 | 材料包页面状态（选中材料 + 参数）用 `localStorage` 持久化，防止触屏超时丢失 |
| 错误兜底 | 任何子任务失败展示具体材料名 + 错误原因 + 重试按钮；全部超时则提示"请联系现场工作人员" |

### 7.3 Terminal Agent

| 要点 | 说明 |
|------|------|
| 顺序执行 | Agent 收到材料包子任务时，按顺序执行；当前任务未完成时不接收新任务 |
| 状态上报 | 每个子任务独立上报 `pending → claimed → printing → completed/failed` |
| 间隔控制 | 两个子任务之间增加 2 秒间隔，防止打印机缓冲区溢出 |

## 8. 风险与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| 打印机缓冲区溢出 | 中 | 高——连续下发多份任务可能溢出 | 后端顺序下发 + Agent 间隔 2s + 前一份完成后下发下一份 |
| 部分材料生成失败（岗位列表 PDF 导出等） | 中 | 中——用户可能无法获得完整材料包 | 生成失败的材料标记为"不可用"，允许用户跳过或重试生成；不影响其他材料 |
| 触屏超时中断操作 | 高 | 高——公共场所用户可能中途离开 | 关键步骤状态持久化到 localStorage；超时后回到材料包页可恢复 |
| 合规风险——材料包被误解为"平台替用户投递" | 低 | 高——合规红线 | 所有文案严格使用"打印"而非"投递"或"发送"；按钮禁止"一键"；功能说明明确"仅负责出纸" |
| 单份材料失败导致整体卡住 | 低 | 高——用户等待时间过长 | 子任务独立状态管理；失败后自动跳过继续下一份；支持单独重试 |
| 打印队列积压（招聘会高峰多人同时提交材料包） | 中 | 中——等待时间过长 | 后端队列优先级管理；材料包子任务不插队；高峰期可考虑限制材料包最大材料数 |

}
