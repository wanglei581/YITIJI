# 页面功能深度提升方案（2026-07）

> 生成日期：2026-07-02　输入：三端页面逐页只读深度审计（apps/kiosk 28 个主页面 JSX 区块级审阅 + Admin/Partner 现状复核）
> 背景：用户判断"很多功能页面不完整、比较单一"——审计证实：整体内容丰富度约 6.5/10，核心链路完整、合规守得好，但普遍存在"一层列表、缺深度、缺对比、缺下一步引导"。
> 上位文档：[commercial-grade-feature-plan-2026-07.md](./commercial-grade-feature-plan-2026-07.md)（本方案的条目并入其 Phase 2 排期）
> **执行前提（重要）**：2026-06-15 用户定版 UI/UX 冻结（只做真实化/bugfix，禁改布局与入口结构）。本方案将增强项分为两档：
> - **A 档**：在现有页面骨架内补数据字段、补操作按钮、补空态/引导文案——属"真实化+补齐"范畴，不触冻结；
> - **B 档**：需新增内容区块或改变页面信息架构——**逐项需用户解冻确认后才可执行**。

## 二、合规与诚实性过滤（审计建议中**明确不做**的项）

以下审计建议**违反红线或无真实数据支撑，排除出本方案**，防止后续误拾：

| 被排除的建议 | 排除原因 |
|---|---|
| 岗位"你已投递/投递进度跟踪"、"该岗位已有 X 人申请" | 红线：系统不记录、不展示投递结果与第三方后续处理（compliance §4.4） |
| "该建议已被 X 人采纳，平均提升 20%" 类社会证明 | 无真实数据=伪造；且接近效果承诺 |
| 岗位匹配"百分比分数/与前 3 名候选人对比" | 既定合规口径：只给三档参考（较高/中等/偏低），无百分比、不对比他人、不回传企业 |
| 企业评分/用户评价 UGC | 引入 UGC 审核与名誉风险，且贴近招聘平台行为；用"真实浏览量"替代 |
| "行业薪资中位数对标" 直接上线 | 无权威数据源即为编造；仅允许 B 档做**本库同来源岗位**的真实聚合，并标注口径 |
| 面试"录用结果反馈闭环" | 红线：不记录录用结果 |
| 分享简历报告给招聘官 | 贴近"简历回流企业"红线，不做 |

## 六、执行规则与排期并入

1. **A 档不需要解冻**，按"入口真实化/状态补齐"口径直接排入 Phase 1/2 对应分支；**B 档汇总后逐项请用户解冻**（建议一次性过一遍勾选）。
2. 沿用工程铁律：一任务一分支；RenshiPage(944)、PrintPreviewPage(707)、InterviewSessionPage(677)、HomePage(739) 四个超限文件**增强前先零行为拆分**；每个增强域补/扩 verify 脚本。
3. 优先顺序建议（价值/成本比）：
   - 第一批（随 Phase 1 支付版）：打印订单详单+再打一份+凭证、价格详单、权益真实库存；
   - 第二批（Phase 2A 开头，全 A 档纯前端居多）：AI 记录摘要、报告历史对比、面试选已有简历+趋势、列表排序、下一步引导全站补齐、助手动态快捷问题；
   - 第三批（B 档解冻后）：本人画像推荐、逛展攻略、深挖单维度、逐条采纳、政策 AI 指引单。
4. 所有图表/计数/推荐**只用真实数据**，无数据出诚实空态；口径文案上屏（"本终端网络浏览次数"等），延续项目"不伪造"传统。
