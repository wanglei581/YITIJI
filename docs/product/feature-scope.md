## 二、一体机前台功能清单

### 2.1 首页

> **导航结构约束（固定，不可扩展）：**  
> 底部 Tab 固定为三项：**首页 / AI助手 / 我的**，不增加第四、五个 Tab。  
> 所有业务入口（AI简历服务、打印扫描、岗位信息、招聘会服务等）均从首页卡片进入，进入后再展示子功能，不在底部导航增加入口。

> **入口稳定与数据闭环约束（2026-06-12 用户确认）：**  
> 当前首页与各业务板块里的功能入口已经定版，后续开发不再新增重复入口 / 同义卡片 / 额外菜单层。开发方式改为：把已有入口真实化，把已有页面数据接真，把已有按钮接线，把已有流程补齐状态，并把结果沉淀到「我的」对应资产分组。  
> 每个入口必须形成闭环：**首页/板块入口 → 业务操作 → 产生用户数据 → 进入「我的」对应资产/记录 → 可查看、继续使用、打印、删除或复用**。  
> 详细矩阵见：[user-data-flow-matrix.md](./user-data-flow-matrix.md)。

| 功能模块 | 说明 | 优先级 |
|---------|------|--------|

## 六、后续特色功能规划

> **当前不开发，记录在此供后续阶段参考。**  
> 所有特色功能必须在 Phase 8.1B（Agent Claim/Heartbeat）完成后才启动。  
> 合规边界：所有功能均不得新增一键投递、候选人管理、企业查看简历、面试邀约、Offer 管理等招聘闭环功能。

| 功能 | 优先级 | 前置依赖 | 说明 |
|------|--------|---------|------|
| **待机宣传屏（广告位）** | P1 | 已交付一期（`feature/kiosk-screensaver-ads`） | 一体机无操作达阈值进入全屏宣传屏（图/视频轮播），触摸唤醒回首页；忙碌态（打印/扫描/AI/上传）不打断。管理员后台「宣传屏」管理素材与播放方案。**二期**：AI 文生图海报（接通义万相/CogView + 内容安全，草稿确认入库）、曝光/唤醒埋点、机构端上传+审核流 |
| **打印材料包** | P1 | Phase 8.1B + Phase 7 AI | 将简历、岗位列表、招聘会手册等合并为一个打印任务，一次排队完成；文案使用"打印材料包"，**禁止使用"一键打印材料包"** |
| **求职打印套餐** | P1 | Phase 8.1B | 预设常用打印组合（如"简历 × 5 份 + A4 封面"），一次选择完成参数配置，减少重复操作 |
| **招聘会现场模式增强** | P2 | Phase 8.1B + 现有 `/job-fairs/:id/*` 页面 | 基于现有 `/job-fairs/:id/*` 子路由（companies / map / materials / stats）扩展，增加签到引导、展区推荐、现场服务快捷入口 |
| **面试练习轻量版** | P2 | Phase 7 AI（非数字人） | AI 出题 + 用户文字/语音作答 + 结构化反馈报告 + 报告可打印；不依赖数字人/摄像头；训练结果只给求职者本人，不推送给企业 |
| **AI求职路线规划** | P3 | 用户画像 + 推荐规则 | 根据简历分析结果、意向岗位、当前岗位列表，为求职者生成个性化求职路线建议（申请哪类岗位、重点修改简历哪些部分、推荐参加哪场招聘会）；仅提供参考，不做平台内投递 |

### 合规要求（特色功能均适用）

- 打印材料包、打印套餐：只负责打印，不替求职者投递简历，不把文件发送给企业
- 面试练习：报告保存在用户账号下，用户主动打印才输出；不新增企业查看、候选人评级功能
- AI求职路线：所有跳转外部平台均使用"去来源平台投递/扫码投递"，不新增平台内一键投递
