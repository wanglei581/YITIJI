# 合作机构权限矩阵

> 版本：Phase 7  
> 更新日期：2026-05-25  
> 说明：定义各 `partnerType` 在系统各功能模块中的权限边界。权限为后台 RBAC 实现依据，不影响前台展示逻辑。

---

## 合作机构类型定义

| 类型标识 | 中文名称 | 典型场景 |
|----------|---------|---------|
| `school_employment_center` | 高校就业中心 | 高校就业处管理校招信息和招聘会 |
| `public_employment_service` | 公共就业服务机构 | 人社局、人才市场等政府部门 |
| `licensed_hr_agency` | 持证人力资源服务机构 | 持人力资源服务许可证的 HR 公司 |
| `fair_organizer` | 招聘会主办方 | 专业招聘会策展公司或高校 |
| `enterprise_source` | 企业来源方 | 直接提供自身岗位数据的企业（不含招聘闭环） |

## `public_employment_service` 特殊权限

公共就业服务机构（如市人社局、人才市场）具有以下额外权限：

1. **跨区域查看**：`serviceLevel = 'municipal'` 时可查看全辖区所有终端服务数据
2. **官方数据源**：可标记数据来源为 `official`，在一体机端展示"官方来源"标识
3. **政策公告发布**：可向辖区内所有终端推送政策公告
4. **招聘会审核协助**：可提交招聘会数据后由管理员一键审核（优先处理通道）
5. **展位签到管理**：可管理招聘会现场企业签到状态

## 权限审批流程

```
合作机构申请功能权限
         ↓
提交材料（营业执照 / 许可证 / 政府公文）
         ↓
管理员人工审核
         ↓
审核通过 → 更新 sceneConfig.enabledModules
         ↓
合作机构后台生效
```

## 合规承诺

所有合作机构类型，无论功能权限多寡，均受以下约束：

> 1. 不得通过本系统平台内投递简历
> 2. 不得接收求职者简历数据
> 3. 不得进行候选人筛选、面试邀约、Offer 管理
> 4. 所有岗位信息仅限展示，跳转通过来源平台二维码实现
> 5. 违反以上约束的账号将被立即停用并保留操作日志

此约束在系统层面通过 `PROHIBITED_MODULES` 硬编码，不可通过任何配置绕过。
