## 一、用途说明

Windows Terminal Agent 是运行在一体机 Windows 系统上的**本地后台服务**，负责前端页面无法直接访问的硬件资源交互：

| 职责 | 说明 |
|------|------|
| 打印机通信 | 调用奔图 CM2820ADN Windows 驱动，执行打印任务 |
| 扫描仪通信 | 调用 TWAIN/WIA 驱动或监听 SMB/FTP 扫描目录 |
| U盘监听 | 检测 U 盘插入，读取文件供用户选择上传 |
| 扫码器监听 | 接收 USB HID 扫码器输入 |
| 心跳上报 | 定期向后端 API 上报终端在线状态 |
| 告警上报 | 检测到硬件故障时向后端 API 上报告警 |

---

## 二、运行环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 x64 / Windows 11 x64 |
| 架构 | x64（一体机标准配置） |
| 网络 | 有线网络（与打印机同一局域网，或有外网访问后端 API） |
| 权限 | 需要管理员权限（用于访问打印机驱动、监听 USB 设备） |
| 开机自启动 | 通过 Windows 服务或任务计划程序 |
| 依赖 macOS | **否**，完全独立于 macOS 环境 |

## 三、开发语言选项（待定）

| 选项 | 优点 | 缺点 |
|------|------|------|
| Node.js | 与前端同生态，团队熟悉 | 打包体积较大，原生硬件访问需 native addon |
| .NET (C#) | Windows 原生支持最佳，打印机/USB 访问能力强 | 需要额外学习成本 |
| Python | 库丰富（pywin32、pyusb），开发快 | 运行时依赖较多 |

> **当前倾向**：Node.js（通过 `node-printer`、`usb`、`node-hid` 等库），与项目主体保持同一生态。  
> 若 Windows 原生打印机访问出现兼容性问题，改用 .NET 封装后通过 HTTP 本地 API 暴露给 Node.js 调用。

## 四、计划中的功能模块

### 4.1 打印任务模块

- 启动时注册为 Windows 服务，开机自动运行
- 定期轮询（或 WebSocket 订阅）后端 API 获取待执行打印任务
- 下载打印文件到本地临时目录（使用临时签名 URL）
- 调用 Windows 打印 API 发送到奔图打印机
- 打印完成后回调后端 API 更新任务状态
- 临时文件打印完成后立即删除

### 4.2 扫描模块

- **方案 A**（优先）：调用 TWAIN/WIA 扫描驱动，触发扫描，获取 PDF
- **方案 B**（备用）：监听打印机扫描到 SMB 共享目录，检测新文件后上传
- 扫描完成后将 PDF 上传到后端存储，通知前端页面

### 4.3 U盘监听模块

- 监听 USB 存储设备插入/拔出事件
- 设备插入后读取根目录文件列表（支持格式：PDF、DOCX、JPG、PNG）
- 将可打印文件列表通知前端页面（通过本地 HTTP API 或 WebSocket）

### 4.4 心跳与状态上报模块

- 每 30 秒向后端 API 上报终端心跳
- 上报内容：终端 ID、在线状态、打印机状态、IP 地址、时间戳
- 打印机状态包括：在线/离线、纸张余量、墨粉余量、故障代码

### 4.5 告警上报模块

- 检测到以下情况时上报告警：
  - 打印机离线
  - 缺纸
  - 墨粉不足（低于 20%）
  - 卡纸/硬件故障
  - 磁盘空间不足
  - 网络连接中断超过 5 分钟

## 五、与后端 API 的交互方式

```
Terminal Agent (Windows)
       ↕ HTTP / WebSocket
后端 API (services/api)
       ↕
  数据库 / 对象存储
```

| 接口类型 | 用途 |
|---------|------|
| HTTP 轮询 | 拉取待打印任务 |
| WebSocket（可选） | 实时推送任务，减少轮询延迟 |
| HTTP POST | 上报心跳、告警、任务完成状态 |
| 文件下载 | 通过临时签名 URL 下载打印文件 |
| 文件上传 | 上传扫描结果（multipart 或预签名 PUT） |

## 六、本地 HTTP API（供前端页面调用）

Terminal Agent 在本地监听一个 HTTP 端口（如 `http://localhost:9527`），供一体机前台页面调用：

| 接口 | 说明 |
|------|------|
| `GET /status` | 获取终端和打印机状态 |
| `GET /usb/files` | 获取 U 盘文件列表 |
| `POST /print` | 触发本地打印任务 |
| `POST /scan` | 触发扫描 |
| `GET /scan/result` | 获取最新扫描结果 |

> **安全注意**：本地 API 仅监听 `localhost`，不对外暴露。前端页面通过 `http://localhost:9527` 访问，不经过互联网。

## 八、开发注意事项

1. **不依赖 macOS**：Agent 代码不能使用任何 macOS 专有 API、路径格式或命令。
2. **路径处理**：全部使用 `path.join()` / `path.resolve()`，不硬编码路径分隔符。
3. **临时文件清理**：打印完成后必须删除临时文件，不保留用户文件。
4. **错误重试**：网络请求失败需有重试机制，不能因网络抖动停止工作。
5. **日志**：所有操作必须写日志，包括打印任务执行记录、告警上报记录。
