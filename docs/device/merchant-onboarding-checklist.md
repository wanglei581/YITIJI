# 商户开通材料与配置清单（微信 Native + 支付宝当面付，W-D ②③）

> **草案，未经真机/真实商户验证。** 2026-07-06 起草。
> 商户平台的页面路径/入口名随渠道改版可能变化，以渠道官方文档为准；本清单以「产出物 → 对应 env 变量名」为锚点，页面找不到时按产出物名搜索渠道帮助中心。
> 密钥口径：所有产出物中的密钥/证书**只进生产服务器 env 或服务器本地文件**（见 env 清单草案 §4），不进仓库/聊天/截图。
> 代码事实：微信实现为 **APIv3 公钥模式**（新商户默认；老商户平台证书模式暂不支持，见待确认）；支付宝实现为 **RSA2 公钥模式**（非证书模式）。

## 一、微信支付（Native 动态码 + APIv3 公钥模式）

### 1. 商户开户材料（提交前备齐）

| 材料 | 说明 |
|---|---|
| 营业执照 | 彩色扫描件/照片，主体名称与后续结算账户一致 |
| 法定代表人身份证 | 正反面；若经办人非法人另需授权函〔待确认：当前微信商户开户是否要求授权函〕 |
| 结算银行账户 | 对公账户（开户许可证/基本存款账户信息）；个体工商户可用法人卡〔待确认〕 |
| 经营场所/业务说明 | 线下自助打印终端业务描述、终端照片可能被要求补充〔待确认〕 |
| 联系人手机号 + 邮箱 | 接收商户号开通与验证信息 |

### 2. 开通与配置步骤（每步产出物 → env 变量）

| # | 步骤 | 操作位置 | 产出物 → env 变量 |
|---|---|---|---|
| 1 | 提交商户开户申请（选择「Native 支付」产品；如后续做 W-E 付款码收款，同时申请「付款码支付」） | 微信支付商户平台 pay.weixin.qq.com | 商户号 → `WECHAT_PAY_MCHID` |
| 2 | 准备并绑定 appid：Native 下单接口要求商户号关联一个 appid（服务号/小程序/APP 均可）；若公司尚无任何微信应用主体，需先注册一个（周期最长的一步，尽早启动）〔待确认：用哪个主体的 appid〕 | 商户平台 → 产品中心/账户中心 → APPID 账号管理 | 已绑定 appid → `WECHAT_PAY_APPID` |
| 3 | 设置 APIv3 密钥（自设 32 字符；**设置动作在商户平台完成，值当场抄录进服务器 env，别处不留存**） | 商户平台 → 账户中心 → API 安全 → 设置 APIv3 密钥 | 32 字符密钥 → `WECHAT_PAY_APIV3_KEY`（代码校验恰好 32 字节） |
| 4 | 申请商户 API 证书（用官方证书工具生成；私钥文件 `apiclient_key.pem` 只落生产服务器） | 商户平台 → API 安全 → 申请 API 证书 | 私钥 → `WECHAT_PAY_PRIVATE_KEY_PATH`（或 `_PEM` 内联）；证书序列号 → `WECHAT_PAY_MCH_SERIAL_NO` |
| 5 | 获取**微信支付公钥**及公钥 ID（公钥模式回调验签用；新商户默认公钥模式。若页面只有「平台证书」没有「微信支付公钥」，说明商户号是老证书模式——当前代码不支持，须停下走待确认项） | 商户平台 → API 安全 →（微信支付公钥区域）下载 | 公钥 PEM → `WECHAT_PAY_PUBLIC_KEY_PATH`（或 `_PEM`）；公钥 ID（`PUB_KEY_ID_…`）→ `WECHAT_PAY_PUBLIC_KEY_ID` |
| 6 | 回调地址确认：Native 的 notify_url 由每笔下单请求携带（本项目为 `{PAYMENT_NOTIFY_BASE_URL}/api/v1/payment/callback/wechat`），要求 https 公网可达；商户平台侧是否还需配置回调域名/白名单〔待确认：现行 Native 是否有域名配置项，JSAPI 的「支付授权目录」不适用本项目〕 | — | 域名 → `PAYMENT_NOTIFY_BASE_URL`（见 nginx 草案） |
| 7 | 费率与结算周期确认（Native 线下场景常见 0.6%，实际以签约单为准）〔待确认〕；同时确认**退款权限**默认开通（W-B 真实渠道退款依赖） | 商户平台 → 产品中心/签约信息 | 无 env；记录进部署清单 |

### 3. 微信侧自查

- [ ] 商户号状态「已开通」，Native 产品「已开通」
- [ ] appid 与商户号绑定关系确认（下单报 `APPID_MCHID_NOT_MATCH` 即此项没做对）
- [ ] API 安全页可见：APIv3 密钥已设置、API 证书序列号、微信支付公钥 ID —— 三者与服务器 env 一致
- [ ] 确认商户号为「公钥模式」而非「平台证书模式」

## 二、支付宝（当面付 precreate 动态码 + RSA2）

### 1. 签约材料

| 材料 | 说明 |
|---|---|
| 企业支付宝账户 | 需以营业执照完成企业认证 |
| 营业执照 | 与支付宝账户主体一致 |
| 经营内容说明 | 线下自助终端收款场景；当面付签约可能要求门店/设备照片〔待确认〕 |

### 2. 开通与配置步骤（每步产出物 → env 变量）

| # | 步骤 | 操作位置 | 产出物 → env 变量 |
|---|---|---|---|
| 1 | 签约「当面付」产品（扫码支付能力）；确认费率（当面付公开价常见 0.38%~0.6%，以签约页为准）〔待确认〕 | 支付宝商家平台 b.alipay.com → 产品中心 | 无 env；费率记录进部署清单 |
| 2 | 创建「网页/移动应用」并添加「当面付」能力 | 支付宝开放平台 open.alipay.com → 控制台 → 创建应用 | APPID → `ALIPAY_APP_ID` |
| 3 | 本地生成 RSA2 应用密钥对（官方「支付宝开放平台密钥工具」，选 RSA2/2048/PKCS8）；**私钥只落生产服务器，从不上传、不进聊天** | 本地/服务器运行密钥工具 | 应用私钥 → `ALIPAY_APP_PRIVATE_KEY_PATH`（或 `_PEM` 内联） |
| 4 | 上传应用公钥，选择加签方式为**「公钥模式」**（不要选「公钥证书模式」——当前代码按公钥模式实现，只读 `ALIPAY_PUBLIC_KEY`，没有证书链逻辑） | 开放平台 → 应用详情 → 开发设置 → 接口加签方式 | 页面回显的**支付宝公钥**（不是你上传的应用公钥）→ `ALIPAY_PUBLIC_KEY_PATH`（或 `_PEM`） |
| 5 | 应用提交审核并上线；把当面付能力与商家账户签约关联（绑定商户） | 开放平台应用详情 | 无 env |
| 6 | 异步通知地址：由每笔 precreate 请求携带 `notify_url = {PAYMENT_NOTIFY_BASE_URL}/api/v1/payment/callback/alipay`，要求 https 公网可达；开放平台「应用网关」配置项对当面付异步通知是否必填〔待确认：一般仅生活号等场景需要〕 | — | 域名 → `PAYMENT_NOTIFY_BASE_URL` |
| 7 | 确认退款能力（`alipay.trade.refund` 随当面付默认可用，W-B 依赖）〔待确认〕 | 开放平台接口列表 | 无 env |

### 3. 支付宝侧自查

- [ ] 应用状态「已上线」，当面付能力「已生效」
- [ ] 加签方式为「公钥模式」，页面支付宝公钥与服务器 env 一致（贴错成应用公钥是最常见错误——用它验 notify 必失败）
- [ ] APPID 与签约商家主体对应

## 三、常见混淆对照（贴 env 前最后一遍核对）

| 容易搞混的两个东西 | 区别 |
|---|---|
| 微信「APIv3 密钥」 vs 「商户 API 证书私钥」 | 前者是 32 字符对称密钥（回调解密）→ `WECHAT_PAY_APIV3_KEY`；后者是 RSA 私钥 PEM（请求签名）→ `WECHAT_PAY_PRIVATE_KEY_*` |
| 微信「商户证书序列号」 vs 「微信支付公钥 ID」 | 前者标识**你的**证书（请求头 serial_no）→ `WECHAT_PAY_MCH_SERIAL_NO`；后者标识**微信的**公钥（回调 Wechatpay-Serial 命中）→ `WECHAT_PAY_PUBLIC_KEY_ID` |
| 支付宝「应用公钥」 vs 「支付宝公钥」 | 前者是你生成上传的，服务器**不需要**；后者是平台回显的，用于验签 → `ALIPAY_PUBLIC_KEY_*` |
| 支付宝「公钥模式」 vs 「公钥证书模式」 | 代码只支持前者；选错模式验签口径完全不同 |
