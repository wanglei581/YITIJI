## 四、扫描集成方案

由于设备不支持云端远程发起扫描，扫描功能有以下三种集成方案：

### 方案 A：Windows 本地驱动调用（推荐）

- Terminal Agent（运行在 Windows 一体机本地）直接调用 TWAIN/WIA 扫描驱动
- 扫描完成后 Agent 将文件上传到后端存储
- 优点：体验最顺畅，无需用户手动操作
- 要求：Terminal Agent 必须在 Windows 环境运行，已安装奔图驱动

### 方案 B：打印机扫描到 SMB/FTP 文件夹，系统监听

- 配置打印机将扫描文件发送到 SMB 共享文件夹或 FTP 服务器
- Terminal Agent 监听该目录，检测到新文件后自动上传
- 优点：不依赖 TWAIN 驱动，网络打印机通用方案
- 要求：需配置网络共享文件夹或 FTP 服务

### 方案 C：用户扫描到 U 盘后手动上传

- 用户将文件扫描到 U 盘
- 一体机读取 U 盘后，用户选择文件上传
- 优点：最简单，无额外配置
- 缺点：需用户手动操作，体验较差
- 适用场景：备用方案

## 五、打印集成方案

### 方案 A：Windows 本地驱动调用（推荐用于一体机）

- Terminal Agent 拉取打印任务
- 下载文件到本地临时目录
- 调用 Windows 打印 API 发送到奔图打印机
- 打印完成后上报状态

### 奔图 API 说明

奔图有官方云打印 API（适用于部分型号）。若 CM2820ADN 支持，需注意：

- appKey / appSecret 只保存在服务端
- 前端不得接触签名逻辑
- 详见安全要求：[compliance-boundary.md](../product/compliance-boundary.md#六打印接口安全要求)

## 六、设备状态监控

Terminal Agent 应定期上报以下状态：

| 状态项 | 说明 |
|--------|------|
| 在线/离线 | 打印机网络连通性 |
| 纸张状态 | 是否缺纸 |
| 墨粉状态 | 各色墨粉余量 |
| 故障代码 | 卡纸、开盖等硬件故障 |
| 打印队列长度 | 当前待打印任务数 |

## 七、开发注意事项

1. **不要假设 A3**：所有打印功能设计只考虑 A4 及以下纸张。
2. **不要假设 WiFi**：设备只有有线网络，一体机必须通过网线连接。
3. **不要假设远程扫描 API**：扫描由本地驱动或文件目录监听实现。
4. **测试环境**：开发阶段可用虚拟打印机（如 PDF 打印机）模拟，上线前必须在真机测试。
5. **驱动安装**：Windows 一体机部署时需预装奔图 CM2820ADN 驱动程序。
