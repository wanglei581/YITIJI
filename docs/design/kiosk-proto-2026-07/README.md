# Kiosk 前台高保真原型 · 75 屏(2026-07 定稿)

> **本目录是 Kiosk 前台用户页面的唯一视觉与功能基准。前端开发一律以这里的每一屏为准 1:1 实现——布局、字号、间距、颜色、文案、状态、触控尺寸全部照做,不自行发挥。**

## 一、这套原型是什么

- 覆盖项目全部 73 条 Kiosk 路由对应的真实功能 + 2 屏商用补缺(60 会话超时 / 61 断网异常),共 **75 屏**
- 画布 1080×1920 竖屏(27 寸触控一体机);62/63 为 390×844 手机落地页
- 视觉体系:fusion-youth 青绿米纸 V1 定稿(圆角 14-18px、轻阴影、品类色卡片、宋体标题)
- 生产流程:Claude 手写 → 对照 `apps/kiosk/src/pages/` 真实源码逐屏补功能 → Claude×Codex(GPT-5.6-sol xhigh)双模型三轮联审 → 75 屏布局体检全绿(零出界/零裁切/零折行)
- 零外部依赖:无 CDN/远程字体/图片,断网可渲染;图标全部内联 SVG 线性(24×24, stroke 1.6)

## 四、开发实现规则(强制)

1. **1:1 还原**:每屏的区块结构、文案、状态、示例数据形态照原型实现;差异需先在评审中提出,不得默默改
2. **token 化**:`shared.css` 的 `:root` 变量对应迁移为项目 CSS 变量/Tailwind token;禁止硬编码色值
3. **组件对齐**:tile/card/chip/btn/steps 等公共组件一次实现、全站复用;同名组件不允许两种长相
4. **触控标准**:主按钮 ≥56px、可点击 ≥48px,不得因还原走样
5. **诚实状态**:原型中的"待设备能力开放/即将上线/后台可配/校方开启"等标注必须保留语义,不得虚构已开通
6. **动态专区规则**(01 首页底部):百宝箱/智慧校园由后台开关驱动;两个都开=左右半栏,只开一个=`:only-child` 自动通栏并多露预览签,都关=整行不渲染(规则已写在 01-home.html 注释)
7. **验收方式**:开发完成后与原型同屏截图对比(1080×1920),布局偏差超过 ±4px 视为未达标

## 五、合规红线(与 CLAUDE.md / docs/compliance 一致,不可违反)

- 按钮白名单:查看岗位、去来源平台投递、扫码投递、查看招聘会、去来源平台预约、扫码预约;**禁止**一键投递/立即投递/平台投递/收简历/候选人管理
- 岗位/招聘会/企业信息必须展示来源机构、同步时间、外部ID
- 岗位匹配只用三档参考(较高/中等/偏低),禁止展示百分比
- 线下招聘机构(74/75 屏):仅信息展示 + 到店咨询指引 + 打印带走;不代收简历、不代收费用、不做预约登记
- 金额一律标注"以现场公示价为准";不硬编码打印机型号(用 `printerName` 配置)

## 六、与真实源码的关系

原型逐屏对照 `apps/kiosk/src/pages/` 的真实组件产出(功能区块、按钮、状态、API 契约字段),但**原型是设计定稿,不是代码搬运**——个别屏按商用完整性做了超出现状的规划(60/61 系统屏、74/75 线下机构双轨),开发时属于新增功能,按正常任务流程立项。
