# 视觉设计参考标准

> 本文档为 AI求职打印服务终端的历史视觉设计参考，不再作为所有页面必须遵守的硬性规范。
> 适用于：kiosk（一体机前台）、admin（管理员后台）、partner（合作机构后台）
> 当前页面 UI 可按具体任务、设计稿或用户确认方向覆盖本文档中的颜色、圆角、阴影、动效、卡片和图标建议。
> 不可覆盖的底线仅包括：合规边界、真实数据 / 真实状态表达、触控可达性、可访问性、硬件安全和隐私安全。
> 参考：Linear、Stripe Dashboard、Notion、Vercel Dashboard

---

## 一、核心原则

**历史设计目标：真实、克制、专业、可信赖**

| 原则 | 说明 |
|------|------|
| 真实 | 不追求炫技，每个元素都有存在的理由 |
| 克制 | 颜色、阴影、动画都要有节制地使用 |
| 专业 | 像运营级工具软件，不是展示型网站 |
| 可信赖 | 用户信任这个系统来处理他们的简历和打印 |

**旧版保守基线（非硬性限制）：**
- 紫色 / 蓝色渐变背景
- 卡片发光效果（box-shadow 彩色 glow）
- 毛玻璃效果（backdrop-blur 大面积使用）
- 霓虹色文字
- 每个模块都用主色强调
- emoji 作为 UI 图标
- 过度圆角（>16px）
- 浮动粒子 / 动态背景
- AI 机器人 / 对话气泡作为主要 UI 元素

以上不再作为绝对禁止项。实际页面可以按当前设计目标使用这些手法，但必须保证文字可读、性能可接受、状态真实、合规边界不被弱化。

## 二、颜色系统

### 2.1 主色（Primary / 科技蓝）

**旧版建议：主色优先用于关键操作按钮、选中态、当前态指示**

```
Primary 700   #1d4ed8  — 主要按钮 hover
Primary 800   #1e40af  — 主要按钮 active
Primary 50    #eff6ff  — Primary 背景浅色（用于 badge 背景等）
```

| 端 | 背景色 | 说明 |
|----|--------|------|

## 三、字体系统

### 3.1 字号阶梯

**27 寸竖屏触控显示器 + 桌面后台，一体机阅读距离 50-80cm**

| 级别 | 字号 | 行高 | 字重 | 使用场景 |
|------|------|------|------|---------|
| text-xs | 12px | 1.5 | 400 | 标签文字、辅助说明 |
| text-sm | 14px | 1.5 | 400 | 次要信息、表格内容 |
| text-base | 16px | 1.6 | 400 | 正文（Kiosk 主要文字） |
| text-lg | 18px | 1.6 | 500 | 卡片标题、模块标题 |
| text-xl | 20px | 1.5 | 600 | 页面主标题 |
| text-2xl | 24px | 1.4 | 700 | 一级大标题 |
| text-3xl | 30px | 1.3 | 700 | Hero 标题（仅 Kiosk 首页用） |

**后台（Admin/Partner）text-base 可以降为 14px，因为是桌面操作距离。**

### 3.2 字重使用规范

| 字重 | 使用场景 |
|------|---------|
| 400（Regular） | 正文、说明文字 |
| 500（Medium） | 需要强调的正文、次级标题 |
| 600（Semibold） | 模块标题、侧栏菜单 |
| 700（Bold） | 页面标题、数字强调 |

### 3.3 字体族

```
font-sans: 'Inter', 'PingFang SC', 'Noto Sans SC', ui-sans-serif, system-ui, sans-serif
```

## 四、间距系统

**基于 4px 基准，复用 Tailwind spacing scale**

| Token | 实际值 | 使用场景 |
|-------|--------|---------|
| 1（4px） | 4px | 紧凑元素内部间距 |
| 2（8px） | 8px | 小间距、表单项内 |
| 3（12px） | 12px | 标准元素内间距 |
| 4（16px） | 16px | 卡片内间距、表单项间距 |
| 5（20px） | 20px | 模块内间距 |
| 6（24px） | 24px | 区块间距 |
| 8（32px） | 32px | 大区块间距 |
| 12（48px） | 48px | 大区块间距、触控安全区 |

**触控安全区：主要操作按钮四周不低于 16px 空隙，避免误触。**

---

## 五、圆角系统

| Token | 实际值 | 使用场景 |
|-------|--------|---------|
| radius-sm | 6px | Badge、标签、小型输入框 |
| radius-md | 8px | 按钮、标准卡片、标准输入框 |
| radius-lg | 12px | 大卡片、面板、弹窗背景 |
| radius-xl | 16px | 大弹窗、特殊卡片（仅必要时用） |
| radius-full | 9999px | 胶囊按钮、圆角头像 |

**原则：不要对所有元素都用同一个圆角值。按钮 8px、卡片 12px、badge 6px，分层清晰。**

---

## 六、阴影系统

**克制使用阴影，不要让页面看起来"浮在空中"**

| Token | 实际值 | 使用场景 |
|-------|--------|---------|
| shadow-none | none | 无阴影，不要用 |
| shadow-sm | 0 1px 2px 0 rgb(0 0 0 / 0.05) | 卡片默认阴影（白底+1px border+shadow-sm） |
| shadow-md | 0 4px 6px -1px rgb(0 0 0 / 0.1) | 弹窗、下拉菜单 |
| shadow-lg | 0 10px 15px -3px rgb(0 0 0 / 0.1) | Modal、浮层 |

**卡片标准样式：白底 + 1px Gray 200 边框 + shadow-sm**

## 九、组件视觉标准

### 9.1 按钮（Button）

| 类型 | 样式 | 使用场景 |
|------|------|---------|
| primary | 蓝色背景（Primary 600）+ 白字 | 主要操作（打印、确认、提交） |
| secondary | 灰色背景（Gray 100）+ 深色字 | 次要操作（取消、返回） |
| ghost | 透明背景 + hover Gray 100 | 辅助操作（更多、设置） |
| danger | 红色背景（Error）+ 白字 | 危险操作（删除、清空） |
| outline | 1px 边框 + 透明背景 | 切换态、文案按钮 |

**触控按钮：h-14（56px），文字 text-base（16px）**

### 9.2 卡片（Card）

**标准样式：**
- 背景：白色（white）
- 边框：1px Gray 200
- 阴影：shadow-sm
- 圆角：radius-lg（12px）
- 内边距：p-5（20px）或 p-6（24px）
- 标题：text-lg font-medium
- 描述：text-sm text-gray-500

### 9.3 输入框（Input）

- 高度：h-12（48px）触控友好
- 边框：1px Gray 300，focus 时 2px Primary 500
- 圆角：radius-md（8px）
- 错误态：边框红色 + 下方红色提示文字

### 9.4 表格（Table）

- 表头：bg-gray-50，文字 Gray 500，text-sm font-medium
- 行高：56px（触控友好）
- 斑马纹：奇数行 bg-white，偶数行 bg-gray-50
- 边框：底部 1px Gray 200

### 9.5 状态标签（StatusBadge）

| 状态 | 颜色 | 使用场景 |
|------|------|---------|
| 进行中 | Blue / bg-blue-50 text-blue-700 | 进行中、打印中 |
| 成功 | Green / bg-green-50 text-green-700 | 已完成、在线、正常 |
| 警告 | Orange / bg-orange-50 text-orange-700 | 待处理、缺纸、警告 |
| 失败 | Red / bg-red-50 text-red-700 | 失败、离线、错误 |
| 禁用 | Gray / bg-gray-100 text-gray-500 | 已取消、禁用 |

### 9.6 弹窗（Dialog/Drawer）

- 背景遮罩：black/50（rgba(0,0,0,0.5)）
- 弹窗背景：白色 radius-xl（16px）
- 阴影：shadow-lg
- 内边距：p-6（24px）
- 标题：text-xl font-semibold
- 关闭按钮：右上角 X，字号 20px

## 十、图标规范

**旧版建议使用 lucide-react 线性图标；新设计任务可选择其他图标体系或 emoji，但需保证语义清晰、风格一致、不会误导状态。**

| 规则 | 说明 |
|------|------|
| 图标风格 | 线性（stroke-based），24px 或 20px |
| 图标颜色 | 与文字同色，Gray 500 或 Gray 700 |
| 图标与文字间距 | 8px（gap-2） |
| 旧版不推荐 | emoji、带颜色的第三方图标库 |

## 十一、空态/加载/错误规范

### 11.1 空态（EmptyState）

- 图标：Gray 300，64px 或 48px
- 标题：text-lg font-medium Gray 800
- 描述：text-sm Gray 500，1-2 行
- 操作按钮：可选，放在描述下方

### 11.2 加载态（LoadingState）

- Spinner：Primary 500，32px
- 文字：text-sm Gray 500，"加载中…" / "处理中…"
- 全屏 LoadingOverlay：白色半透明遮罩 + 居中 Spinner + 文字

### 11.3 错误态（ErrorState）

- 图标：错误红色，64px 或 48px
- 标题：text-lg font-medium Gray 800，"出错了"
- 描述：text-sm Gray 500，具体错误信息
- 操作：重试按钮（primary）+ 可选返回按钮

## 十二、旧版限制清单（不再强制）

**以下做法是旧版保守视觉基线，不再作为项目硬性禁止项。若当前设计需要，可以在保证可读性、性能和合规边界的前提下使用。**

| 旧版限制项 | 旧版替代方案 |
|---------|---------|
| 渐变背景 | 纯色白底/浅灰底 |
| 卡片发光（box-shadow 带颜色 glow） | 白色卡片 + 1px边框 + shadow-sm |
| 大面积毛玻璃 | 必要时单层 backdrop-blur，不做装饰用 |
| 霓虹色文字 | Gray 700 / Gray 800 |
| 每个模块都用主色强调 | 只有关键操作/当前态用主色 |
| emoji 作为 UI 图标 | lucide-react 线性图标 |
| 圆角大于 16px | 圆角统一按圆角系统 |
| 浮动粒子/动态背景 | 不要任何动态装饰 |
| AI 机器人形象作为主要 UI | 纯文字/线性图标 |

## 十三、参考验收标准

实现每个页面时可参考以下自检：
- [ ] 如使用渐变、发光、毛玻璃或强视觉背景，能说明其信息层级目的，并保证文字可读和性能可接受
- [ ] 按钮颜色与当前页面主题一致，主次操作清晰
- [ ] 卡片、面板或区块样式服务于当前信息结构，不强制固定为 1px 边框 + shadow-sm
- [ ] 触控区域不小于 48×48px
- [ ] 字体大小和行高保证阅读距离下可读
- [ ] 状态标签表达真实状态，不用颜色误导成功 / 失败 / 离线
- [ ] 空态、加载、错误态不伪造业务结果
- [ ] 图标体系可按当前设计选择，不强制 lucide-react
- [ ] 页面可使用 `tokens.css` 已定义语义 token，也可在任务范围内扩展主题 token
- [ ] Admin/Partner 保持可操作的信息密度，除非当前任务明确是展示型页面
- [ ] Kiosk 主操作路径不超过 3 步，主要按钮高度优先 56px

## 十四、Phase 9 视觉升级补充规范

> **适用范围：Kiosk 一体机前台**  
> **版本：Phase 9**  
> **与前述规范的关系：本节是对第一至第十三节的历史补充和局部调整。前述规范中的"禁止渐变/毛玻璃/发光"条款曾在本节调整为"可控使用"；现在本节整体降级为参考，不再精确定义绝对禁止范围。后台（Admin/Partner）也可按当前任务重新设计。**

---

### 14.1 调整背景

旧版规范曾全面禁止渐变、毛玻璃和发光效果，使 Kiosk 前台视觉层级偏平。触控显示器屏幕尺寸大（27 寸竖屏）、亮度高、观看距离近（50–80cm），适度的视觉层次感有助于用户快速识别核心功能入口，而不是视觉炫技。

参考原则：**有目的地使用，而不是有克制地禁止。** 渐变、半透明、发光等效果建议服务于信息层级、引导视线或强化功能识别；若当前设计任务需要纯氛围表达，也应保证不遮挡内容、不误导状态、不拖慢设备。

---

### 14.2 功能色系统（Kiosk 专用）

Kiosk 六大核心模块各配一套功能色。功能色用于模块图标背景、区块点缀，不用于大面积填色。

| 模块 | 功能色 | Tailwind 参考 | 用途示例 |
|------|--------|--------------|---------|
| AI简历服务 | 蓝色 | `blue-500 / blue-50` | 图标底色、选中背景 |
| 打印扫描 | 青色/蓝绿 | `cyan-500 / cyan-50` | 图标底色、进度指示 |
| 招聘会信息 | 橙色 | `orange-500 / orange-50` | 图标底色、日历标记 |
| 岗位信息 | 绿色 | `green-500 / green-50` | 图标底色、状态标记 |
| 政策服务 | 金色/红橙 | `amber-500 / amber-50` | 图标底色、公告标记 |
| AI助手/面试训练 | 紫色 | `violet-500 / violet-50` | 图标底色、对话指示 |
| 我的记录 | 蓝色体系 | `primary-500 / primary-50` | 身份区渐变基色 |

**旧版约束（现为参考）：**
- 同一页面主视觉色建议控制在 3 个左右（背景中性色不计入）
- 功能色分类建议控制在 6 类左右，新模块也可按设计任务扩展颜色
- 功能色可做点缀，也可按当前设计任务扩大使用范围
- 语义色（成功绿、警告橙、错误红）与功能色冲突时，语义色优先

---

### 14.3 渐变使用参考

#### 允许的场景

| 场景 | 规格 | 示例 |
|------|------|------|
| 首页顶部身份区 | 线性渐变，从 `primary-600` 到 `primary-700`，高度建议不超过屏幕 1/4 | 欢迎语、设备状态 |
| 我的页面顶部身份区 | 同上，可叠加用户头像/姓名 | 登录态 banner |
| 重要 CTA 区块 | 渐变仅用于单一按钮或小面积行动引导卡片，不铺满整屏 | "开始扫描简历" 按钮组 |
| 功能图标小面积背景 | `from-{color}-400 to-{color}-600`，旧版图标容器约 56×56px | 首页六大功能图标背景 |

#### 旧版不推荐的场景

| 旧版不推荐做法 | 原因 |
|---------|------|
| 大面积全屏渐变背景 | 文字对比度不可控，违背"可信赖"原则 |
| 无功能目的的装饰渐变 | 增加视觉噪音 |
| 紫蓝 SaaS 大背景渐变 | 与系统定位（公共服务工具）不符 |
| 多色混合渐变 | 超过 2 个颜色端点即为滥用 |
| 渐变文字（`bg-clip-text`） | 可能影响可读性 |

---

### 14.4 毛玻璃使用参考

#### 允许的场景

| 场景 | 规格 | 示例 |
|------|------|------|
| 小面积浮层 | `backdrop-blur-sm`，旧版建议浮层面积 ≤1/4 屏幕 | 状态弹出提示、快捷操作浮层 |
| 列表项选中态背景 | `bg-white/60 backdrop-blur-sm`，仅限选中行 | 招聘会列表当前选中项 |
| 顶部状态卡轻量半透明 | `bg-white/80` 不开 blur，或 `bg-white/60 backdrop-blur-sm` | 状态 banner |

#### 旧版不推荐的场景

| 旧版不推荐做法 | 原因 |
|---------|------|
| 整页毛玻璃 | 文字可读性严重下降 |
| 卡片套卡片毛玻璃 | 多层半透明叠加导致混乱 |
| 毛玻璃作为纯装饰层 | 无信息层级区分目的不用 |
| 深色底 + 白字 + blur 全屏遮罩做常驻 UI | 仅弹窗/Modal 遮罩场景允许 |

### 14.5 发光与阴影参考

#### 允许的场景

| 场景 | 规格 | 示例 |
|------|------|------|
| 图标容器低透明度彩色底光 | `shadow-[0_4px_12px_0_rgb({color}/0.15)]`，旧版建议透明度 ≤15% | 首页功能图标悬浮状态 |
| 关键状态卡片轻微彩色光晕 | 旧版建议透明度 ≤10%，颜色与功能色对应 | 设备在线状态卡 |
| 按钮 hover 轻微提升 | `shadow-md`，无彩色，纯灰黑阴影 | 主操作按钮 hover |

#### 旧版不推荐的场景

| 旧版不推荐做法 | 原因 |
|---------|------|
| 霓虹发光（`shadow-[0_0_20px]` 高透明度彩色） | 违背"可信赖、专业"原则 |
| 大面积彩色阴影 | 视觉污染 |
| 卡片强烈悬浮（`shadow-2xl` 或更大） | 破坏页面整体层次感 |
| 纯装饰性发光圆圈/光晕动画 | 无信息价值 |

### 14.6 首页视觉升级方向

当前首页六个入口视觉层级平，只有文字和图标，缺乏模块识别度。Phase 9 升级方向：

#### 入口卡片结构

```
┌─────────────────────────────────────┐
│  [功能色图标容器 56×56px]            │
│  模块名称 text-base font-semibold    │
│  副标题/子功能数量 text-sm text-gray-500 │
│                                     │
│  子功能入口（3-6 个）               │
│  [图标] 功能名  [图标] 功能名  ...  │
└─────────────────────────────────────┘
```

#### 信息层级要求

- **主功能**（6 个模块）：功能色图标 + 模块名 + 简述，卡片样式
- **次功能**（每模块 3–6 个）：小图标 + 文字，内嵌于主功能卡片
- **主次层级建议通过尺寸、字重、色彩等多种方式同时区分，不强制固定组合**

#### 顶部身份区

- 已登录：小头像 + 姓名 + 今日数据（如剩余打印量）
- 游客：简短欢迎语 + "快速开始" 引导
- 顶部身份区高度控制在屏幕 20% 以内，不喧宾夺主

#### 设备状态区

- 弱化至右上角小徽标或折叠区域
- 异常时才扩展显示详情

### 14.7 我的页面视觉升级方向

#### 顶部身份区

- 已登录：渐变底色（`primary-600 → primary-700`） + 头像 + 姓名 + 注销按钮
- 游客：中性色底 + "未登录"提示 + 扫码/手机号登录入口
- 高度：120–160px

#### 四大资产区

每个资产区有独立标题行 + 数量徽标 + "查看全部" 链接：

| 资产区 | 内容 | 空态处理 |
|--------|------|---------|
| 我的简历 | 已上传/扫描的简历文件列表 | "暂无简历，去上传" |
| 扫描文件 | 近期扫描文件，标注保存期限 | "暂无扫描文件" |
| 打印订单 | 近期打印记录，含金额和状态 | "暂无打印记录" |
| AI服务记录 | AI解析/优化历史，标注有效期 | "暂无AI服务记录" |

#### 敏感文件清理提示

- 每个文件显示剩余保留时长（如"7天后自动删除"）
- 用户可手动提前删除
- 已过期文件用 `text-red-500` 标注，不展示文件内容入口

#### 游客 vs 登录模式

- **游客模式**：只显示"我的记录"标题 + 登录引导卡片，资产区全部隐藏
- **登录模式**：完整显示四大资产区，顶部渐变身份区展开

### 14.8 Phase 9 验收标准

实现 Phase 9 任何页面时，除参考第十三节通用标准外，还需自检：

- [ ] 页面视觉方向符合当前任务和用户确认，不再受旧版“克制风格”固定限制
- [ ] 颜色、渐变、毛玻璃、发光或动态背景服务于信息层级，不遮挡关键操作
- [ ] 当前页面的视觉复杂度不会影响触控命中、阅读和设备性能
- [ ] 首页主功能与次功能层级建议通过尺寸、字重、色彩等多种方式区分
- [ ] 触控按钮仍保持 ≥48px，主要按钮 ≥56px
- [ ] 所有文字在对应背景上对比度符合 WCAG AA（普通文字 4.5:1，大文字 3:1）
- [ ] 业务逻辑、合规边界、按钮文案规范（去来源平台投递/扫码投递等）不受视觉升级影响
- [ ] Admin 和 Partner 后台页面如引入强视觉效果，不降低表格、筛选、审核和运营任务效率

## 十五、Kiosk 首页历史参考（2026-06-08）

> **状态：历史参考。** 本节记录 2026-06-08 的 Kiosk 首页实现依据，不再作为唯一锁定标准。后续首页 UI 可以按新的设计任务重做，但必须继续保留真实功能、合规文案、状态诚实和触控可用性。
> **与 §14.6 的关系：** §14.6 为 Phase 9 探索方向，本节为当时定稿记录。后续如与新设计任务冲突，以用户最新确认的设计目标和合规 / 真实状态底线为准。
> **决策来源：** 当时选择去除"秒哒 AI 味"（摆拍 Hero / 毛玻璃浮卡 / 渐变按钮 / 营销标语）+ 克制打底 + 竖屏 + 严守合规边界（CLAUDE.md §2 / §10）。

### 15.1 设备方向与画布

- 一体机为**竖屏 9:16**（约 1080×1920），首页**纵向滚动**。
- 画布背景 `bg-canvas`（neutral-50 `#f9fafb`）。
- 深色 Hero 大色块、整屏渐变、图片背景等可在新设计任务中重新评估；使用时必须保证文案可读、操作可见和状态真实。

### 15.2 顶栏（机器头顶状态栏）

- 首页由 `HomePage` 自带**机器头顶状态栏**，并在 `KioskRoot` 对 `/` 设置 `hideHeader`，避免与全局 `KioskLayout` header 重复；其余页面继续使用全局 header。
- **左**：一体机名 **「AI求职打印服务终端」**（项目正式名，已确认）。
- **右**：状态栏 + **实时时间**。状态项 = 打印机 / 网络（语义色圆点 + lucide 线性图标 + 文案），时间为实时 `HH:MM`（可带星期/日期）。
- **状态诚实**：首页无 `terminalId` 心跳上下文时，打印机/网络显示**中性**（"以打印前检测为准"），**不写死"正常"绿点**；真实状态在打印流程页通过心跳查询。

### 15.3 身份区（登录前后两态 + 匿名态）

- 复用现有 `AuthContext` 三态：**未登录（初始）/ 匿名使用（guestMode）/ 已登录**。"保留登录前后两态"= 三个子态都要在首页清晰呈现，不得删减。
- **已登录**：展示脱敏身份、"进入我的"、"退出登录"和**数据概览**；视觉形式可按当前设计调整。
- **数据概览数字规则（强约束）**：
  - 只接**后端允许记录字段**：简历 / 文档 / AI服务记录 / 收藏（来源 `services/api/memberAssets`：`getMyResumes / getMyDocuments / getMyAiRecords` + `memberFavorites`：`getMyFavorites`）。
  - **仅登录态**凭本人 token 拉取真实计数；为 0 显示 0；加载中显示骨架/`—`。
  - **禁止占位假数据**（如 27/3/12/862）；**禁止**"已投递 / 待面试 / 完整度"等编造或越界字段。
  - 不叠加"本次会话记录"，避免与会话记录双算（与 `ProfilePage` 一致）。
- **匿名 / 未登录**：登录引导卡（中性，**不强推权益**），文案如"登录后可查看本人历史记录"/"先使用"。

### 15.4 两级信息架构（大模块 → 子功能瓦片）

**铁律：首页子功能瓦片必须镜像各模块服务中心页「真实存在」的功能，禁止臆造不存在的小功能。** 新增 / 下线子功能时，先改服务中心页，再同步首页瓦片。"即将上线"沿用中心页占位态（带标记，可跳说明页或置灰不可点）。

有真实子功能的四大模块：

| 模块 | 功能色 | 入口 | 子功能瓦片（镜像来源 → 路由）|
|------|--------|------|------|
| AI简历服务 | 蓝 | `/resume` | 镜像 `ResumeHomePage.ENTRIES`：AI简历诊断→`/resume/source` · AI简历优化→`/resume/source` · 简历素材库→`/resume/templates` · 面试准备（即将上线，不可点） |
| 打印扫描 | 青 | `/print-scan` | 镜像 `PrintScanHomePage.CAPABILITIES`：文档打印→`/print/upload` · 材料扫描→`/scan/start` · 照片打印→`/print/upload`(state.category=photo) · 证件照 / 格式转换 / 签名盖章→`/print-scan/feature/:key`（即将上线） |
| 岗位信息 | 绿 | `/jobs` | 镜像 `JobsPage.TYPE_OPTIONS`：全职→`/jobs?category=fulltime` · 实习→`?intern` · 校招→`?campus` · 兼职→`?parttime` |
| 政策服务（人社） | 金 amber | `/renshi` | 镜像 `RenshiPage` 四 Tab：就业政策→`/renshi?tab=policy` · 社保指南→`?social` · 就业登记→`?register` · 政策公告→`?notice` |

- 深链约定：岗位用 `?category=`（值 ∈ fulltime / intern / campus / parttime），政策用 `?tab=`（policy / social / register / notice）；两页均由 `useSearchParams` 读取并同步 URL 状态，非法值回退默认态。
- **合规命名（红线）**：不得出现非白名单投递/预约语义（如"一键投递 / 立即投递 / 平台投递 / 名企直达 / 智能匹配 / 简历直投 / 求职平台 / 面试邀约"等）；岗位详情按钮用 **查看岗位 / 去来源平台投递 / 扫码投递**；招聘会用 **查看招聘会 / 去来源平台预约 / 扫码预约**。

### 15.5 无独立子功能的模块与附加专区

- **招聘会信息** `/job-fairs`：本质是列表 + 状态筛选，**无独立子功能页** → 首页用**单行入口**（不臆造"招聘会预约 / 扫码签到"等子功能），保留功能色橙。
- **校园招聘专区** `/campus`、**AI在青岛** `/qingdao`：单行入口保留。
- **AI助手** `/assistant`：既在底部导航，也保留首页引导入口。

### 15.6 历史表皮参考

- 当时实现采用白底、细边框、低阴影和功能色点缀。
- 后续首页 UI 不再受此表皮限制，可以使用新的色彩、背景、渐变、毛玻璃、插画、图标和动效体系。
- 无论采用何种表皮，必须保留真实入口、真实状态、合规命名和可点击区域。

### 15.7 触控与可访问

- 主入口/按钮触控高度 **≥56px**，所有可点区域 **≥48px**，瓦片整块可点。
- 文案对比度满足 WCAG AA；图标体系可按当前设计选择。

### 15.8 首页验收（叠加 §13 / §14.8）

- [ ] 顶栏左为「AI求职打印服务终端」，右有状态栏 + **实时时间**
- [ ] 三态身份区齐全；已登录概览数字**全部来自后端真实计数，无任何占位假数据**
- [ ] 四大模块两级瓦片 + 招聘会单行入口；命名无越界；功能色按 §14.2（政策=amber）
- [ ] 校园 / 青岛 / AI助手入口未丢
- [ ] 当前视觉方案符合用户确认方向；如使用渐变、毛玻璃或彩色发光，不影响可读性、性能和状态表达
- [ ] 触控 ≥56 / 48px；竖屏纵向滚动正常
